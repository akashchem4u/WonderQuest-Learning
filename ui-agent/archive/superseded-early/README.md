# Superseded Early UI Files

These files were useful exploration steps, but they are no longer the preferred
top-level references for WonderQuest implementation.

Engineering should treat the current top-level `*-canonical-v2.html` and latest
`*-v2.html` route slices as the main source of truth instead.

Files in this folder stay available for:

- historical comparison
- niche layout ideas worth reusing later
- understanding how a route evolved before canonicalization

They should not be the first files opened when implementing or refreshing a
live route.
