Engineering note — Priority 24 / Tab 1: Counting answer cards Counting cards show objects visually (emoji dots) + a number below. Labels are secondary to the visual. Card sizes: 2×2 grid for 4 options, or 2×1 for 2 options. Object emojis scale with count: small counts use larger dots. Zero accuracy percentage anywhere. Correct state: green border + checkmark. Retry state: soft red border + shake animation, NO lockout — child can try again immediately.

Engineering note — Priority 24 / Tab 5: Shared card states All 4 card types share the same 4 states: (1) Default — neutral purple border. (2) Selected — darker purple border + subtle bg glow. (3) Correct — green border + green checkmark badge + correctPop animation. (4) Retry — soft red border + ✕ badge + try-again banner below prompt — never say "wrong", never lock out. Cards not selected = disabled (opacity .35) after answer is confirmed. No accuracy % ever shown.

State rules
• Retry → cards remain tappable, try again immediately
• Correct → all other cards go to disabled (opacity .35)
• Never show accuracy % in any state
• correctPop: scale 1→1.06→1 over 300ms
• Retry shake: translateX ±4px 3 times over 300ms
• CTA button: disabled until a card is selected
