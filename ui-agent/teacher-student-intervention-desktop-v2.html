teacher-student-intervention-desktop-v2.html — Engineering Spec

Purpose: Deeper teacher action surface for a single student at intervention level. Used when a teacher needs to seriously address a student's needs.
Persona: Teacher · Sora font · #f0f4f8 bg · #2a5080 nav · desktop min 1024px
Layout: 3-col CSS grid: 260px 1fr 300px
SNS tokens: Flag = #9a2a2a bg / #ff9090 text  |  Watch = #2a1800 bg / #f0a030 text  |  Strong = #0a2a15 bg / #50e890 text
API Routes
MethodRoutePurpose
GET/teacher/student/:id/fullFull student profile, stats, SNS history, skill snapshot
GET/teacher/student/:id/notesAppend-only notes timeline (all types, teacher-visible only)
POST/teacher/student/:id/notesAppend new note (type + text + creator)
GET/teacher/student/:id/intervention-planFull intervention plan (goal, skills, strategies, check-ins)
PUT/teacher/intervention-plan/:idSave plan changes (goal text, skill/strategy edits)
GET/teacher/student/:id/assignmentsAll assignments ever assigned to this student, sortable
POST/teacher/student/:id/assignmentsCreate new assignment for student
Desktop Layout
Grid: 260px 1fr 300px
Left sidebar: student profile card, stats, world progress, risk flag alert
Center main: primary content per tab (chart / timeline / plan / assignments)
Right panel: contextual actions + summary cards per tab
Top nav: #2a5080 with breadcrumb: "Mrs. Rivera's Class › Emma Chen"
Subnav: white bar, student title + tier chip + quick action buttons
Min width: 1024px; columns collapse to 1-col below breakpoint
SNS History Chart
Data: 8-week weekly aggregation of skill SNS tiers
Color rule: bar color = dominant tier that week (most skills in that tier)
Stacked bars: each bar shows flag / watch / strong segments proportionally
Week selector: all 8 wks / 4 wks / 2 wks / this week chips
No decimals: counts only; no percentage labels on bars
Intervention Plan
Created: per-student; triggered by SNS Flag tier event (Watch → Flag)
Contains: goal text + focus skills (up to 3) + support strategies (up to 3) + check-in schedule
Duration: default 6 weeks; teacher can adjust
Editable: goal textarea + skill/strategy inline update links
Save: PUT on save-bar button; save bar is sticky bottom of center panel
Print/Export: link triggers PDF export (server-side rendered)
Notes System
Append-only: notes cannot be edited or deleted after saving
6 types: Note / Concern / Check-in / Activity / Parent Message / Tier Change
Visibility: teacher-only; not surfaced to parent app
Timeline: ordered newest-first; "Load more" pagination (10 per page)
Dot colors: note=navy, activity=teal, tier-change=amber, parent-msg=violet, concern=red, check-in=green
Assignment List
Scope: all assignments ever assigned to this student
Sort: default by status (overdue first, then in-progress, not started, complete); sortable
Status chips: Complete=#0a2a15/#50e890 | In Progress=#2a1800/#f0a030 | Not Started=neutral | Overdue=#9a2a2a/#ff9090
Actions: "View Results" (completed/in-progress) | "Reassign" (not started/overdue)
