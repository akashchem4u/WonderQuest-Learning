Engineering · Tab 5 — Spec Reference Full API contract, severity SLAs, incident state machine, alert taxonomy, auto-escalation rules, and design tokens for this surface. Use this as the source of truth during implementation and review.
API Routes
Incidents GET /owner/incidents POST /owner/incidents PUT /owner/incidents/:id
Alert Feed GET /owner/alerts/feed
Route Health GET /owner/routes/health
Incident Severity SLAs
P0 Immediate response required. SLA: 2h to resolution. Auto-escalate if no update in 1h.
P1 Urgent. SLA: 24h to resolution. Escalate if 50% SLA elapsed with no progress.
P2 Non-critical. SLA: 72h to resolution. Monitor for escalation if no owner activity.
Incident State Machine
Flow Open → Investigating → In Progress → Resolved → Post-Mortem
Transitions Any state can move to Resolved. Post-Mortem required for P0 after resolution.
Alert Types
Deploy New code pushed to an environment. Includes commit hash + service restart events.
Runtime Error rate spike / latency threshold breach / memory pressure. Real-time via WebSocket.
System Infrastructure events: node failure, CDN outage, DB failover. Requires owner ack.
Auto-Escalation Rules
P0 No-Update P0 incident not updated in 1h → auto-ping on-call + escalate severity flag.
SLA 50% SLA countdown at 50% elapsed → notify assigned team + add to triage queue.
Data Refresh
Route Health Poll every 2 minutes via GET /owner/routes/health.
Alert Feed Real-time via WebSocket. REST fallback: poll every 30s if WS disconnected.
Dashboard KPIs Rolling 24h window. Recalculated server-side every 5 min.
