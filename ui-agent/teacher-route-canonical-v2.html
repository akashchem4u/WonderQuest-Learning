📋 Class Support Queue — Spec Sorted: support first, then watch, then strong. Each row: 3px left severity bar + avatar + name + skill label + tier chip + time. Selected row highlights right-panel drilldown. Filter chips narrow the queue. No accuracy % shown.
Class Support Queue
Left column on desktop. Full-width on mobile. Always sorted support → watch → strong.

Row Anatomy

ElementValueNote
Severity bar3px left bordercoral=support, amber=watch, mint=strong

Sort & Filter rules

Default sort: support first (by urgency), then watch (by recency), then strong (alphabetical).
Filter chips narrow to one tier. "All" shows full list.
No accuracy % shown — ever. Tier is based on engagement + skill signal, not correct/incorrect ratio.
📅 Intervention Timeline — Spec Per-student chronological log. Three event types: note added, play session, skill state change. Newest first. Timeline lives in the right-panel drilldown on desktop; full-screen push on mobile.
Intervention Timeline
Per-student event log within the drilldown panel.

Note entry

Tap "Add note" CTA → bottom sheet on mobile, inline form on desktop. Text field (max 200 chars) + Save/Cancel. Note auto-tagged with teacher name + timestamp. No rich text — plain text only.
🔍 Filter / Segment Controls — Spec Filter row above queue. Three segments: Support / Watch / Strong. Filter by skill name from the skill drilldown view. Segment state persists within the session. "All" is always the default.

Skill filter (from Skills tab)

Tap a skill in the Skills tab → queue narrows to students with that skill active. Pill shows active filter (e.g. "Letter Sounds ×"). Tap × to clear. Queue re-sorts within the skill filter (support first).
