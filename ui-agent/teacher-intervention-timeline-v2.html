Teacher Intervention Timeline — Spec

Event types

TypeDot colorWho createsNotes
flag#ff7b6b coralSystem (auto)Triggered when student drops to Support tier. Non-deletable.
note#5586e0 blueTeacherFree-text observation. Editable within 24h.
contact#2c6fad navyTeacherParent/guardian contact log. Shows tags: Parent, Guardian.
progress#58e8c1 mintSystem (auto)Tier shift events. Non-deletable. Always shows old→new tier.
assign#9b72ff violetTeacherTargeted practice assignment to child's play queue.
system#aaa greySystemMisc system events (e.g., band transitions).
resolve#2ec49a tealTeacher or SystemTeacher-initiated or auto-resolved when Strength tier reached.

Sort order

DefaultRationale
Newest first (reverse chronological)Teachers need to see what just happened — recent notes most actionable
TODAY dividerGold horizontal divider marking current date, only shown for active interventions

Filter options

ChipEvents shown
All eventsAll types
Notesnote only
Parent contactcontact only
Progressprogress + flag + resolve
Systemsystem + flag + progress

Resolve rules

  • Teacher can manually resolve any active intervention at any time
  • System auto-resolves when student reaches Strength tier on the skill
  • Resolved interventions are archived (read-only) but remain accessible via "Resolved" section
  • Action footer: resolved state → single disabled "Resolved" button, no "+ Add note" CTA
  • Confirm modal must warn teacher if student has NOT reached Strength tier yet
