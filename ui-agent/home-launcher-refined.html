Engineering note: This is a real-time component. Poll /api/status every 30 seconds using setInterval. If the endpoint is unreachable or returns a non-200, hide the strip entirely — do not show a stale "healthy" status. Use an AbortController with a 5s timeout per request to avoid hanging polls.

Design intent: The status strip is secondary context — it supports trust and is not required for any user action. Keep it visually quiet (semi-transparent cream, no borders on calm state). Only the amber alert chip should draw attention, and only when truly degraded. Never show "System Healthy" if you aren't certain — silence (hidden strip) is better than false confidence.

└── ChipAmber: distinct bg + border variant, only when status = "degraded"

Engineering note: The trust strip is a static component — no data fetching required. Render it server-side. The four items and the age note are hardcoded strings, not CMS-driven, to ensure they are never accidentally removed or broken by content changes. TrustStrip — Specs
