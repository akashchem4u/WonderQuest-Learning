⚙ Engineering Hint — Tab 5: Spec Reference This tab is for engineers and designers. It summarizes mobile-specific decisions, API routes, design tokens, and UX rules for this assignment creator flow.
API Routes
Skill list GET /teacher/skills — same endpoint as desktop; add ?band= & ?q= filters
Student list GET /teacher/class/:id/students — returns tier, band, name
Create POST /assignments/create — full payload including send mode
Mobile vs Desktop Flow
Desktop 4 steps: Skill → Configure → Learners → Review → separate Send screen
Mobile 3 steps: Skill → Configure → Learners → Review+Send combined (one screen)
Why merged Reduces tap count; send options fit single scroll on 390px phone
Question Count Rules
Min3 questions
Max10 questions
Default5 questions
InputStepper (not slider) — safer for thumbs, avoids accidental mis-swipes
Difficulty Levels
GentleBelow-grade prompts; same logic as desktop
StandardGrade-level; default selection
ChallengeAbove-grade; stretch questions
Learner Group Chips
Whole ClassAll 24 students pre-selected
Needs SupportAlert tier (SNS) — 8 students
On TrackWatch + OK tier — 11 students
AdvancedTop band — 5 students
NoteSame SNS tier logic as desktop; chips are additive toggles

Touch targetsAll interactive elements ≥ 44px height
Skill inputVertical list — NOT grid (too cramped at 390px)
Student inputVertical list — NOT table
Q countStepper — NOT slider

Scrollscreen-body scrolls; tab bar and sticky bars remain fixed
