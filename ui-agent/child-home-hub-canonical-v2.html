Tab 5 — Engineering Spec
Full design spec, routes, priority order, safety rules, and design tokens for the Child Home Hub.
API Routes
Route Used by Response key fields
GET /player/home Tab 1 & Tab 2 activeQuest, streak, recentWins, dailyChallenge
GET /player/progress Tab 3 xp, level, xpToNextLevel
GET /player/rewards Tab 3 totalStars, badges, rewardHistory
GET /player/worlds Tab 4 worlds[] with status, starsEarned, currentNode
Home Hub Priority Order
1️⃣
Continue Quest — always dominant if activeQuest !== null. Hero card takes ~40% screen height. Breathe-animated gold CTA.
2️⃣
Start First World — shown if activeQuest === null. World picker row with 3 starter worlds. Float animation on rocket hero.
3️⃣
Daily Challenge teaser — compact card at bottom of home. Gold-tinted, "NEW" badge if not started today. Always below hero.
4️⃣
World Map shortcut — quick-action card in the 4-up grid. Secondary navigation, never promoted above hero.
Quick-Action Grid Rules
#EmojiLabelDestination
1🌟ChallengeDaily challenge screen
2🏅BadgesBadge shelf / collection
3🗺️WorldsWorld map
4⭐StarsStar count / reward shelf
Always 4 items. Always same order. Never reorder based on state.
Session Resume Logic
▶️
Continue = resumes at last incomplete node in current world.
✅
If last node is complete → auto-advance to next node on session start. Show "New node unlocked! 🎉" micro-toast.
🌍
If all nodes in world complete → prompt "World complete! Pick your next world?" with world picker.
Child Safety Rules
🚫
No timers on any child screen. No countdown pressure.
🚫
No grade labels (K, 1st, 2nd). World names only.
🚫
No skill IDs or internal codes visible to children.
🚫
No wrong-answer shame. No "incorrect" screens — just "try again!" framing.
🔒
Locked worlds = friendly prompt ("Finish Ocean World first! 🌊"). Never "You're not ready" or error state.
✅
Max 8 words on any visible label. Emoji-first. High-visual, low-text throughout.
