Teacher Support Queue — Mobile Spec v2
Fast intervention tool for between-class use. Target: act in under 60 seconds.

API Routes

GET /teacher/support-queue — Returns all support students for the authenticated teacher, sorted by tier (Flag first) then by days-inactive descending within tier. Includes student id, name, tier, skill gap, last active date, days in support.

POST /teacher/interventions — Create an intervention note. Body: { student_id, type, note_text, timestamp }. Types: Note / Activity / Concern / Parent Contact / Observation / Referral. Notes are append-only (immutable after save). Returns intervention_id.

POST /assignments/create — Create a practice assignment for one student. Body: { student_id, skill_id, question_count, difficulty, due_date }. Difficulty: gentle / standard / challenge. Due date: ISO 8601 timestamp. Assignment is private to student until teacher publishes. Returns assignment_id.

Queue Sort Logic

Tier 1 — Flag: Students with ≥ 2 weeks in support tier. Sorted within tier by days-inactive descending (most inactive first).

Tier 2 — Watch: Students with exactly 1 week in support tier. Same secondary sort by days-inactive.

Cleared: Students moved out of support this week — shown in collapsible section at bottom. Loaded lazily (separate endpoint). Not counted in main badge.

Mobile-Specific Rules

Min touch target: 48px on all interactive elements. No table layouts. No multi-column layouts on <440px viewport. All secondary content is collapsible. Horizontal scrolling allowed for chip rows only.

Swipe-right gesture: Reveals Quick Actions row (Note / Parent / Assign). Visual "→ Actions" hint shown on first load via localStorage flag wq_swipe_hint_seen. After first swipe, hint disappears. Implemented as CSS translateX transform, not a native gesture library.

Quick Assign due dates: Simplified to Tomorrow / This Week / Custom only. No full calendar picker on mobile. "Custom" opens native <input type="date">. "Tomorrow" = server-side now+1d at 23:59 local time. "This Week" = upcoming Sunday 23:59 local time.

Intervention notes: Append-only after save — no edit or delete. 6 supported types. Auto-focus textarea on screen load. Draft not persisted server-side; warn user on back navigation if text >0 chars.
