📋 Resolution Spec
Routes / API
EndpointPurpose
PATCH /feedback/:id/assignAssign to team + on-call member
PATCH /feedback/:id/resolveMark resolved with type + notes + related ID
PATCH /feedback/:id/reopenReopen with reason + new severity
POST /feedback/:id/notesAppend note with type (internal/escalation/owner)
GET /feedback/:id/notesFull append-only note log
GET /teams/oncallCurrent on-call member per team
Resolution State Machine
FromActionToRules
OpenAssignIn ProgressTeam + on-call required for P0/P1
In ProgressResolveResolvedResolution type + notes required
ResolvedArchiveArchivedNo re-open after archive
ResolvedReopenReopened→In ProgressReason + new severity required
AnyAdd Note(same state)Append-only; never edit/delete
Resolution Types
Fixed: Issue resolved; related PR/issue ID required for P0/P1.
Acknowledged: Known issue; added to backlog; no immediate fix.
Won't Fix: Out of scope or by design. Owner must note reason.
Duplicate: Duplicate of existing item; link canonical FB ID.
Escalation on Reopen
Auto-escalate severity: If reopened within 48h of resolution, bump severity by 1 level (P2→P1, P1→P0). Owner can override.
P0 reopen: Always remains P0 unless owner explicitly downgrades.
Notification: Slack ping to original assignee + on-call when reopened.
