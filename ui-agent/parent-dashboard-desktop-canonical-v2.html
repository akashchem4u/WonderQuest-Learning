Engineering · Tab 5 — Specification Reference This tab is the canonical design-engineering contract for the parent desktop dashboard. All other tabs must conform to these rules.
WonderQuest 🌟
Home Progress Messages Settings
Mon, Mar 23
SP
Parent Dashboard Desktop — Canonical Spec v2
Routes
  • GET /parent/dashboard — Dashboard home (Tab 1)
  • GET /parent/child/:id/summary — Progress view for active child (Tab 2)
  • GET /parent/messages — Message center (Tab 3)
  • PUT /parent/settings — Save notification and account settings (Tab 4)
Layout
  • Desktop surface — no phone frame. max-width: 1100px, centered, border-radius: 16px
  • Dashboard home: 320px sidebar + 1fr main two-column grid
  • Settings: 260px settings nav + 1fr panel two-column grid
  • Top nav: height: 56px, logo left, center nav links, date chip + avatar right
Parent Language Rules
✅ Use: stars earned, streak, badges, tier labels (support / growing / strength), trend notes (Getting stronger / Building foundations / Ready for more), world-name framing, growth language

❌ Never use: accuracy %, "missed", "weak", "failed", "incorrect", raw scores, performance percentages

Hero summary: 2 sentences maximum. Must include world name. Forward-looking and growth-focused.

Win cards: forward language only — always frame what the child is achieving, not what they lack.

Building section: never say "struggling" or reference a score. Frame as exploration and next steps.
Child Switcher
  • Shows all linked children in left sidebar
  • Active child highlighted with border-left: 4px solid #2c7a6e and teal background tint
  • Active child selection drives all main panel content (hero, next practice, progress snapshot)
  • Each child card shows: avatar emoji, name, world chip, tier label, last-active indicator
Engagement Chart
  • 7-day CSS bar chart — one bar per day of the week
  • Active days: #2c7a6e bar
  • Light-activity days: #d0e8e4 bar
  • Empty/no-session days: #ede8de neutral bar (very short height) — no label, no "missed" text
Messages
  • Teacher-initiated only — "New Message +" button is disabled for parents
  • Inline thread expands on click — no separate page navigation
  • Teacher bubbles: background: #e0f5ee (teal tint)
  • Parent bubbles: background: #fff with light border
  • Reply send button disabled in prototype — routes through contact-safe system in production
  • Safety note chip always visible in thread: "Replies route through WonderQuest's contact system"
Design Tokens
  • Background: #f5f0e8 (warm parchment)
  • Page bg: #ede8de
  • Nav: #2c7a6e (teal)
  • Accent / CTA: #2c7a6e
  • Next practice strip: #fffbea bg, #f0c040 border
  • Font: Sora (Google Fonts), weights 400 / 600 / 700 / 800
  • Hint box: background: #fffbea; border: 2px solid #f0c040
  • Tier chip colors — strength: #e0f5ee / #1a5a40 · growing: #e8f4fc / #1a4a6a · support: #fff0e0 / #8a4a00
