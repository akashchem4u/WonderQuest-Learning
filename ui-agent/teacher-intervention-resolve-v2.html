Panel 5 — Spec Notes: Canonical spec for teacher-intervention-resolve. Route: /teacher/student/:studentId/intervention/:skillId/resolve

Teacher Intervention Resolve — Spec

Resolve reasons

ReasonAvailable whenNote required?
Student reached Strength tierStudent is in Tier 1 (system confirmed)No (optional)
Classroom observationAlwaysYes — required when overriding data
Skill no longer relevantAlwaysNo (optional)

Warning states

ConditionWarningBlock?
Student still in Support tierOrange warning box — "student is still in Support tier"No — "Strength tier" reason disabled; others available
No reason selectedCTA stays disabled, "select a reason" hintYes — until reason chosen
Override selected but no noteNote field gets required stylingYes — until note is filled

Post-resolve actions available

  • View next intervention for same student (if any)
  • Back to class queue
  • View student history (archived timeline)
  • Message parent (quick link)

Bulk resolve rules

  • Only students at Strength tier appear in bulk resolve list
  • Students still in Support tier appear in list but are pre-unchecked and labelled "in progress"
  • Bulk resolve applies reason "Strength tier reached" to all selected — no per-student notes
  • Access: queue header "Bulk resolve X ready" button, visible when ≥2 students at Strength
