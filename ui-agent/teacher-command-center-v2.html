Engineering · Tab 5 — Spec Reference
All routes, layout rules, design tokens, trigger thresholds, and behavioral notes for the Teacher Command Center v2. This tab is a living reference for engineers and designers implementing this component.

API Routes

  • GET /teacher/class/summary — class KPIs, SNS counts, engagement chart data, recent wins, band activity
  • GET /teacher/support-queue — sorted queue (Flag → Watch, by days-inactive desc), cleared list, suggested actions
  • GET /teacher/at-risk — at-risk students with trigger type, weeks in support, skill gap, SNS trend
  • GET /teacher/assignments — all assignments with status, completion counts, due dates
  • POST /teacher/assignments — create/update assignment; body: skill_id, band, student_ids[], due_at

Desktop Layout

  • 3-col CSS grid: 260px / 1fr / 300px — left nav/summary, center main, right quick-actions
  • Collapse at 768px: single column; right col becomes a bottom drawer (sticky footer panel)
  • Left col: collapsible header strip on mobile (class name + SNS counts row)
  • Right col: position: sticky; top: 16px on desktop scroll
  • Tab 3 (At-Risk): 260px / 1fr — no right col, main content full-width table
  • Min desktop width: 1024px for full 3-col layout

SNS Tier Display Rules

  • Flag — student in support tier for ≥2 consecutive weeks. Card: bg #9a2a2a / text #ff9090
  • Watch — student in support tier for exactly 1 week OR declining from Strong. Card: bg #2a1800 / text #f0a030
  • Strong — student currently in strength tier. Card: bg #0a2a15 / text #50e890
  • Support queue sort: Flag first → Watch, then by days-inactive descending within each tier
  • Left border on queue/table rows: Flag = #c03030 (4px), Watch = #d08020 (4px)

At-Risk Triggers

  • Flag tier for ≥14 consecutive days → shown in at-risk table with "2+ wks support" filter
  • No activity for ≥7 days → shown with "No activity 7d" filter (regardless of tier)
  • 3-week declining SNS trend (Strong→Watch→Flag or Watch→Watch→Flag) → "Declining trend" filter
  • Bulk action: select all visible flag students → single "Message Parents" API call with student_ids[]

Assignment Behavior

  • Resend nudge threshold: completion rate <60% at 24h before due_at → show amber warning chip + "Resend" CTA
  • Progress bar: completed / total_assigned students, teal fill (#4a90d9)
  • Status chips: Active (teal), Scheduled (amber), Closed (grey), Draft (dashed outline)
  • Create CTA at bottom of list: dashed border, navy, always visible
  • Completion alert threshold for insights card: flag any student with 0 assignments started in current week

Design Tokens

  • Page bg: #f0f4f8 (cool steel)
  • Nav / primary: #2a5080 (navy)
  • Accent / teal: #4a90d9
  • SNS Flag bg: #9a2a2a / text #ff9090
  • SNS Watch bg: #2a1800 / text #f0a030
  • SNS Strong bg: #0a2a15 / text #50e890
  • Font: Sora (Google Fonts) — weights 400, 600, 700, 800

Mobile Notes

  • Right col moves below main content area on screens <768px (acts as a bottom drawer, collapsible)
  • Left col collapses to a sticky header strip: class name chip + 3 SNS tier count pills, tappable to expand
  • Queue items: action buttons collapse to icon-only row (📝 ✉️ 📅) with tooltips
  • At-risk table: horizontal scroll wrapper, action buttons stacked per row as small icon buttons
  • Nav class dropdown: becomes a modal bottom sheet on mobile for class switching
  • Quick-action buttons in right col: visible as a sticky horizontal bar at bottom of viewport on mobile
