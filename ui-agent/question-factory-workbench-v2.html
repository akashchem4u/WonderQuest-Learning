Engineering hint (Tab 5): This tab is the live spec reference for the content team and engineering. Keep in sync with docs/content-admin-spec.md. API routes documented below are canonical; implementation must match. QA rules are enforced server-side; client disables approve button for hard blocks but server also rejects approve requests for blocked questions.

API Routes

  • GET /content/questions/summary — band × status aggregates for overview dashboard
  • GET /content/questions/queue?status=&band=&sort=&page= — paginated question list with QA flags
  • GET /content/questions/:id — full question detail including options + explainer
  • PUT /content/questions/:id/approve — approve single question (hard block if no correct answer)
  • PUT /content/questions/:id/return — return to draft with optional note
  • DELETE /content/questions/:id — soft-delete question
  • GET /content/batches/:id — batch metadata + question list + auto-check results
  • POST /content/batches/:id/bulk-approve — approve all passing questions in batch
  • GET /content/gaps — gap nodes sorted by severity (critical → warning → ok)
  • POST /content/batches/generate — trigger AI batch generation for node IDs

Question State Machine

  • Draft → submitted for review → In Review
  • In Review → approved → Approved
  • Approved → published to live content → Published
  • In Review → returned → Draft (with return note)
  • Returned is a transient display state; storage state = Draft with return flag

Hard QA Blocks (cannot approve)

  • No correct answer marked on the question
  • Duplicate answer options (identical text in two or more options)
  • Prohibited content detected (slur, brand name, violence keyword)
  • Fewer than 2 answer options provided (minimum 4 required)

Warnings (can approve with override)

  • Borderline word length: question stem >18 words or any option >12 words
  • Missing emoji in question for pre-reader context (P0 band only)
  • Thin skill coverage: question is the only Q for this node (coverage = 1)
  • Readability score outside band-appropriate range (Flesch-Kincaid deviation >2 grades)

Auto-check Rules

  • Runs automatically on save/import; results cached until next edit
  • Hard blocks prevent approval via UI and API
  • Warnings flag the card for human review but do not block
  • Batch ingest runs auto-check on all imported questions before queuing
  • Manual override of warnings requires editor role or above

Coverage Thresholds

  • On target: ≥6 approved questions per node
  • Warning: 4–5 approved questions per node
  • Critical: <4 approved questions per node
  • Counts are approved-only; draft and in-review do not count toward threshold

Batch Review Flow

  • AI-generated batches land in batch review queue; status = Draft + batch_id set
  • Auto-check runs on ingest and tags each question: pass / warn / block
  • Editor reviews batch, can bulk-approve all passing questions
  • Each approved question individually transitions to Approved state
  • Hard block questions remain in Draft; editor must edit + resubmit
  • Batch is "complete" when 0 hard blocks remain (warnings can be left for next pass)
