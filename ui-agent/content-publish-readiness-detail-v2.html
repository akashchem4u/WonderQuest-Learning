Engineering hint [Tab 5 — Spec]:
Routes:
  GET /content/publish-gate → readiness score, component scores, blocker list
  GET /content/hardblocks → all hard-block issues with status, type, age
  GET /content/publish-checklist → per-item checklist state
  POST /content/publish/band/:id → publish single band (checks band-scoped gate)
  POST /content/publish/all → publish all bands (requires gate score ≥ 85, no hard blocks)
  POST /content/publish/sign-off → content-admin sign-off
  POST /publish/owner-exception → record owner exception for warning items
  PATCH /content/hardblocks/:id/resolve → mark hard block resolved
