⚙ Engineering Hint — Tab 5: Full Spec Reference Routes, routing decision logic, no-guilt language rules, reward preservation guarantees, fresh-start behavior, and design tokens for the entire session resume overlay system.
API Routes
MethodRoutePurpose
GET /player/session-state Returns current session state, world, node, question index, rewards pending, streak data, recently visited worlds
POST /session/resume Continues session from saved q_index. No star change. Used for mid-session resume and next-node start.
POST /session/restart Resets node to Q1. Body: { node_id, preserve_stars: true }. Stars from all prior attempts are always kept.
Resume Routing Logic
ConditionShow
session_state === "in_progress"Tab 1 — Resume Overlay (mid-session)
session_state === "complete"Tab 3 — Session Complete Return
hours_since_last_session > N (configurable)Tab 4 — Long-Absence Return
Child taps "Start fresh instead" (Tab 1)Tab 2 — Fresh Start overlay
No-Guilt Rules — Banned Language
  • "you left" — never reference the child's departure
  • "you abandoned" — implies negative intent, never use
  • "you missed" — implies failure, never use
  • "you failed to complete" — never frame incompletion as failure
  • Timers, countdowns, or urgency language — zero pressure allowed
  • "you'll lose" / "you lost" — stars are always safe, never threaten loss
Rewards Preservation Rules
ScenarioStars / XP Outcome
Child exits mid-session (e.g. Q2 of 3)All stars from completed questions banked immediately. Q3 stars remain pending until played.
Child resumes mid-sessionContinues from saved q_index. Previously-earned stars already in account balance.
Child restarts node (Tab 2 option)preserve_stars: true — all stars from previous attempt are kept. No penalty of any kind.
Child switches worlds (Tab 4 chip)Previous world session state preserved. Can return to saved position at any time.
Fresh Start Rule
ActionBehavior
Restart a nodeSession resets to Q1. Stars earned from any prior play of this node remain in the player's balance. No deduction.
Resume a nodePicks up exactly at last q_index. Pending reward chips shown.
