Spec & Design Tokens
Parent Linking Recovery v2
API Routes
GET /parent/linked-children
POST /parent/link-child
DELETE /parent/unlink-child/:id
GET /parent/verify-code/:code
Child Code Format
6-character alphanumeric (e.g. "WQ-A3F7")
Auto-uppercase on input; "WQ-" prefix display only
Verified server-side before linking proceeds
Privacy Rules
Child profile (name, progress) visible only to linked parent
Unlink: removes parent access, preserves all child data
Each child's data is scoped independently per parent
No cross-sibling data leakage in multi-child view
Wrong-Link Detection
Triggered by: teacher/school confirmation mismatch
Triggered by: parent manual report via "No, this isn't right"
Alert state: linkedChild.status === "mismatch"
Unlink Flow
Step 1: Confirm → DELETE /parent/unlink-child/:id
Step 2: Data retained for child account (nothing lost)
Step 3: Parent access removed → relink option shown
Multi-Child
Max 5 children per parent account
Each child: independent progress view
Switcher chip at top when 2+ children linked
Trust & Privacy Cues
Lock icon + "data is safe" on every unlink/relink step
Privacy card always visible on empty state (Tab 3)
Warning tone: amber only — never red for parent-facing errors
Design Tokens
Background: #f5f0e8 (warm parchment)
Nav / Primary: #2c7a6e (teal)
Warning: amber #f0c040 (never red)
Font: Sora (Google Fonts), weights 400/600/700
Phone frame: 375px wide, border-radius 40px
Canvas: white (#fff) — phone centered on page
