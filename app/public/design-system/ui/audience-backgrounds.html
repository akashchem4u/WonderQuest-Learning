Home Route — Dual Identity System (Trust-led & Energy-led)

Design intent: The home page has two personalities — A: Trust-led for first-time parents who need warmth and safety signals, and B: Energy-led for returning families where a child has an active quest. Route logic determines which variant renders. Both must be production-ready.

Engineering note: variant selection is determined server-side based on session state — if a returning child has an active quest, render the Energy-led (B) shell. Otherwise default to Trust-led (A) for all first-time / unauthenticated visitors. Do not expose the toggle as a client-side preference; it should be transparent to the user.
