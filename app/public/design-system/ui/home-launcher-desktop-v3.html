Tab 4 — Multi-Role Switcher · Engineering Hints Shown at GET /home when jwt.claims.role is an array with length > 1 (e.g. ["teacher","parent"]). Role chip colors map: child → violet, parent → teal, teacher → navy, owner → gold, content → cyan. "Last used" timestamp comes from localStorage:wq_role_last_{role} or GET /api/user/role-sessions. Home Launcher Desktop — v3 Spec
File: home-launcher-desktop-v3.html · WonderQuest UI Agent

Routes

GET /Unauthenticated → route picker (Tab 1). No JWT present or JWT expired.
GET /homeAuthenticated entry. Role detection fires; redirects to specific surface.
GET /home/childChild world picker. Requires parent_link_id in JWT claims.
GET /home/parentParent family dashboard home surface.
GET /home/teacherTeacher class home — most-recent class pre-loaded.
GET /home/ownerOwner ops command center.
GET /home/contentContent admin review workbench.

Role Detection Logic

1. Decode JWT → read claims.role field.
2. If string → redirect to /home/{role} directly.
3. If array, length = 1 → same as string case.
4. If array, length > 1 → render Multi-Role Switcher (Tab 4).
5. If default_role preference set → skip switcher, go directly to that role's home.

Child Entry Rules

Child entry at /home/child always checks for jwt.claims.parent_link_id.
If missing: world cards remain visible but CTA buttons disabled; parent-link overlay shown (calm teal, no red/blocking).
World cards fetched from GET /api/child/worlds. XP is live data.
Offline mode: show last-cached world list with (offline) badge; disable XP updates.

V3 Changes vs V2

  • Added Tab 4 — Multi-Role Switcher: shown when JWT role is an array of length > 1. V2 had no switcher; multi-role users were dropped on the generic home with no guidance.
  • World Cards in Child Flow (Tab 3): V2 had a plain text list. V3 adds emoji world icons, XP count, level, and per-world CTA buttons.
  • Returning-User Personalized Header (Tab 2): V2 used a generic "Welcome" heading. V3 pulls the display name from JWT claims.name and surfaces active session + recent-activity strip.
  • Parent-Link Overlay redesign: V2 used a red warning banner. V3 uses a calm teal card; world cards stay visible and non-blocking.
  • Announcements Card: new in V3 — conditionally rendered from /api/announcements. Absent in V2.
  • Role card hover glows: V2 had a generic white glow. V3 maps glow color to each role's palette token.
