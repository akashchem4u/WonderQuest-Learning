Tab 5 — Spec Notes
Engineering spec for the owner feedback detail push screen. All design tokens, interaction rules, type-specific fields, and action bar logic.
Route & Navigation
Push screen from Feedback Queue (/owner/feedback). Route: /owner/feedback/:feedbackId. Back chevron returns to queue. Deep-link from route health alert banner navigates directly here bypassing queue list.
Type-Specific Fields
Type Unique fields CTA primary
Critical Bug Stack trace · Impact strip (affected sessions + error rate) · Related reports · Auto-escalate audit entry after 15 min Create Ticket
Bug Stack trace (if available) · Affected user count · Repro steps (if provided) Create Ticket
Content Content preview with highlighted error · Suggested fix · Fix-in-editor deep-link · Related flagged nodes in same skill Fix in Content Editor
UX Screenshot/recording attachment · Device tags · Route + component path · Related UX patterns on same route Create UX Ticket
Praise User profile card (role, tenure, streak) · Quote highlight · Usage context (band, sessions, avg/week) · Highlight-worthy score Highlight for Marketing
Triage Rules
Rule Detail
Critical non-dismissible Archive and dismiss buttons hidden until ticket created OR resolved. Back-nav allowed but queue row maintains unread state.
Auto-escalate Critical reports unacknowledged for 15 min trigger system audit entry and page on-call. Surfaced in audit trail on detail screen.
Ticket badge After ticket creation, ticket ID badge appears in topbar area. Badge links to ticket system external URL.
Post-resolve Row in queue becomes greyed with "Resolved" badge. Detail screen shows audit trail with resolve entry. Archive available after resolve.
Content Fix-in-Editor Deep-link
Deep-link format: /content/editor?node=:nodeId&highlight=answer_key&source=feedback&feedbackId=:id. Opens content CMS with the specific node pre-loaded and answer key section highlighted. Available only on Content-type feedback. If node is archived/deleted, show "Content no longer available" inline state with Archive as only action.
Praise Highlight Flow
Highlighted quotes are added to a "Marketing highlights bank" accessible at /owner/marketing/highlights. Auto-tags applied based on: user role (parent/teacher) + child band + feedback content keywords. Owner can edit tags before confirming. Highlighted quotes cannot be used externally without a separate export approval step.
Audit Trail
Every state change appended to audit trail chronologically newest-first. Entries: created · viewed · escalated (system/manual) · ticket created · ticket linked · status changed · highlighted · resolved · archived. System-generated entries shown in muted color; owner-generated entries show display name.
Related Reports Grouping
Related reports grouped by: same error signature (stack trace hash match) OR same route + same type + within 24 hours. Grouped reports share a single detail screen. "View all N" link navigates to a filtered queue view pre-set to matching criteria.
