Engineering hint (Tab 5 — Spec Reference): This tab is a living spec reference for content-admin alpha readiness. Update endpoint paths and threshold values here as the backend contract evolves. Score weights must sum to 100. Coverage thresholds apply to all bands equally — no per-band overrides in alpha scope. Publish gate is AND logic: all conditions must pass. Action queue priority algorithm is deterministic: no ML, purely rule-based sort.
API Routes
  • GET /content/alpha/summary — Overall readiness score, band summaries, blocker list
  • GET /content/coverage/band/:id — Node-level Q count for a band (id = p0/p1/p2/p3)
  • GET /content/explainers/status — Full explainer list with status, script, mediaHints, QA fields
  • GET /content/publish/readiness — Per-band publish gate evaluation with blocking items array
Coverage Thresholds (Q/node)
  • OK (green) — ≥6 Q/node. Acceptable for alpha release.
  • Warning (amber) — 4–5 Q/node. Not a release blocker but degrades learner experience.
  • Critical (red) — <4 Q/node. Release blocker. Band cannot publish.
  • Target — 8 Q/node. Full-quality experience. Used as target line in charts.
Alpha Readiness Score (100pts total)
  • Question Coverage (40pts) — % of nodes at ≥4Q threshold (weighted to total weight)
  • Explainer Coverage (25pts) — % of required explainers in Published or In-Review status
  • QA Pass Rate (20pts) — % of submitted explainers that passed QA (no hard blocks)
  • Publish Readiness (15pts) — % of bands meeting all publish gate conditions
  • GO threshold: ≥85/100 — Alpha launch approved when score ≥85 AND no critical blockers in P0 or P1
Publish Gate (AND logic — all must pass)
  • All nodes in band ≥4 Q/node (minimum threshold)
  • All required explainers for band are in "Published" status (not Draft, not In Review)
  • No QA hard blocks on any explainer in band
  • All media hints fields populated (audio transcript, visual cues)
Action Queue Generation Algorithm
  • Step 1: Filter all open gaps (critical nodes, missing explainers, QA blocks)
  • Step 2: Sort by blocker status — items that are release blockers float to top
  • Step 3: Secondary sort by band priority — P0 before P1 before P2 before P3
  • Step 4: Tertiary sort by estimated effort ASC — low-effort quick wins surfaced early
  • Step 5: Annotate each action with action.scoreImpact (pts gained if resolved)
  • Max 8 items shown in queue. "View all" loads full gap list in separate view.
Explainer Status Flow
  • Missing → No record exists. Content team must create from scratch.
  • Draft → Script in progress. Not yet submitted for QA. Cannot publish.
  • In Review → Submitted for QA review. Awaiting approval.
  • QA Approved → Passed QA review. Ready to publish.
  • Published → Live in production. Counts toward publish gate.
  • Hard blocks can occur at Draft → In Review (broken media) or In Review → QA Approved (script issues). Hard blocks return status to Draft.
