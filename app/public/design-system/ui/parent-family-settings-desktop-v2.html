Engineering Hint — Tab 3: Quiet Hours Route: GET /parent/settings/quiet-hours · PUT /parent/settings/quiet-hours
Stored as: { start: "20:00", end: "07:00", days_bitmask: 127 } — bitmask bit 0=Mon through bit 6=Sun.
Emergency bypass: teacher messages + safety alerts always delivered regardless of quiet hours — server-enforced, not client UI.
Weekly reports: if quiet hours active at delivery time, delay to end + 1 hour (default 8:00 AM next day).
Current status card: computed from now() vs. schedule — show active/inactive state with time until change.

Engineering Spec — Parent Family Settings Desktop v2
Routes:
  GET  /parent/settings                      → Full settings object (children, notifications, quiet-hours, privacy)
  PUT  /parent/settings/notifications      → Body: { push, email, sms } channel objects with enabled + per-category booleans
  PUT  /parent/settings/quiet-hours       → Body: { start: "HH:MM", end: "HH:MM", days_bitmask: 0-127, exceptions: {...} }
  GET  /parent/privacy/data-summary      → Visibility breakdown per child, teacher info, school district
  DELETE /parent/child/:id/data         → Initiates soft-delete (30-day grace) with parent email confirmation
  GET  /parent/privacy/download          → Async job; returns signed S3 URL when complete

Desktop Layout:
  260px fixed settings nav sidebar / 1fr main content area — CSS grid
  Collapse sidebar at 768px breakpoint (single column, sidebar hidden)
  Teal nav header #2c7a6e fixed at top; sidebar bg #ede8dc; main bg #f5f0e8

Notification Channels:
  Push — requires Notification API permission; permission status chip read-only from browser
  Email — masked display (a***@g***); update via modal flow with re-verification
  SMS — optional; if phone_number=null render "Add number" prompt; if added, show subcategory toggles
  All channels: teacher messages subcategory always_deliver=true, server-enforced (UI shows locked toggle)

Quiet Hours:
  Storage: start/end as "HH:MM" strings + days_bitmask (bit 0=Mon, bit 6=Sun, 127=every day)
  Emergency bypass: teacher messages + safety alerts — server ignores quiet hours for these event types
  Weekly reports: if delivery time falls in quiet window, delay to quiet_end + 60min
  Status card computed server-side: include is_active, next_change_at, minutes_remaining

Privacy Levels (per child):
  Full    → All progress visible to teacher: world progress, sessions, skill tier, streak
  Limited → Skill tier only; no session count, no streak detail
  Minimal → No data shared with teacher; teacher sees child enrolled but nothing else

Data Deletion Flow:
  Step 1: Parent clicks delete → confirmation modal with child name re-entry
  Step 2: Soft delete initiated → data hidden from all surfaces, 30-day timer begins
  Step 3: Parent emailed at soft-delete + at 25 days (5-day warning) + at hard-delete
  Step 4: Hard delete after 30 days — irreversible, all child data purged from all systems

Design Tokens:
  bg: #f5f0e8  ·  nav: #2c7a6e  ·  sidebar: #ede8dc  ·  card: #ffffff
  font: Sora (Google Fonts)  ·  teal: #2c7a6e  ·  amber: #f0c040 (warning only — never red for parent settings)
  Toggle on: #2c7a6e  ·  Toggle off: #c0b8a8  ·  Destructive: #c05a2a (amber-shifted, not pure red)
  Locked toggle opacity: 0.55, cursor: not-allowed

Accessibility notes:
  All toggles must have aria-label and aria-checked attributes in production
  Locked toggles: add aria-disabled="true" and title tooltip explaining why
  Settings sidebar: nav role="navigation" with aria-label="Settings navigation"
