Design specification for global-permission-denied-v2.
4 Denied-Access Variants
Wrong RoleShows user's actual role + what's needed
Not LinkedClear path to link + privacy note
Session ExpiredCalm, state-saved, countdown, child hand-off
Coming SoonPositive framing, notify CTA, no "blocked" language
Language Rules (all variants)
✅ Never say "forbidden," "blocked," "denied," "unauthorized"
✅ Explain what the user CAN do, not just what they can't
✅ Child surface: zero jargon, hand off to grown-up
✅ Always show account context strip (who they're signed in as)
✅ Always include a primary CTA and a secondary "go back" path
✅ Privacy note shown on "Not Linked" variant
Session Expired Rules
State saved?Yes — returns to same page after sign-in

Child versionHand off to grown-up, not sign-in form
Timeout trigger30 min inactivity (parent/teacher), 8hr (owner)
