child-next-quest-card-v2 — Specification

Persona: Child · Mobile 375px · Nunito 900 · #100b2e bg

API: GET /child/next-quest → { world_name, world_emoji, node_name, progress_dots[], reward_stars_waiting, daily_challenge? }

API: POST /child/session/start { node_id } → redirect to session

  • Next Quest: dominant single CTA "Play Now!", world name, node name (friendly), progress dots, reward preview
  • Progress dots: done=mint, current=violet glow, next=outlined — NO percentages or counts shown to child
  • Choose Different: world picker compact with locked worlds showing "keep playing" not "not available"
  • Daily Challenge: special bonus card with extra reward preview, opt-in — never required
  • Empty/New state: welcoming, no pressure framing, single big CTA
  • No timers, no grade labels, no accuracy %, streak shown as encouragement not pressure
  • Node names: friendly adventure-style names only (max 8 words for pre-reader compatibility)
  • Reward preview: stars waiting phrasing — always positive, replay can't lose stars
