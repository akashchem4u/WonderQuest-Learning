Design System

🤖 For the Implementing Agent — Read First

  1. Link design-system.css in every new screen. All color, spacing, radius, shadow, animation, and typography tokens live there. Never re-declare tokens inline.
  2. Audience separation is strict: Child play → Nunito font + dark cosmic #100b2e. Parent/Teacher → Sora + warm light #f5f0e8. Owner → Sora + dark pro #0d1117.
  3. Player colors are sacred: p0 = #ffd166 gold · p1 = #9b72ff violet · p2 = #58e8c1 mint · p3 = #ff7b6b coral. Never reassign.
  4. Use the v2 files as source of truth. Files marked Legacy are superseded — only reference them for v1 context. Always port from the newest version.
  5. Prefer the tabbed reference pattern. Each file has all states in one HTML with tab switching — open the file and tab through states before implementing.
  6. Check the yellow hint boxes inside each tab panel — they contain implementation notes specific to that state.
  7. Glass morphism rule: background: rgba(255,255,255,0.07) + backdrop-filter: blur(12px) + border: 1px solid rgba(255,255,255,0.13).

Design System Quick Reference

🎨 Audience Themes

Child play · dark cosmic #100b2e · Nunito
Parent/Teacher · warm light #f5f0e8 · Sora
Owner · dark pro #0d1117 · Sora

🎭 Player Colors (Sacred)

Never reassign these between players.

⚡ Animation Easings

Spring — buttons, popups, card hover
cubic-bezier(0.34,1.56,0.64,1)
Ease-out — screen transitions, fades
cubic-bezier(0.16,1,0.3,1)

🪟 Glass Morphism Rule

background: rgba(255,255,255,0.07)
backdrop-filter: blur(12px)
border: 1px solid rgba(255,255,255,0.13)
Three opacity variants: 7% / 11% / 16%
