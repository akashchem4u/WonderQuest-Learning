TAB 5 — Engineering Spec for Play Session Shell (Tablet v2)
Routes, dimensions, question types, answer card rules, recovery trigger logic, word limits, child safety rules, design tokens, and tablet-vs-mobile differences.
Spec
📋 Engineering
API Routes
MethodRoutePurpose
GET/session/questionFetch next question for active session
POST/session/answerSubmit child's answer; returns outcome + XP delta
GET/session/hintFetch hint content for current question (triggers after 2 wrong)
POST/session/completeFinalize session; emit star + XP totals (never reduce)
Tablet Frame Dimensions
  • Frame: 768 × 1024 px (iPad portrait)
  • Border radius: 28px · Border: 4px solid #2e1f6e
  • Progress strip: full 768px width, always visible, ~56px tall
  • Question zone: min-height 380–420px, generous padding 48px sides
  • Tab bar: full width, 64px tall, 4 items
  • Answer card min-height: 110px · min touch target: 80px
  • Voice coach FAB: 88px (active) / 96px (recovery mode)
Question Types

Answer Card Rules
  • Always exactly 4 options per question
  • All options must be visually distinct (different emoji/text)
  • Exactly 1 correct answer per question
  • Every card must have an emoji present (48px pre-reader / 24px K-1)
  • Card label max: 1–2 words (pre-reader) / 5 words (K-1)
  • Correct state: mint #58e8c1 border + gold-pop animation
  • Wrong state: dim + soft-shake animation, auto-clear after 600ms
  • Recovery glow: gold border only — never labeled "correct"
Recovery Logic
  • Auto-show hint overlay after 2 wrong picks on the same question
  • Manual coach tap always shows hint immediately (any attempt count)
  • Recovery card: coach bubble + enlarged stimulus + sound/phoneme hint
  • Original question stays visible above hint — never replaced
  • Correct card gets subtle gold glow only (no "correct" text or label)
  • Wrong-attempt count is never shown to the child
  • No red colors anywhere in recovery state
Word Limits
  • Pre-reader: max 8 words per question text
  • K-1: max 15 words per question text
  • Answer card label: max 2 words (pre-reader) / 5 words (K-1)
  • Celebration / recovery messages: plain, emoji-first, <10 words
Child Safety Rules
  • No timers of any kind (no countdown, no elapsed time)
  • No grade level labels visible to child (no "K-1", "Grade 1", etc.)
  • No skill IDs or curriculum codes shown in UI
  • Wrong answer = gentle visual only — dim + soft-shake, no buzzers, no red
  • Wrong-attempt count never displayed
  • Star/XP count can never decrease (replay safe)
  • Session completion always gives positive outcome message
