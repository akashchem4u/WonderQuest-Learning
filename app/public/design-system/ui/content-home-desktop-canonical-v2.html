Engineering Spec — Content Admin Home (Canonical Desktop v2)

Routes:
  GET /content/dashboard — Tab 1 summary data (KPIs, pipeline counts, recent activity)
  GET /content/coverage — Full 48-node coverage map with band/status/question counts
  GET /content/coverage/summary — Aggregated health per band (for sidebar health bars)
  GET /content/qa-queue — QA queue list; filter params: ?status=pending|flagged|approved|rejected&band=P0|P1|P2|P3
  GET /content/qa-queue/stats — Avg review time, approved today, blocked count
  POST /content/publish — Publish single item: body { itemId, band, approvedBy }
  POST /content/publish/batch — Publish all ready items
  GET /content/publish/blockers — Active hard blocks
  GET /content/publish/history — Published item log
  POST /content/questions/create — Create new question for a skill node

Coverage Thresholds:
  ≥ 8 questions → OK (green)
  4 – 7 questions → Warning (amber)
  < 4 questions → Critical (red) — release blocker, hard block on publish

QA Flow:
  Draft → QA Review → Approved → Published
  Flagged items return to Draft with reviewer notes.
  Rejected items are archived and must be re-created.

Hard Blocks (prevent publish):
  · Missing transcript on explainer/video content
  · Fewer than 4 questions on a skill node
  · Prohibited content flag raised by QA reviewer
  · No approved QA sign-off

Design Tokens:
  Background: #10161e  |  Card: #141e28  |  Border: #1e2e3e
  Accent: #58c8f0 (cyan)  |  Text: #d0e8f0  |  Muted: #7a9ab0
  OK: #4ade80  |  Warning: #fbbf24  |  Critical: #f87171
  Font: Sora (Google Fonts) — weights 400, 500, 600, 700, 800

Layout:
  Desktop only. Max-width 1200px, centered. Tab 1 uses 300px sidebar + 1fr main grid.
  Top nav: 52px, sticky. Tab bar: below nav, sticky. No phone frame.

Band Labels:
  P0 = Foundation (pre-K core)  |  P1 = Early Literacy & Numeracy
  P2 = Reading & Math Foundations  |  P3 = Advanced Concepts & Extension
