Panel 5 — Spec Notes: Canonical spec for /owner/feedback (mobile feedback queue). Owner-only route. Aggregates feedback from all audiences.

Owner Feedback Queue Mobile — Spec

Feedback types

TypeLeft borderBadge colorSourcesAction
Critical Bug#ff6b6bRedAuto-detected OR user-reportedTriage / Escalate — non-dismissible without triage
Bug#ff9944AmberUser-reportedTriage / Dismiss
Content Issue#9b72ffPurpleTeacher-reportedFix in editor / Dismiss
UX Feedback#5599eeBlueAny audienceTriage / Dismiss
Praise#58e8c1GreenAny audienceArchive (auto after 30 days)

Sort order

PrioritySort
1stCritical bugs (unread first)
2ndNormal bugs (newest first)
3rdContent issues
4thUX feedback
5thPraise
BottomResolved (collapsed section, greyed)

Triage actions

ActionResultAvailable for
Create engineering ticketOpens ticket in Jira/Linear with prefilled context; row shows ticket ID badgeBug, Critical Bug
Mark in progressMoves to in-progress section; shows assignee pillAll types
Mark resolvedMoves to resolved section (greyed, strikethrough)All types
Escalate to engineeringPages on-call via same mechanism as route health alertsCritical Bug only
Fix in editorDeep-links to content editor for that question IDContent Issue
ArchiveRemoves from queue, stored in historyPraise only
DismissRemoves from active queue (not an error)Bug, UX, Content

Dismissal rules

  • Critical bugs: cannot be dismissed without first triaging (must choose an action)
  • Normal bugs: can be dismissed with 1 tap (no confirmation required)
  • Praise: no dismiss — only Archive action available
