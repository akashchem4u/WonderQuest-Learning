Privacy Rules

Child
Never identified in feedback records. No name, avatar, or learner ID is stored or surfaced in routing views.
Parent
Identified by cohort label only (e.g. "Cohort A-12"). No personally identifiable information is retained.
Teacher (B2B)
Identified by school name under the B2B data-sharing agreement. Individual teacher names are not stored.

Business Rules

  • BR-1 Every feedback item must be classified before routing. Items with status = 'new' are never directly assigned to a team; the classifier pipeline must run first.
  • BR-2 A P0 bug automatically triggers an on-call alert to Engineering within 5 minutes of classification. No manual routing step is required for P0 items.
  • BR-3 The "Route All Unrouted" action is disabled (greyed out) when zero items have status = 'classified'. This prevents accidental no-op submissions.
  • BR-4 Resolution log rows are retained for 90 days then archived to cold storage. The board UI only queries the active 90-day window.
  • BR-5 Routing decisions are immutable once set. Re-routing requires an explicit override action that is logged with the acting owner's ID and a mandatory reason field.
