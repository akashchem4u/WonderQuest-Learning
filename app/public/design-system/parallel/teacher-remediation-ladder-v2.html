DB Schema

CREATE TABLE student_support_flags ( id UUID PRIMARY KEY DEFAULT gen_random_uuid(), student_id UUID NOT NULL REFERENCES students(id), flag_type ENUM('low_engagement','revisit_pattern', 'quest_exit_early','not_started') NOT NULL, flag_severity ENUM('watch','action','urgent') NOT NULL, created_at TIMESTAMPTZ NOT NULL DEFAULT now(), resolved_at TIMESTAMPTZ, resolution_notes TEXT );

Flag Type Definitions

flag_typeTrigger condition (engagement-based)Default severity
low_engagementFewer than 2 quest sessions in a 7-day windowaction
revisit_patternStudent revisited the same quest band ≥3 times without progressingwatch
quest_exit_earlyStudent exited a quest before the midpoint on ≥2 occasions in a weekaction
not_startedZero quest activity for 7+ calendar daysurgent

All flags are derived exclusively from session and engagement data. No assessment scores, accuracy rates, or correctness data are used.

Privacy & FERPA

  • Display student first names only — never last names, IDs, or photos.
  • Accuracy percentages and assessment scores must never appear in this view.
  • Flags are derived from engagement signals only; no academic outcome data is surfaced.
  • Resolution notes are stored server-side and visible to the assigned teacher only.
  • Student support data is school-linked and is not disclosed to external parties (FERPA §99.31).
  • All data is scoped to the authenticated teacher's assigned class; cross-class queries are rejected at the API layer.

Language Rules

Prohibited termApproved alternative
strugglingneeds a check-in / may benefit from extra support
behindearlier in their journey
below average(omit entirely — never quantify relative to peers)
incorrect / wrong(omit — engagement flags only, not accuracy)
failingnot yet started / just getting started
low performerstudent who may benefit from a warm invite to re-engage

All copy must use positive, supportive framing. Automated text generation for recommendation cards must pass a prohibited-term filter before rendering.

Business Rules

  • A student may appear on at most one rung at a time — the highest applicable severity wins.
  • Flags auto-expire after 14 days if no teacher action is taken; an audit record is retained.
  • Marking a flag resolved requires a non-empty resolution note (minimum 10 characters).
  • A resolved flag moves to the Resolution Log immediately on the client; re-flagging requires a new DB record.
  • Platform messages triggered from this view are logged against the flag record for continuity.
  • Rung counts are refreshed nightly; real-time WebSocket updates are out of scope for v2.
