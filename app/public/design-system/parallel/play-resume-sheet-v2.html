Staleness Rules

  • Same Day (<2h): resume primary, no fresh-start option
  • Next Day (2h–48h): resume primary, fresh start as secondary
  • Recent (2–7 days): fresh start shifts to primary, resume secondary
  • Old (7+ days): fresh start only — don't show old session preview
  • Time from: session-paused-at timestamp in local storage

Context Restore

  • On resume: restore exact question state (question text + choices + progress)
  • Restore: star count, progress segs, skill context
  • Question preview shows: question text + Q# + skill name + band
  • If session data corrupt: fall back to "Start fresh" only
  • ❌ Never restore to a mid-wrong-answer state

Star Safety

  • Stars from paused session are ALWAYS kept regardless of staleness
  • Same Day: star row shows "Stars earned so far: N ⭐"
  • Next Day: star row shows "Stars still safe from yesterday: N ⭐"
  • Stale: star-safe banner "Your ⭐ N stars from that session are saved!"
  • ❌ Never forfeit stars from abandoned sessions

Layout

  • Sheet: same bottom sheet pattern as pause sheet (24px 24px 0 0 radius)
  • Same Day: ~40% height; Next Day: ~50%; Stale: ~50%
  • Question preview: 2rem emoji + question text + Q# + skill
  • Fresh-start card: contained within sheet, not a separate overlay
  • Primary CTA always 54px; secondary always 46px
