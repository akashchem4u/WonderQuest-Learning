display-mode-toggle-v2.html — Display mode toggle component.
4 modes: Dark (default), Light, High Contrast (WCAG AAA), Print-Friendly (ink-saving).
Stored in: user_preferences.display_mode (ENUM dark/light/high_contrast/print).
Applied via: CSS class on <html> element (class='mode-dark'/'mode-light' etc).
Respects prefers-color-scheme media query as default.
Available to: all adult personas. Child personas: Dark mode only — no toggle shown.
High Contrast mode: minimum contrast ratio 7:1 (WCAG AAA).
Print-Friendly: removes backgrounds, uses black text on white.

Child persona: Locked to Dark mode only — no toggle is rendered. High Contrast mode preserves emojis and colour-coding; contrast ratios are enforced at 7:1+ minimum across all interactive elements.

Keyboard access: All toggle variants are keyboard-navigable. Segmented control uses arrow-key roving focus. Icon buttons expose aria-label. Dropdown toggle is labelled and focusable.

Database Schema

-- user_preferences table (relevant columns) CREATE TABLE user_preferences ( user_id UUID PRIMARY KEY REFERENCES users(id), display_mode ENUM( 'dark', 'light', 'high_contrast', 'print' ) NOT NULL DEFAULT 'dark', updated_at TIMESTAMPTZ DEFAULT NOW() );

API Endpoints

Method Endpoint Body / Response Notes
GET /api/user/preferences → { display_mode: "dark" } Returns current stored preference
PATCH /api/user/preferences { display_mode: "light" } Updates preference; synced across devices

Application Logic

// 1. On page load — apply stored preference async function initDisplayMode() { const prefs = await fetch('/api/user/preferences').then(r => r.json()); document.documentElement.className = 'mode-' + prefs.display_mode; } // 2. setMode() — called by all toggle variants function setMode(mode) { // Apply immediately (optimistic UI) document.documentElement.className = 'mode-' + mode; // Persist to server fetch('/api/user/preferences', { method: 'PATCH', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ display_mode: mode }) }); }

Fallback & Sync Rules

// Fallback — before user preference is loaded const prefersDark = window.matchMedia('(prefers-color-scheme: dark)').matches; const prefersLight = window.matchMedia('(prefers-color-scheme: light)').matches; document.documentElement.className = prefersDark ? 'mode-dark' : 'mode-light'; // Print mode — auto-apply on browser print dialog window.matchMedia('print').addEventListener('change', (e) => { if (e.matches) document.documentElement.className = 'mode-print'; }); // Child persona — lock to dark, ignore preference field if (currentUser.persona === 'child') { document.documentElement.className = 'mode-dark'; // Toggle UI is not rendered for child accounts }

Sync & Storage Notes

Persistence Server-side via user_preferences table — synced across all devices
Fallback storage localStorage wq_display_mode used only when unauthenticated
Child persona Mode locked to dark; preference field ignored; toggle not rendered
Print auto-apply window.matchMedia('print') fires; does not overwrite stored preference
OS preference Used as initial default only; explicit user selection always wins
