📋 play-no-audio-fallback-v2 · Engineering Spec

Principle

  • Audio failure is NEVER blocking — question always answerable in visual-only mode
  • Child never sees a technical error message ("Audio API failed", "404", etc.)
  • Language: warm, forward-focused — "Sound isn't working right now" not "Error loading audio"

Fallback Scenarios

  • Audio Error API/file load failure — immediately switch to visual-only mode, show text question, amber banner, "Retry audio" button
  • Muted Device Detected via browser audio context check — show banner "Try turning up volume", offer "Visual only" button to dismiss banner, question still live
  • Low Bandwidth Audio loading >2s — show inline loading bar + "Skip and answer visually" link, question visible immediately

Visual-Only Mode (Audio Error)

  • Replaces audio hero button with a text question card (.visual-question)
  • Shows "🔇 Visual-only mode" badge (amber)
  • Replay button changes to "🔄 Retry audio" — attempts to reload audio
  • If retry fails: button shows "Retry failed" for 2s then resets to "Retry audio"

Text Question Format

  • Pre-K audio question → "What letter makes the beginning sound in 'bee'?"
  • K-1 phonics → same format: "What sound does [word] start with?"
  • Word or sentence shown in visual question card with key term highlighted in violet

Banner Colors

  • Amber (#f0a020) — not red, not alarming. Audio issue is minor, not a failure.
  • Background: #2a1a00 — warm dark, not harsh red
