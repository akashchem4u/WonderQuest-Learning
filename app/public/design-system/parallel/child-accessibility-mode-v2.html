ENG NOTE — child-accessibility-mode-v2.html
Child accessibility settings. Managed by parent in parent settings (not directly by child). 5 accessibility modes: (1) Standard, (2) High Contrast, (3) Large Text, (4) Reduced Motion, (5) Audio-Only mode (no visuals required). Settings stored per child in child_accessibility_prefs(child_id, mode, font_scale, motion_reduced, audio_only, high_contrast). Applied on child's device on next session start. WCAG 2.1 AA minimum across all modes; AAA in High Contrast mode.

Data Model
child_accessibility_prefs (
  child_id              UUID PRIMARY KEY REFERENCES children(id),
  mode                  ENUM('standard','high_contrast',
                             'large_text','reduced_motion','audio_only'),
  font_scale            FLOAT DEFAULT 1.0,
  motion_reduced        BOOLEAN DEFAULT false,
  audio_only            BOOLEAN DEFAULT false,
  high_contrast         BOOLEAN DEFAULT false,
  screen_reader_optimised BOOLEAN DEFAULT false,
  slow_voice            BOOLEAN DEFAULT false,
  always_show_text      BOOLEAN DEFAULT true,
  updated_at            TIMESTAMPTZ DEFAULT NOW()
)
API
GET /api/child/:id/accessibility Returns child_accessibility_prefs for the given child

Response applied to child session immediately on load. Prefs fetched at session start; cached locally for offline resilience.

CSS Application

Prefs are applied via a CSS class on the <html> or <body> element on the child's device at session start.

mode-standard mode-high-contrast mode-large-text mode-reduced-motion mode-audio-only
System Media Query Overrides
prefers-reduced-motion
Always respected regardless of mode setting. If the OS reports reduced-motion preference, animations are suppressed even in Standard mode.
prefers-contrast: more
Respected in addition to the mode setting. Applied on top of the current mode — bumps contrast ratios and removes grey-on-grey elements.

Managed By

Accessibility settings are parent-managed only. Children cannot change their own accessibility settings. Settings surface in the parent's Settings > Child > Accessibility panel. Applied from the child's next session start — not mid-session.
