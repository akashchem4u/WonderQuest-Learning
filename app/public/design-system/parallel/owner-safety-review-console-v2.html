Specification

owner-safety-review-console-v2 · Item 234
🔒 Owner Access Only

Access Control

  • Visibility: is_owner=true only. Not visible to teachers, parents, or content team.
  • All owner actions are logged to governance_log and privacy_audit_log — dismiss, block, delete, resolve.

Auto-Flag Sources

  • Safe language checker — nightly batch scan across all active skill content
  • WCAG scanner — triggered on each CI/CD deploy; results posted to safety_flags
  • PII scan — real-time on message save and content publish
  • Skip rate threshold — daily aggregate; flags skills where skip rate exceeds 60%
  • Retention job scheduler — nightly check for overdue deletion or anonymisation jobs

P0 Safety Flag Behaviour

  • Immediately sets delivery_locked=true on the affected item — no child can access it.
  • Pages on-call via configured alerting channel.
  • Non-dismissible owner banner persists until the flag is resolved.
  • P0 flags must be resolved before the next production deploy is permitted.

COPPA Compliance Check

  • Daily job: verifies every child record where age_years < 13 has parental_consent_at IS NOT NULL.
  • Any violation generates a P0 privacy alert — not dismissible until consent is confirmed.
  • Counter shown in console is a count only; no individual child data surfaced here.

No Student Data in View

The console displays counts and aggregates only. No individual child records are shown. The deletion job table surfaces UUID partial (first 8 chars) solely for operator identification of the job.
