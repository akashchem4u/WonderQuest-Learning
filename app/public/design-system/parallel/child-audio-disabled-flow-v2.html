Parent Settings Panel

Controls audio preferences per child. Changes take effect at the start of the next session.

Audio Settings for Mia
These settings apply only to this child's sessions.
Read questions aloud A voice reads each question when it appears
Play background music Gentle music during adventure sessions
Sound effects Taps, transitions, and interactive sounds
Celebration sounds Fanfares when badges and rewards are earned
When audio is off, all questions remain fully playable using text and visuals only. Your child won't miss any content.
🔒 Audio recordings are never made. Questions are read by pre-recorded voice or text-to-speech.

Accessibility & Fallback Rules

Engineering spec for audio-independent gameplay and graceful degradation.

Audio Independence Rule WCAG 1.2.1
→ WCAG 1.2.1: Audio-only content must have a text alternative. All WonderQuest question content satisfies this by design.
→ All question content is text-first. Audio is an enhancement layer — never the primary delivery mechanism.
→ No question type requires audio to answer. Images, text, and emoji are always sufficient to play.
Fallback Hierarchy
  1. 1
    Pre-recorded voice asset (primary)
    Stored audio file streamed from CDN. Fastest, most consistent quality.
  2. 2
    Web Speech API TTS
    speechSynthesis, rate 0.9 — fallback when asset unavailable or fetch fails.
  3. 3
    Text display only (silent fallback)
    When both audio sources fail. All UI audio elements hidden. Play continues fully.
  4. 4
    Never block gameplay
    Audio failure must never prevent a child from answering or advancing.

Device Audio Block Detection 
async function checkAudioAvailable() {
  try {
    const ctx = new AudioContext();
    await ctx.resume();
    return ctx.state === 'running';
  } catch { return false; }
}
Called once on session init. If false, device audio unavailable state is shown (Frame 3). Does not affect gameplay flow.
