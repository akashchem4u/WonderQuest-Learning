loading-state-system-v2.html — Loading state components. 3 strategies: (1) Skeleton Screen — for content that has a known shape (lists, cards, tables); (2) Spinner — for indeterminate short waits (<2s); (3) Progress Bar — for multi-step operations with known steps (file upload, batch import). All loading states use @keyframes shimmer (skeleton) or @keyframes spin (spinner). Target: never show spinner >2s — if loading >2s, switch to skeleton. Child loading states use playful animated star/sparkle instead of generic spinner.

Strategy Decision Tree

// Loading strategy decision tree if (knownShape && loadTime > 500ms) → skeleton screen else if (loadTime < 2s) → spinner (inline or centered) else if (multiStep) → step progress bar else → skeleton screen (fallback)

Timing Thresholds

Duration Indicator Rationale
< 100ms None Instant — no indicator avoids flash
100ms – 500ms Inline spinner only Short wait — full skeleton would flash awkwardly
500ms – 2s Skeleton screen Best perceived performance — user sees layout immediately
> 2s Skeleton + time estimate Long wait needs expectation setting; reduces abandonment

Shimmer CSS

@keyframes shimmer { 0% { background-position: 200% 0 } 100% { background-position: -200% 0 } } .skeleton-block { background: linear-gradient( 90deg, #161b22 25%, #1e2530 50%, #161b22 75% ); background-size: 200% 100%; animation: shimmer 1.5s ease-in-out infinite; border-radius: 6px; }

Spinner CSS

@keyframes spin { to { transform: rotate(360deg) } } .spinner { border: 2px solid rgba(255,255,255,.1); border-top: 2px solid #50e890; border-radius: 50%; animation: spin 0.8s linear infinite; } /* Child-facing star spinner */ @keyframes twinkle-spin { 0% { transform: rotate(0deg) scale(1) } 50% { transform: rotate(180deg) scale(1.15) } 100% { transform: rotate(360deg) scale(1) } }

Accessibility Requirements

  • role="status" All spinner elements must carry role="status" so screen readers announce the loading state without interrupting the user.
  • aria-label Spinners must have descriptive aria-label="Loading" (or more specific e.g. "Loading score"). Do not rely on visible text alone.
  • aria-busy Skeleton screen containers must set aria-busy="true" on the wrapper element while data is loading; remove the attribute (or set to false) when data arrives.
  • role="progressbar" All determinate progress bars require role="progressbar" with aria-valuenow, aria-valuemin="0", and aria-valuemax="100" kept current as progress changes.
  • prefers-reduced-motion Wrap all animation declarations in @media (prefers-reduced-motion: no-preference); fall back to static skeleton blocks and an opacity pulse for users who have reduced motion enabled.

Usage Rules by Route

Route audience Spinner style Notes
Child (play, home) Star / sparkle spinner Never use ring spinner in child-facing routes
Parent, Teacher Ring spinner (SM–LG) Standard ring; skeleton for >500ms waits
Owner / Admin Branded WQ spinner (XL) Use branded variant for full-page loads only
Content team Ring spinner (SM–MD) Inline + skeleton; step progress for batch ops
