Spec — Beta Tester Feedback Summary Owner Only
Privacy
Identifier: All feedback identified by tester_label = school name or cohort (never personal name unless tester_consent_to_name = true)
Aggregate only: This view shows no individual feedback items — those are in beta-feedback-resolution-loop and beta-issue-report-sheet
Beta-only data: Tagged with beta_source=true; hidden after GA in owner dashboard
Owner gate: is_owner=true required; content team sees only content-related themes
Sentiment Analysis
Method: Keyword extraction from feedback body; manual classification by content team
Human-in-the-loop: No AI-only sentiment — human review required for negative themes
Word cloud source: Frequency-ranked terms from aggregated feedback text
Classification: Positive / Negative / Neutral assigned by content team, not automated
Database Schema
Table: beta_feedback_summaries
Columns: week, total, bugs, suggestions, ratings, observations, resolved_count
JSONB column: top_themes JSONB — stores ranked theme list
Refresh: Weekly refresh cycle; content team updates manually after weekly review session
Access Control
Owner only: Full summary view requires is_owner=true gate
Content team: Scoped to content-related themes only (P-band coverage, explainer clarity)
GA behaviour: beta_source=true items hidden from dashboard after general availability launch
Linked views: Individual items visible in beta-feedback-resolution-loop-v2 and beta-issue-report-sheet-v2 only
Data Flow & Governance
Collection
Feedback submitted via beta-feedback-launcher
Structured fields: type, body, tester_label, session_id
Auto-tagged beta_source=true at submission
Processing
Content team keyword extraction weekly
Theme ranking by mention count
Human classification for negative sentiment
Output
This summary (owner view)
Content team scoped report
Resolution loop for engineering
