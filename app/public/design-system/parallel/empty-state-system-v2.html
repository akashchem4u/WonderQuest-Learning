empty-state-system-v2.html — Empty state components. Shown when: table has 0 rows, list is empty, search returns 0 results, no notifications, no content yet. 3 categories: (1) First-Run empty (user hasn't set things up yet — encouraging, actionable), (2) Search/Filter empty (no results for query — suggests refinement), (3) No-data empty (data exists but period has no entries — factual, neutral). All adult empty states include a primary CTA. Child empty states use playful language and emoji. Never show 'No data' to children — use 'Nothing here yet, keep exploring!'.

Empty State Categories 3 types
Category When shown Tone
First-Run User hasn't set things up yet. No data has ever existed. Encouraging, actionable. Explain what will appear and how to start.
Search / Filter Query or filter combination returns 0 results. Neutral, helpful. Always provide a clear escape hatch (clear filters).
No-Data Data exists in other periods but this period has no entries. Factual, neutral. Show the period, suggest widening scope.
Copy Guidelines by Persona Writing rules
Persona Tone & pattern Example title
Adult (all) Factual + actionable. "No [items]" + reason + next step. "No students added yet"
Child Celebratory / encouraging. Never "empty" / "no data" / "nothing". "More quests coming soon!"
First-Run Explain what will appear here when data exists. "Students will appear here once they join…"
Search empty Always include a "clear filters" escape hatch. Show spell-check if applicable. "No results for 'fraxons'"
Permission No CTA. Explain restriction, point to administrator. "No content available for your role"
Never Do Forbidden
✗ Never use "Error loading data" in an empty state — that belongs in an error state component.
✗ Never show "No data" or bare null states to children — use positive framing always.
✗ Never show financial data, school admin context, or intervention labels in child-facing empty states.
✗ Never show a message reply / compose CTA in child messaging empty states.
✗ Never leave a search empty state without an escape hatch (clear filters, broaden date, etc.).
Always Do Required
✓ All adult empty states include a primary CTA (unless permission-blocked or purely informational).
✓ Icon circle uses a soft background matching the persona color — never a hard-filled circle.
✓ First-run empty states explain what will appear once data exists.
✓ Child empty states always use emoji, encouraging language, and positive framing.
✓ Use animated: true for child empty states to improve delight.
