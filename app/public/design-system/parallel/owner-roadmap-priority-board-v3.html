Component: owner-roadmap-priority-board-v3.html
Product roadmap kanban. 4 columns: Now / Next / Later / Won't Build. Quarter selector. Drag-and-drop in production. Evidence count links to feedback items.

Evidence Count

Each card shows a count of feedback/feature requests linked to that roadmap item. Feedback items are linked via a roadmap_item_id foreign key when an owner marks a feature request as "Add to Roadmap". This count is the primary signal for prioritisation discussions.

Won't Build Column

Items in Won't Build are not deleted — they remain visible with 60% opacity so the team remembers what was considered and why it was rejected. "Won't Fix" feedback replies reference these items by description. Won't Build items can be reopened to Later if context changes.

Effort Estimates

S = 1 sprint, M = 2–3 sprints, L = 1 quarter, XL = multi-quarter. These are rough estimates — not commitments. Engineering provides refined estimates during sprint planning.

Privacy

Roadmap items reference aggregate patterns ("23 feedback requests" not individual names). No individual teacher or school names on cards. Internal strategic notes (if any) visible to owner only.
