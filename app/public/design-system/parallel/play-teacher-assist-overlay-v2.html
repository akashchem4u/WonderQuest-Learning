Assist actions
Session note PRIVATE — TEACHER ONLY
Stored per class_id · FERPA education record · teacher of record + school admin only
Teacher sees: first name, current skill, band, XP, questions attempted.
Teacher does NOT see: per-question accuracy, individual attempt counts, or which specific questions were answered.
Access & Scope
  • Who can use: Teacher persona only, during active classroom sessions
  • Activation: Teacher opens assist panel from their own dashboard — not from or on any child device
  • Availability: Only while a class session is live (session.status = 'active')
Child Privacy
  • Teacher sees: first name, current skill name, band, XP total, questions attempted (not answered correctly)
  • Teacher does NOT see: per-question accuracy, individual attempt counts, or specific questions answered
  • Session notes are private to teacher only — not visible in parent dashboard, not exported in parent reports
Encouragement Messages
  • Free-text entered by teacher; max 80 characters; auto-profanity-filtered before delivery
  • Stored in teacher_messages table; platform-routed only — never email or SMS
  • Child sees as play banner only; cannot reply; banner is non-blocking
  • WCAG: role="status" aria-live="polite"; auto-dismisses after 5 seconds
  • Positive framing only — no "you got that wrong"-type language; enforced by profanity + sentiment filter
Hint Unlock
  • Sets hint_unlocked=true on the active question attempt record
  • Identical outcome to child-requested hint — no additional data stored, no XP adjustment
Skip Skill
  • skip_reason='teacher_assist'; XP not deducted; event logged to play_assist_actions
  • Escalation-level action — not intended for routine use
Session Notes
  • Private to teacher only; stored as FERPA education record per class_id
  • Accessible only to teacher of record + school admin
  • Never visible in parent dashboard; never exported in parent-facing reports
DB Schema
play_assist_actions (
  id           UUID,
  session_id   UUID,
  question_id  UUID,
  action_type  ENUM('encouragement','hint_unlock','skip_skill'),
  triggered_by UUID,  -- teacher_id
  student_id   UUID,
  created_at   TIMESTAMPTZ
)

teacher_messages (
  id              UUID,
  from_teacher_id UUID,
  to_child_id     UUID,
  body            TEXT CHECK (length(body) <= 80),
  delivered_at    TIMESTAMPTZ,
  seen_at         TIMESTAMPTZ,
  channel         ENUM('play_banner','notification') DEFAULT 'play_banner'
)
Child Safety Rules
No timers
No countdown or time pressure anywhere in the assist flow or child play screen.
No accuracy %
Teacher stats show attempts only — never accuracy percentages or pass/fail indicators.
Positive framing only
Teacher messages must be encouraging. Auto-filter blocks negative-framing language before delivery.
No names in parent messages
Individual student first names never appear in parent-facing reports or notifications.
Adventure language throughout — all assist actions described as "helping on the quest," not corrections or failures.
FERPA & WCAG
  • FERPA: Session notes = education record; stored per class_id; accessible only to teacher of record + school admin
  • WCAG: Encouragement banner uses role="status" and aria-live="polite" — announced to screen readers without interrupting play flow; auto-dismisses (not blocking)
