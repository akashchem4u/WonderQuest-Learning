Database Schema

content_publish_events {
  id               UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  title            VARCHAR(255) NOT NULL,
  content_type     ENUM(
                     'theme_pack',
                     'quest_batch',
                     'band_update',
                     'system_update'
                   ) NOT NULL,
  status           ENUM(
                     'draft',
                     'in_review',
                     'approved',
                     'scheduled',
                     'published',
                     'deferred'
                   ) NOT NULL DEFAULT 'draft',
  scheduled_at     TIMESTAMPTZ,
  published_at     TIMESTAMPTZ,
  assigned_to      UUID REFERENCES users(id),
  release_version  VARCHAR(20),
  notes            TEXT,
  created_at       TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at       TIMESTAMPTZ NOT NULL DEFAULT now()
}
-- Privacy: No child PII stored in content metadata columns

Status Lifecycle

draft → in_review → approved → scheduled → published
↳ Any state (except published) can transition to deferred — requires notes field to be populated.

Business Rules

  1. No child PII — Content metadata (title, notes, release_version) must never contain child personal identifying information. Validated server-side before write.
  2. P0 quest batch owner approval — Quest batches tagged as P0 priority require explicit Owner role approval before status can advance beyond in_review. Content Team alone cannot approve P0 batches.
  3. Theme pack child safety sign-off — All theme_pack items require a child safety review sign-off (separate safety_sign_off_at timestamp) before transitioning to approved or scheduled.
  4. Deferred items require notes — An item cannot be set to deferred status unless the notes field is non-empty. UI enforces this; API validates server-side and returns HTTP 422 on violation.
  5. Calendar max 3 events per day cell — The calendar view renders at most 3 event cards per day. If a day has more than 3 events, a "+N more" overflow chip is shown; clicking it opens the full list filtered to that date.
