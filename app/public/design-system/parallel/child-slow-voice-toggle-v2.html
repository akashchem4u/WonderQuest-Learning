Child-controlled setting: The slow voice toggle is the one audio accessibility setting children can control directly during play. Audio on/off is managed by a parent. Speed (Normal vs Slow) is the child's choice — they can flip it at any time using the 🐢/🐇 buttons or the pill in the top-right of the play screen. If a parent has locked the speed to Slow, the toggle is hidden and the child always hears the slower pace.

During-play speed control — child-accessible

Play Screen — Speed Toggle Demo
🐢 Slow
▶ Normal
The toggle pill lives in the top-right corner of every play screen. Only two states: 🐢 Slow and ▶ Normal. No 0.5×, no 2×. Tap once to switch — it stays that way for the whole session.
Speed persists across questions within the session — resets to Normal at next session start (unless parent-locked to Slow).
Default: Normal speed. Parent can override default to Slow via settings.
🔒
Parent-locked state: When a parent has enabled "Lock voice to slow speed," the speed pill is not shown to the child. The question is always read at 0.75× rate silently in the background. The child sees no speed button and experiences smooth, slow narration by default.

DB schema

child_audio_prefs — relevant columns
CREATE TABLE child_audio_prefs (
  child_id                    UUID    PRIMARY KEY,
  voice_speed                 ENUM('normal', 'slow')  DEFAULT 'normal',
  voice_speed_locked_by_parent BOOLEAN DEFAULT false,
  -- audio_enabled managed separately by parent
  updated_at                  TIMESTAMPTZ DEFAULT now()
);
Web Speech API — readAloud() implementation
function readAloud(text, speed = 'normal') {
  const utterance = new SpeechSynthesisUtterance(text);
  utterance.rate  = speed === 'slow' ? 0.75 : 0.9;
  utterance.pitch = 1.0;
  speechSynthesis.speak(utterance);
}
Access & Control
Who can toggle
Child (during play), unless parent-locked. Parent can always change via Settings.
When parent-locked
Speed pill is hidden from child. Always reads at slow rate. Child cannot override.
Default state
Normal speed (rate: 0.9) unless parent has locked to Slow.
Speed Values — SpeechSynthesis API
Normal
utterance.rate = 0.9 — slightly below browser default (1.0) for child clarity
Slow
utterance.rate = 0.75 — deliberate pace, supports emerging readers
Not available
0.5×, 1.25×, 2× — intentionally excluded. Two states only: Normal and Slow.
Pitch
utterance.pitch = 1.0 — natural voice pitch, not modified by speed change
Pre-recorded Audio Fallback
Pre-recorded files
If using static audio assets, slow speed is not available via time-stretching. Falls back automatically to Web Speech API TTS for slow mode.
Fallback UX
Child sees no difference in the toggle UI. System silently uses TTS when slow is requested and a pre-recorded file exists for that content.
Timing & Persistence
Timer during readout
No timer. Answers never auto-advance. The child answers when ready, after listening at their own pace.
Persistence scope
voice_speed stored in child_audio_prefs. Session caches it. Does not reset per question. Resets to Normal at next session start (unless locked).
Accessibility Benefit
Supports
Emerging readers, English Language Learners (ELL), auditory processing differences, younger children in P0 Starters band.
WCAG 1.4.2
WCAG 1.4.2 Audio Control User can pause/stop audio. Speed toggle satisfies user control requirement. Slow speed as default (when locked) reduces cognitive load without removing control.
Band Considerations
P0
Starters — Short words, simple sentences. Slow reading strongly recommended as default. Parent-lock to Slow encouraged during setup for youngest readers.
P1
Explorers — Default Normal. Child can self-select Slow if needed. Most will toggle off after a few sessions as confidence grows.
P2
Adventurers — Default Normal. Slow rarely needed but kept available for multi-syllable vocabulary questions.
P3
Champions — Default Normal. Speed toggle still accessible but content is designed for fluent readers; slow mode rarely invoked.
