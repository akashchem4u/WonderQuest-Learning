Disclaimer & Methodology

🔎 What This Comparison Shows

Included in this comparison

  • Activity patterns — which skills Emma has explored and which quests she has attempted
  • These patterns compared to same-age peers (children aged 8) using WonderQuest
  • Three simple buckets: exploring earlier, exploring at a similar pace, or exploring later
  • Updated monthly using aggregated, anonymised cohort data

Not included in this comparison

  • How well Emma performed on specific questions or activities
  • Whether Emma is "advanced" or "behind" — we don't use that language
  • Any prediction of future academic performance
  • Accuracy scores, assessment grades, or standardised test results

Data source

Anonymised WonderQuest users Age band: 8-year-olds Min cohort: 50 children Updated monthly
  • Data is aggregated at the cohort level; no individual child's data from other users is shared with you
  • Emma's data is not shared with other parents or visible to the cohort
  • Cohorts with fewer than 50 children do not generate a comparison to protect privacy
Legal note
FERPA: This data is used for informational purposes only and does not constitute an educational record disclosure to third parties.
No causal claims: WonderQuest makes no claims about learning effectiveness based on this comparison.
No external benchmarks: This comparison does not reference any standardised external assessment (e.g., state standards, national assessments).
Engineering & Product Spec

🔒 Privacy

Cohort aggregation: Comparison uses cohort aggregates only (min n=50); no individual child data from other users is exposed.
Emma's data: Emma's activity data is NOT shared with the cohort or other parents.
Privacy threshold: Cohorts below 50 children suppress comparison output entirely.
Storage: Cohort buckets stored at child_age_comparisons; no raw peer data retained.

✅ Disclaimer & Consent

Gate: Checkbox acknowledgment required before first view.
Storage: Acceptance stored in parent_preferences.comparison_disclaimer_accepted_at (ISO timestamp).
Re-show: Gate re-shown if methodology version number changes (version stored alongside timestamp).
UX: "View Comparison" button disabled until checkbox is checked.

💬 Language Rules

Never "below average", "above average", "struggling", "behind", "failing", percentile numbers (e.g. "72nd percentile")
Use "exploring earlier", "exploring at a similar pace", "exploring later"
Tone Positive, forward-looking, adventure-framed language throughout. All subject labels end with 🌟.

👥 Personas

Parent: Sees per-child comparison card (this component).
Teacher: Sees class-level comparison only, not per-child data.
Child: Never sees comparison data. This card is parent-only.
Admin: Full cohort stats in separate admin dashboard (out of scope here).

📄 DB Schema

child_age_comparisons ( child_id UUID NOT NULL REFERENCES children(id), age_years SMALLINT NOT NULL, subject_id UUID NOT NULL REFERENCES subjects(id), comparison_bucket ENUM( -- values: 'earlier', 'similar', 'later' ) NOT NULL, cohort_size INT NOT NULL CHECK (cohort_size >= 50), methodology_ver SMALLINT NOT NULL DEFAULT 1, updated_at TIMESTAMPTZ NOT NULL DEFAULT now(), PRIMARY KEY (child_id, subject_id) ); -- Refreshed monthly via scheduled job. -- cohort_size enforces privacy threshold at DB level.
