Component: owner-route-health-compact-v3.html
Compact route strip. Used as: (A) home page route health widget card, (B) horizontal pill strip embedded in any owner page. Overwrite of Batch 1 version — updated to match v3 owner design tokens.

Sort Order

Down routes always first. Then degraded/slow. Then healthy sorted alphabetically. This ensures issues are never hidden when strip is truncated on narrow viewports.

Widget vs Strip

Widget (home page card): shows summary chips (healthy/degraded/down counts) + list of top routes. "View all →" navigates to full route health page. Strip (inline): horizontal pill row; degraded routes have inline expand on click. Both use same data source.

Reports Route SLA Exception

Reports/Export route has a custom p95 threshold of 500ms (not the global 200ms). This is intentional — report generation is expensive. The inline expand note clarifies this when the owner clicks on it. The pill still shows amber to indicate "different from standard" rather than "problem".

Refresh

Both widget and strip inherit the 60s refresh cadence from the route health monitoring system. During active P0: accelerates to 10s. A "Refreshed N ago" label is shown in the strip header. Widget on home page shows last refresh time in the card footer.
