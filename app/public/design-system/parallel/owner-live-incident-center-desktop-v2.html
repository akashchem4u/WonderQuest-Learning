Component: owner-live-incident-center-desktop-v2.html
Active incident management workspace. 3 states: Active P0 / All Clear / Post-Incident Summary. Distinct from route health (passive monitoring).

Incident Declaration Flow

"+ Declare Incident" (from All Clear state) opens a modal: severity (P0/P1/P2), route(s), description. On submit: PagerDuty alert created, #incidents Slack message sent, teacher-facing status page updated ("We're investigating"), incident center switches to Active state.

Auto-Declaration

Incidents can be auto-declared by the monitoring system (when route goes DOWN or error rate exceeds threshold). Auto-declared incidents appear with "Auto-detected" label in the timeline. Owner can add updates and resolve.

Teacher Status Page

On P0 declaration: auto-post to public status page: "We are currently investigating an issue. Play sessions continue to work normally." On resolution: auto-update: "This issue has been resolved." Individual route or technical details are NOT disclosed on the teacher-facing status page.

Slack Integration

All timeline updates are posted to #incidents Slack channel automatically. Resolution closes the thread with a summary. Post-mortem link added to thread on scheduling.

Post-Mortem

Post-incident summary auto-generates: root cause, affected schools, TTD/TTA/TTR metrics, SLA status. "Schedule Post-Mortem" creates a calendar invite for the engineering team. Post-mortem document template pre-filled with incident data.
