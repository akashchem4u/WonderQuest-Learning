🗄 DB Schema

-- Current platform status (one active row at a time) CREATE TABLE platform_status ( id UUID PRIMARY KEY, status status_enum NOT NULL, message TEXT, affected_components TEXT[], started_at TIMESTAMPTZ, resolved_at TIMESTAMPTZ, eta_mins INT, updated_at TIMESTAMPTZ NOT NULL ); -- Timeline entries for incident history CREATE TABLE status_incidents ( id UUID PRIMARY KEY, status_id UUID REFERENCES platform_status, message TEXT NOT NULL, created_at TIMESTAMPTZ NOT NULL ); -- Per-user dismissal + remind state CREATE TABLE status_dismissals ( user_id UUID REFERENCES users, status_id UUID REFERENCES platform_status, dismissed_at TIMESTAMPTZ NOT NULL, remind_at TIMESTAMPTZ, PRIMARY KEY(user_id, status_id) ); -- Active sessions: live count -- active_sessions_now = SELECT COUNT(*) FROM play_sessions WHERE last_ping_at > NOW() - INTERVAL '90 seconds';

🔌 API & Polling

-- Poll endpoint (60 s interval, MVP) GET /api/status/current -- Response shape { "status": "operational", "message": null, "affected": [], "started_at": null, "eta_mins": null, "updated_at": "2024-03-24T09:35:00Z", "incidents": [], // timeline entries "active_sessions_now": 1247, "uptime_30d_pct": 99.97, "dismiss_state": { // null if not dismissed "dismissed_at": "...", "remind_at": "..." } } -- No WebSocket for MVP. Polling via setInterval(60000). -- On tab focus, an immediate poll is triggered.

→ Visibility enforced server-side in API middleware; client never receives strip data for hidden personas.

🔔 Dismissal & Notifications

→ operational: dismissible (session-only via sessionStorage). No server record needed.
→ degraded: not dismissible while status is active. Strip must remain visible.
→ incident (P0): not dismissible. Also triggers push notification (FCM/APNs) within 60 s of status change to all teacher/parent/owner devices with notifications enabled.
→ maintenance: dismissible. Dismissal writes to status_dismissals with remind_at = window_start - 1h. Push reminder sent at remind_at.
→ On next poll after remind_at, API returns dismiss_state: null, causing strip to re-appear.
→ Strip is re-shown immediately when status escalates (e.g., operational → incident), regardless of dismiss state.
