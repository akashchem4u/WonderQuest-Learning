⚠ Drag handle (bottom sheet) — 40px tall
The visual drag handle pill is only 40px tall, 4px below the 44px minimum.
Resolution: Wrap in a transparent hit-area container with min-height: 44px and vertically center the visual element inside. This maintains the visual design while meeting the touch target requirement.
⚠ Speed toggle pill — 32px tall
The speed toggle pill is 32px tall, well below minimum. This is a secondary control that appears in some play screens.
Resolution: Apply padding: 6px 0 to expand the hit area to ≥44px while keeping the visual pill at 32px. Alternatively, increase the pill height to 44px for P0–P1 bands. Use ::before pseudo-element hit area expansion as a last resort.
/* WonderQuest touch target minimum enforcement */ .wq-interactive { min-height: 44px; min-width: 44px; /* For small visual elements, expand hit area with padding */ padding: calc((44px - 1em) / 2); }
Standard
WCAG 2.1 Success Criterion 2.5.5 (AA) — Target Size ≥ 44×44 CSS pixels for all interactive elements presented to users.
WonderQuest Target
≥ 44×44px for all interactive elements (WCAG AA baseline). Children's primary play elements target 56px+ height. Spacing between adjacent tappable elements must be ≥ 8px to prevent mis-taps.
Hit Area Expansion
For visually small elements, use CSS padding or a ::before pseudo-element to expand the interactive hit area beyond the visual boundary. The visual appearance need not change — only the tappable region is enlarged.

Special Consideration — P0 Band (ages 4–6)
Answer buttons scaled to 72px+ height. Extra spacing applied between all interactive elements. No small controls (speed toggles, close icons) rendered without expanded hit area. Layout must accommodate imprecise motor control typical of pre-K children.
Testing Protocol
All child-facing screens must pass touch-target audit in CI using automated tool + manual check on real device. Smallest supported device: iPhone SE 4.7" (375px logical width). Two-pass approach required: automated scan first, then manual verification on physical device.
Automated Test (Cypress)
cy.get('[data-interactive]').each(el => { expect(el.outerHeight()).to.be.gte(44) })
Never
Never use small icon-only buttons in child play screens without an expanded hit area. Never place interactive elements closer than 8px to another interactive element. Never rely solely on visual size to meet touch target requirements — always verify the rendered hit area.
