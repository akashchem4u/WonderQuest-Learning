Adventure Band Assignment
How age and grade automatically map to each child's adventure level. Band labels are always shown with adventure names — never grade comparisons.
Age Grade Adventure Band Internal ID
4 Pre-K ⭐ Starter Explorer P0
5 K ⭐ Starter Explorer P0
6 Grade 1 ⚡ Rising Explorer P1
7–8 Grade 2–3 🎯 Quest Master P2
9–11 Grade 4–5 🏆 Champion Explorer P3
Important: Band labels shown to children and parents use adventure names only (e.g. "Quest Master", "Rising Explorer"). Internal band IDs (P0–P3) are used in the database and shown only in teacher/owner views — never to families.
Manual Override
A parent can adjust their child's adventure band if the auto-assigned level doesn't feel right.

Rules:
  • Override is limited to 1 level up or 1 level down from the auto-assigned band
  • Override is always framed as "change adventure level" — never as grade adjustment
  • Overrides are logged but not surfaced to the child
  • Parent can revert to auto-assigned band at any time from Settings
Data Model & Privacy Spec
Schema and rules governing how household data is stored during and after onboarding.
Database Schema — children table
children ( id UUID PRIMARY KEY, parent_id UUID REFERENCES users(id), first_name TEXT NOT NULL, -- first name only, no last name stored age INT nullable, grade TEXT nullable, -- 'prek','k','1','2','3','4','5' band_id ENUM('p0','p1','p2','p3') NOT NULL, avatar_key TEXT, -- 'bear','fox','owl','rabbit','cat','frog' preferred_themes TEXT[], -- up to 3 theme_ids audio_readout BOOLEAN DEFAULT true, audio_music BOOLEAN DEFAULT true, audio_sfx BOOLEAN DEFAULT true, created_at TIMESTAMPTZ DEFAULT NOW() )
Privacy Rules
🔒
Children: first name only. No surname, no date of birth — only age or grade range stored. Avatar is selected from a pre-defined set; no photos uploaded.
📧
Parent email is used for account login and beta communications only. Never shared with schools, third parties, or used for marketing without explicit opt-in.
🎨
Theme preferences are stored per child and used exclusively for content personalisation — making quests feel relevant and engaging. Not shared externally.
📵
No biometric data, no location data, no uploaded photos. Avatar selection is always from the fixed set of 6 animal icons.
🏷️
Band IDs (P0–P3) are internal only. Children and parents always see adventure names. No performance benchmarks or grade-level comparisons are ever surfaced to families.
Band Assignment Logic
function assignBand(grade) { if (['prek', 'k'].includes(grade)) return 'p0'; if (grade === '1') return 'p1'; if (['2', '3'].includes(grade)) return 'p2'; if (['4', '5'].includes(grade)) return 'p3'; return 'p0'; // fallback for unknown } function assignBandByAge(age) { if (age <= 5) return 'p0'; if (age === 6) return 'p1'; if (age <= 8) return 'p2'; return 'p3'; // age 9–11 }
