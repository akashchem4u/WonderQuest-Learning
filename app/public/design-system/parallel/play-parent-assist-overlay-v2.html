PIN Management
How the Parent PIN works
🔑
Set during onboarding Parent PIN is created during family account setup. Defaults to the account PIN if one exists.
⚙️
Where to change it Parent Settings › Security › Change Parent PIN
🛡️
How it's stored Never stored in plain text. Stored as:
SHA-256(pin + user_id)
in the parent_pins table.
🔝
Offline verification Same-device PIN check is fully local — hashed value cached in secure session storage. No network required.
🚫
Child cannot access The Parent Help chip is visible but PIN-gated. 3 wrong PINs triggers a 30-second cooldown.
Parent Assist — Full Specification
Behavioural rules, DB schema, WCAG notes, and child safety guarantees.
👤 Access & Permissions
Who can use
Parent persona only. Never teacher-triggered. Child cannot open the overlay.
Auth method
Parent PIN (same device) or authenticated parent session (separate device)
Teacher access
Never. Parent assist actions are hidden from teacher view entirely.
Logging
Actions logged to parent's own activity only. Not visible to teachers or admins.
⏸️ Session Behaviour
Pause effect
No question timer exists. Audio readout halts. Adaptive session clock pauses.
Resume
Session resumes exactly where it left off on "End Assist".
XP during pause
No XP is gained or lost while overlay is open. Stars remain safe.
⏭️ Skip Behaviour
skip_reason
'parent_assist' stored on the question attempt record
XP impact
None. XP safe
Marked wrong?
No. Question skipped without any penalty flag. No penalty
Reappearance
Question may return in a future session (spaced repetition continues normally)
UI copy
"Try a different adventure!" — never "Give up" or "Skip"
💡 Hint Behaviour
hint_shown
hint_shown = true stored on question attempt
Star impact
None. Stars and progress unaffected. Stars safe
Overlay state
Overlay closes after hint shown — returns child to active play
Hint display
Gentle glow card below question. Adventure language. Not authoritative.
♿ WCAG / Accessibility
Overlay uses role="dialog" with aria-modal="true"
Focus is trapped inside the overlay while it is open
Focus returns to the "Parent Help" chip when overlay closes
Overlay has aria-label="Parent Assist mode"
All action cards are keyboard-focusable and activatable via Enter/Space
Colour contrast ≥ 4.5:1 for all text against card backgrounds
PIN entry keyboard is screen-reader-labelled (aria-labels on each digit button)
🔐 Privacy
Teacher view
Parent assist actions never appear in teacher dashboards or class reports excluded
Parent view
All assist actions appear in Family Activity log (parent-only screen)
Data retention
Assist actions retained per household data policy (default 12 months)
Shown in-app
Privacy notice shown on every overlay open — "not recorded or shown to teachers"
Database Schema
CREATE TYPE action_enum AS ENUM ('read_aloud', 'show_hint', 'skip'); CREATE TABLE play_assist_actions ( id UUID PRIMARY KEY DEFAULT gen_random_uuid(), session_id UUID NOT NULL REFERENCES play_sessions(id), question_id UUID NOT NULL REFERENCES questions(id), action_type action_enum NOT NULL, triggered_by UUID NOT NULL REFERENCES users(id), -- parent_id created_at TIMESTAMPTZ NOT NULL DEFAULT now() ); -- Index for parent's Family Activity log (parent_id + time desc) CREATE INDEX idx_assist_triggered_by ON play_assist_actions (triggered_by, created_at DESC); -- Index for session-level lookups CREATE INDEX idx_assist_session ON play_assist_actions (session_id);
🌟
No Shaming — Ever
Skipping is presented as "Try a different adventure!" — a positive pivot, never a failure. Hints are celebrated as clues unlocked, not signs of struggle. Parent assist exists to support, encourage, and make learning feel safe. No action taken through this overlay can decrease a child's stars, XP, or progress. The adventure always continues.
