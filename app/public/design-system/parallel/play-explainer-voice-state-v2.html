TTS Integration

  • Use SpeechSynthesisUtterance (Web Speech API) or cloud TTS
  • Start speech on panel open if user has previously interacted
  • If blocked: show Audio Blocked state, wait for tap
  • Step highlight: track by character offset or sentence index
  • Fire boundary event on utterance to advance highlight
  • ❌ Never start TTS without at least one prior user tap this session

Step Highlighting

  • Previous steps: .done-step (green num + faded text)
  • Current step: .active-step (violet bg, bright text, violet num)
  • Upcoming steps: default dim style
  • All steps done → Done state (all .done-step)
  • On "Hear again": reset to Idle, replay from step 1
  • ✅ Steps highlighted in sync, not all at once

Coach Avatar States

  • Idle: idle-ring gentle pulse (2s ease-in-out)
  • Speaking: ring spinning 360° (1.2s linear infinite)
  • Done: idle-ring green tint (#50e89044)
  • Wave bars: visible during Speaking only
  • Coach speech text changes per state
  • 4 personas: Leo 🦁 / Buddy 🐧 / Whisper 🦉 / Zap 🤖

Audio Blocked Handling

  • Detect: speechSynthesis.speak() + check speaking === false
  • Amber banner: #f0a020 — NOT red (minor issue)
  • Tap-to-play button triggers speechSynthesis.resume()
  • After tap: transitions to Speaking state
  • ❌ Never show technical error text to child
  • ✅ Child always sees clear, action-oriented tap target

"Got it!" Gating

  • Disabled (opacity 0.45) during Idle + Speaking + Blocked
  • Enabled after: speech ends (utterance.onend) OR Skip
  • Skip always enabled — never gated
  • "Hear again" only in Done state
  • ❌ Never auto-advance after TTS ends

Fallback Chain

  • 1st choice: Video explainer (if asset exists)
  • 2nd choice: Voice explainer (TTS, this file)
  • 3rd choice: Silent text panel (play-explainer-panel-v3.html)
  • 4th: Abbreviated panel (returning child, 2nd–3rd session)
  • 5th: No panel (4th+ session — child knows this type)
  • All fallbacks use same bottom sheet layout
