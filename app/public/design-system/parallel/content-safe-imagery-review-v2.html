Reviewer Notes
Actions

All images in the WonderQuest asset library must pass these checks before being published to any skill or question. Auto checks run on upload via background job. Manual checks require a content team reviewer to sign off.

1
Flash / strobe risk Auto
Frame analysis checks luminance transitions per second. Three or more high-contrast transitions per second exceeds WCAG 2.3.1 (Three Flashes or Below Threshold). Threshold: ≥3 luminance changes/sec in a region covering >25% of viewport.
🔴 Block if triggered — delivery_locked=true
2
Age-appropriate content Manual
Reviewer confirms image contains no violence, physical distress, frightening imagery, adult themes, or content inappropriate for children aged 3–12. Includes background elements, props, and implied narrative.
🔴 Block if fail — requires human sign-off
3
Representation & diversity Manual
Reviewer checks that characters across the skill set are diverse in appearance, that no single group is over- or under-represented, and that no harmful stereotypes are present in visual depictions.
🟣 Flag if concern — reviewer escalates for art direction
4
Alt-text present Auto
All published images must have a non-empty alt_text value in the asset library. Missing alt-text fails the WCAG accessibility gate in the publish pipeline but does not hard-block delivery.
⚠ Warn if missing — blocks WCAG score in publish gate
5
Image dimensions ≥ 200 × 200 px Auto
Minimum pixel dimensions enforced on upload. Images below threshold display poorly on low-density screens and may appear pixelated on retina displays. Checked against dimensions field in asset_library.
⚠ Warn — reviewer may approve small images with justification
6
File size < 500 KB Auto
Enforces performance budget. Large images increase load times, especially on low-bandwidth connections common in target markets. Checked against file_size_kb. Images 500–800 KB trigger a warning; images >800 KB are rejected at upload.
⚠ Warn (500–800 KB) — hard reject above 800 KB at upload
7
No visible personal data (faces, names, addresses) Auto + Manual
AWS Rekognition (or equivalent) runs face detection on upload. Any face detected in a child-targeted image triggers mandatory manual review. Reviewer also checks for visible names, home addresses, school names, phone numbers, or other PII in image content.
🔴 Block if PII confirmed — requires human sign-off
8
Cultural appropriateness Manual
WonderQuest serves a global audience. Reviewer confirms imagery does not include symbols, gestures, colours, or depictions that carry negative or offensive connotations in known target markets (UK, US, AU, India, SEA). Escalate to regional content advisor if uncertain.
🟣 Flag if concern — requires human sign-off

Approval Flow

1️⃣ All auto checks must pass for a content reviewer to publish the image. A single auto-block (flash risk, PII, 404) prevents publish regardless of manual approval.
2️⃣ Manual checks 2, 3, 7, 8 require at least one content team reviewer to set review_status = approved and record their reviewer_id and reviewed_at timestamp.
3️⃣ Warnings (checks 4, 5, 6) surface in the publish gate dashboard but do not prevent publish. They are tracked in auto_check_results JSONB for audit.
4️⃣ Content team can approve / block / request re-upload via this panel. They cannot access owner-only safety console or school data.
🔒 Block = delivery_locked=true on all questions linked to the blocked image. Learners see a placeholder until a replacement image is approved and re-linked.
Database Schema

📦 asset_library

asset_library ( id UUID PRIMARY KEY DEFAULT gen_random_uuid(), type ENUM ('image','audio','video','document'), filename TEXT NOT NULL, cdn_url TEXT NOT NULL, dimensions JSONB, -- { "w": 480, "h": 480 } file_size_kb INTEGER, alt_text TEXT, review_status ENUM ('new','review','approved','blocked','rejected'), auto_check_results JSONB, -- { flash: bool, face: bool, size_ok: bool, ... } reviewer_id UUID REFERENCES users(id), reviewed_at TIMESTAMPTZ, created_at TIMESTAMPTZ DEFAULT now(), updated_at TIMESTAMPTZ DEFAULT now() )
Auto Check Pipeline

⚙️ Background job — runs on upload

Trigger Asset upload event → enqueues image_safety_check job
Output Results stored in auto_check_results JSONB on the asset row; review_status set to blocked or review as appropriate
Retry 3 retries with exponential backoff; on permanent failure → status = review with error recorded

🎭 Face detection — PII safeguard

Service AWS Rekognition DetectFaces (or equivalent on-prem model)
Trigger condition Any face detected in a child-targeted image → mandatory manual review required
Rationale Real faces in learning content are flagged as PII risk per COPPA / GDPR-K compliance policy
Outcome review_status = 'review'; auto_check_results.face_detected = true

⚡ Flash risk — WCAG 2.3.1

Rule 3 or more luminance transitions per second detected via frame analysis of animated/video assets
Standard WCAG 2.1 Success Criterion 2.3.1 — Three Flashes or Below Threshold
Outcome Auto-block — review_status = 'blocked'; delivery_locked = true on all linked questions

🔗 404 / CDN health check

Frequency Hourly CDN health check job verifies all cdn_url values are reachable (HTTP 200)
On failure Broken image → auto-blocks linked questions + fires safety_flag event to ops dashboard
Recovery Block lifted automatically when CDN URL returns 200 on next hourly check; content team notified

📝 Alt-text rule

Requirement Required for all published images per WCAG 1.1.1 (Non-text Content)
Missing alt-text Warn (not block) — surfaces in publish gate WCAG score; wcag_ready = false until resolved
Min length Alt text must be >5 characters to pass; "image", "photo", "picture" trigger a quality warning
Access Control

🔐 Content team permissions

  • Can approve, block, or request re-upload on images in review queue
  • Can add reviewer notes and dismiss auto-flags after manual inspection
  • Cannot access owner-only safety console (incident logs, school data, PII exports)
  • Cannot view school or learner data of any kind
  • Cannot modify auto-check results — auto checks are system-only writes
