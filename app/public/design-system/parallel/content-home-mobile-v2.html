Component: content-home-mobile-v2.html
Mobile content CMS home. Stat scroll. Alert strip (if blocked). Filter chips. Skill list. Bottom tab bar: Home / Flagged / Blocked / All Skills / More. View-only on mobile.

Mobile Capability Constraints

Mobile is intentionally limited to view and status-change operations. Full edit workflow requires desktop because: (1) question editing involves rich text + image/audio uploads; (2) answer key review requires seeing full question/answer layout; (3) multi-step review workflow is complex for small screen. Mobile is optimised for: checking notification about a new block, viewing skill detail, assigning to a reviewer, and confirming a status change.

Blocked Alert Strip

A red alert strip is shown below the stat scroll when any skills are blocked. It is always visible on the Home tab when blocked count > 0. Tapping it navigates to the Blocked tab. The strip disappears when all blocked skills are unblocked. This strip is different from the owner-ops-alert-banner (which is owner console only). Content team members see this content-specific alert instead.

Bottom Tab Bar

Home (default) · Flagged (badge = total flagged count, red if >0) · Blocked (badge = blocked count, always red when >0) · All Skills (full library) · More (settings, browse by subject/band, sign out). Badge on Flagged and Blocked tabs updates in real time (WebSocket or short poll).
