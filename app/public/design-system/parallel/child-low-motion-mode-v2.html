Trigger Sources
Source 1: child_accessibility_prefs.reduced_motion = true — stored in database, set by parent in child accessibility settings.
Source 2: prefers-reduced-motion: reduce OS-level media query — detected automatically on app load.
Either source triggers Low Motion. DB preference takes precedence if explicitly set by parent.
CSS Implementation
.mode-reduced-motion * { animation-duration: 0.001ms !important; animation-iteration-count: 1 !important; transition-duration: 150ms !important; } /* Exceptions */ .mode-reduced-motion .xp-bar-fill { transition-duration: 300ms !important; /* functional, not decorative */ } /* Auto-detect via media query */ @media (prefers-reduced-motion: reduce) { * { animation-duration: 0.001ms !important; animation-iteration-count: 1 !important; transition-duration: 150ms !important; } }
Exceptions
Loading skeleton shimmer is reduced to animation-duration: 0.001ms (effectively disabled — not removed).
Progress bar fill transition kept at 300ms — this is a functional state change, not decorative motion.
Access Control
Child cannot toggle this setting. It is parent-managed only, accessible via Parent Settings > Child Profile > Accessibility.
No UI control is shown to the child. Setting is silently applied on session start.
Automatic Detection
// On app load const prefersReduced = window.matchMedia('(prefers-reduced-motion: reduce)').matches; const dbReduced = child.accessibility_prefs.reduced_motion; // from session if (dbReduced || prefersReduced) { document.body.classList.add('mode-reduced-motion'); } // Live OS preference change window.matchMedia('(prefers-reduced-motion: reduce)') .addEventListener('change', (e) => { if (e.matches) document.body.classList.add('mode-reduced-motion'); // DB preference is not overridden by live OS change });
DB Schema
CREATE TABLE child_accessibility_prefs ( child_id UUID PRIMARY KEY REFERENCES children(id), reduced_motion BOOLEAN NOT NULL DEFAULT false, updated_by_parent_id UUID REFERENCES users(id), updated_at TIMESTAMPTZ NOT NULL DEFAULT now() );
Content Policy & Accessibility
No content is removed. All UI elements remain visible in both modes — only decorative motion is disabled.
WCAG 2.3.3 (AAA): Animation from Interactions — meets AAA by allowing users to disable non-essential animation triggered by interaction.
WCAG 2.3.3 · AAA · Animation from Interactions
Touch targets: All interactive elements are ≥ 44 × 44 px in both Standard and Low Motion modes.
Child Safety Rules (enforced in all modes)
No timer visible anywhere on screen.
No accuracy percentage shown to the child.
No "correct" or "incorrect" wording — positive and neutral framing only.
Adventure language used throughout ("Quest", "Realm", "Explorer") — not academic framing.
