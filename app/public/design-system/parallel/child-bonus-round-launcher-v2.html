Design Rules

  • 🚫
    No timer, ever No countdown, no expiry language on the bonus offer. The bonus is available at the child's own pace.
  • 🚫
    No accuracy percentage The screen must never display accuracy, wrong-answer counts, or performance metrics of any kind.
  • ✅
    Bonus is always optional "Maybe later →" must always be present and visually equivalent weight. No shaming copy ("Are you sure?", "Don't quit!") permitted.
  • ✅
    No penalty for skipping xp_awarded is only written on bonus completion. Skipping the bonus produces zero DB write and zero negative feedback to the child.
  • 🌙
    Quiet hours suppresses bonus CTA When the parent's quiet hours window is active, render the quiet_hours state — celebration is shown but the Start Bonus CTA is hidden. Never show an error or lock icon.
  • 🔒
    Child-safe language throughout Zero negative language. All copy must be celebratory, warm, and pressure-free. First name only — never display last name, email, or account identifiers.
  • ♿
    WCAG 2.1 AA + Touch targets All interactive elements ≥ 44 × 44 px. Colour contrast ≥ 4.5 : 1 for text. Full-screen layout on mobile (≤ 600 px).
