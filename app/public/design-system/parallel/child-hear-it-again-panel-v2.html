Hear It Again — Specification
Replay Limit
None. Child can replay as many times as needed. No cap. No penalty. Replaying is encouraged, never punished.
Replay Count Tracking
question_replays(session_id, question_id, replay_count, child_id)
Stored for content QA only. High replay count = signal that the question text may be confusing. Never surfaced to the child.
Auto-play on Load
Never. Always wait for a child tap before playing audio. Autoplay is blocked by browsers and can be overwhelming for some children.
Answer Lockout During Readout
Yes. pointer-events:none applied to all answer buttons while audio is playing. Re-enabled when utterance.onend fires (Web Speech API) or equivalent audio end event.
Slow Speed Behaviour
If slow voice is enabled for a child, "Hear it again" plays at the same slow speed. Speed setting is sticky per session. No separate fast/slow chip needed.
Audio Disabled Fallback
If audio is disabled: speaker button hidden entirely. "Hear it again" chip hidden. Question text is always shown regardless — text is never contingent on audio being available.
Accessibility WCAG 1.2.1
Text alternative exists for all audio content. If question is audio-only, visible text equivalent is available to assistive technology via aria-label or visually-hidden element.
Accessibility WCAG 1.4.2
User can pause audio. Tapping the speaker button during playback toggles play/pause. A second tap restarts from beginning. The button's aria-label updates to reflect state.
Touch Target Sizes
Speaker button: minimum 44×44px (WCAG 2.5.5).
"Hear it again" chip: minimum 48px height to support small fingers and motor impairments.
Copy Standard COPY
Always: "Hear it again"
Never: "replay" · "listen again" · "re-listen"
Consistent across all band levels (P0 Gold → P3 Coral). Copy is defined in a single i18n key: hear_it_again.
Long Question Detection
Word count > 15 triggers two-chip layout (Variant C):
"🔊 Hear the whole question" + "🔊 Hear just the last part".
Threshold configurable via CMS flag long_q_word_threshold (default: 15).
Child Safety Rules
· No timer on screen
· No accuracy % shown
· No "correct / incorrect" wording → use "Try a different one! 🌟"
· Adventure language, not academic
· Positive framing only
· Children cannot be penalised for replaying
