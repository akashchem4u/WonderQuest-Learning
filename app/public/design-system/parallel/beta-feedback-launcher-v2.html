Data Model
feedback (
  id              UUID PRIMARY KEY,
  user_id         UUID,
  type            ENUM('bug','suggestion','rating','observation'),
  title           TEXT,
  body            TEXT,
  steps_to_reproduce TEXT,
  severity        ENUM('critical','major','moderate','minor') nullable,
  screen_context  TEXT,
  persona_context TEXT,
  band_context    ENUM('p0','p1','p2','p3') nullable,
  screenshot_url  TEXT nullable,
  rating          INT CHECK(1 <= rating AND rating <= 5) nullable,
  is_beta         BOOLEAN DEFAULT false,
  beta_cohort     TEXT,
  beta_week       INT,
  ref_id          TEXT,  -- REF-YYYYMMDD-NNNN
  submitted_at    TIMESTAMPTZ DEFAULT NOW(),
  auto_priority   ENUM('p0','p1','p2','p3','feature') generated
)
Auto-Priority Mapping
critical → P0 Data loss risk — immediate escalation
major → P1 Blocks usage — high priority fix
moderate → P2 Workaround exists — next sprint
minor → P3 Cosmetic — backlog
rating → Feature Feature feedback queue
Beta Tagging & Ref ID
is_beta = true
beta_cohort auto-populated from user session (e.g. 'teacher_pilot')
beta_week auto-populated from session (e.g. 3)

ref_id formula:
'REF-' || TO_CHAR(NOW(),'YYYYMMDD') || '-' || LPAD(nextval('feedback_ref_seq')::text, 4, '0')

Post-submit route:
INSERT INTO owner_feedback_queue (feedback_id, priority = auto_priority, source = 'beta')
Routing Rules
• All beta submissions tagged with beta_source=true
• P0/P1 items trigger immediate Slack alert to #wq-incidents
• Feature ratings routed to feature-feedback queue, not bug queue
• Observations routed to UX research queue for qualitative tagging
• Screenshots uploaded to S3 bucket wq-beta-screenshots, ref stored as screenshot_url
• Tester identity linked via user_id FK — never anonymised in beta phase
• owner_feedback_queue row created synchronously on submit (not async)
