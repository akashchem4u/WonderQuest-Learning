ENG NOTE — teacher-parent-message-center-v3.html
In-platform messaging between teacher and parents (guardians). Messages are routed through WonderQuest — no direct email/phone disclosed.
Privacy: Teacher sees parent display name only. Parent sees teacher name only. No email addresses, phone numbers, or school admin visible in UI.
Thread model: One thread per student per teacher-parent pair (not per topic). Multiple children = multiple threads.
Scope: Teacher can message individual parent OR send class-wide announcement (to all parents in class). Parent-initiated messages land in teacher inbox.
Templates: Pre-written positive/neutral templates for common scenarios (progress update, band advancement, check-in request, absence follow-up).
FERPA: Messages about one student NOT shared with other parents. Class announcements contain no individual student data.
Retention: Messages retained 2 years after class year end, then archived.

Component: teacher-parent-message-center-v3.html
Data model: messages table — sender_id, recipient_id, student_id (nullable, for thread grouping), body_text, sent_at, read_at. Thread = teacher_id + parent_id + student_id grouping.
Announcement model: class_announcements table — teacher_id, class_id, subject, body, sent_at. No student_id foreign key — class-level only.

Privacy Rules

RuleDetail
Contact detailsParent email/phone NEVER shown to teacher. Teacher email/phone NEVER shown to parent. Routing is platform-only.
Individual threadOne thread per student per teacher-parent pair. Threads not shared across siblings in different classes.
Announcement ruleClass announcements must contain NO individual student names or data (FERPA). Validated server-side via name matching.
Message historyTeacher can view own sent/received. Teachers cannot view other teachers' messages.
