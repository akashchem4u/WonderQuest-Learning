ENG NOTE — content-question-browser-desktop-v3.html
Two-pane question browser for the content team. Left pane: scrollable question list with filter. Right pane: question detail with answer preview, stats, error reports, and skill linkage.
Questions vs Skills: Each Skill (SK-XXXX) contains multiple Questions (QID-XXXX). A skill typically has 4–8 questions. The question browser shows individual questions, not skills.
Flagged view: Questions with active error reports shown with red/amber left border. Wrong-answer-as-correct: the incorrectly marked answer is highlighted red in the answer preview.
Actions per question: Edit (opens editor) · Flag · Block (marks parent skill as blocked if threshold reached) · Dismiss report.
Privacy: Reporter shown as "Teacher, [School]" only. No individual teacher name. No student data.

Two-pane question browser. Left: scrollable question list with status + flag indicators. Right: question detail with answer preview, stats, error reports, skill linkage. Wrong-answer-as-correct highlighted in red. Reporter = "Teacher, [School]" only.

Question vs Skill Relationship

Each Skill (SK-XXXX) contains 4–8 Questions (QID-XXXX). Error reports are filed against individual questions (question_error_reports table with question_id FK). Auto-block threshold is evaluated at the skill level: if ≥ 3 questions in a skill have confirmed error reports, OR total error reports for the skill exceeds threshold → delivery_locked=true on the parent skill. Individual questions can be flagged without blocking the entire skill.

Wrong-Answer-as-Correct Display

When a question has a "wrong answer marked as correct" report, the detail pane highlights the incorrectly marked answer option in red. The correct answers remain highlighted in green. A fix instruction (SQL UPDATE statement) is shown for engineers. The fix must be deployed before the skill can be unblocked. This is a read-only reference — actual edits happen in the CMS editor, not in this view.

Privacy — Reporter Field

Error report cards show "Teacher, [School]" only. No individual teacher name is stored in the content team's view. The full reporter identity (teacher_id) is stored in error_reports table for audit purposes, but is not surfaced in the content browser. Student session IDs are shown as reference only (no student name or demographic data).
