Component: owner-content-health-detail-v3.html
Single-skill content health drill-down. 3 states: Blocked / Under Review / Passed. Shows error reports, question preview, review history, and actions.

Navigation

Reached by clicking a skill row in owner-content-health-desktop-v2.html or from the release gate Content category detail. Back arrow returns to content health overview.

Question Preview

Shows the specific question with the flagged error. Wrong-as-correct answers highlighted in red. Correct answer highlighted in green. SQL fix reference shown for engineers. This is read-only — question editing happens in the content management system, not here.

Actions

Mark Fixed + Unblock: requires a deploy with the fix first. Block Delivery: instant, no deploy needed. Assign to Curriculum Team: creates a review task. Dismiss: marks as false positive with mandatory reason. All actions logged in content_review_log with timestamp and actor.

Privacy

Error reporter = "Teacher, [School]" only. No individual teacher name. No student data (only aggregate skip rate and session count).
