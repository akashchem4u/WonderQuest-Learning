ENG NOTE — owner-route-health-desktop-v3.html
Full-page route health monitoring dashboard for the owner. Distinct from the compact route strip on the home screen.
Layout: Status header (healthy/degraded/down counts + overall uptime) → route table (all routes with p95/p99/uptime) → latency small multiples → sidebar (incident log + 30-day uptime).
Refresh: Route health auto-refreshes every 60 seconds. "Refreshed N min ago" shown in header.
Data: Metrics from infrastructure monitoring (Datadog/Grafana equivalent). p95/p99 thresholds: 200ms/500ms. Uptime threshold: 99.5%.
Route count: 10 routes covered. Collapsed by default; expand to see resolution steps + 6-point sparkline.

Component: owner-route-health-desktop-v3.html
Full route health monitoring page. Distinct from compact home strip. 10 routes. p95/p99/uptime/error rate. Incident banner. Latency small multiples.

Routes Monitored

10 routes: Play Session / Adaptive Engine / Assignment Engine / Teacher Dashboard / Parent Dashboard / Auth SSO / Content Delivery / Mastery Score / Notification Delivery / Reports Export.

Refresh Cadence

Normal: 60s auto-refresh. During active P0 incident: 10s refresh. Manual "Re-check now" forces immediate fetch. Timestamp shown as "Refreshed N ago".

Thresholds

p95 > 200ms = amber warning. p95 > 300ms = degraded. Error rate > 0.5% = amber. Error rate > 2% = red/down. Uptime < 99.5% = amber. Uptime < 99.0% = red.

Latency Sparklines

Mini bar charts show last 6 check intervals (each interval = refresh cadence). Bars coloured by threshold (green/amber/red). p95 displayed below each. Reports route shown as amber even when "healthy" — intentional as it always runs slower than others (generates heavy reports).

Incident Declaration

"+ Declare Incident" button opens an incident form: severity (P0/P1), affected route(s), description. Auto-creates PagerDuty alert + #on-call Slack message + incident log entry. P0 declarations also show non-dismissible banner on owner home screen.
