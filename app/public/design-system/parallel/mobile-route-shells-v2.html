mobile-route-shells-v2.html — Mobile route shell system. Defines the consistent structural skeleton for all mobile routes.
4 shell types: (1) Root Shell — bottom tabs + fixed header, for top-level tabs of each persona; (2) Child Route Shell — back header replacing bottom tabs context, for detail/drill-down screens; (3) Modal Shell — bottom sheet or full-screen modal; (4) Fullscreen Shell — for immersive experiences (play screen, onboarding).
Phone frame: 375×812px, border-radius 36px, notch at top. Safe area insets: top 44px, bottom 34px (iPhone notch+home bar).

✗ No bottom tabs

✓ Progress strip (questions)
✓ Full-bleed dark gradient
✓ Action buttons only at bottom

Onboarding Shell Rules
✓ Progress dots top
✗ No back button (forward-only)
✓ Large content area
✓ Continue CTA at bottom
✓ Skip only on non-critical steps
✓ Safe area bottom respected
Shell Anatomy
Shell Type Header Bottom Nav Scrolling Use Case
Root Fixed header (adult personas) or none (child) Bottom tabs (5 tabs) Content area scrolls Top-level routes for each persona
Child Route Back header (← title + actions) Context tabs or none Content area scrolls Detail / drill-down screens
Bottom Sheet Drag handle + optional title n/a (sits above main tabs) Sheet content scrolls Quick pickers, action menus
Full-Screen Modal ✕ Close + title + action n/a Content scrolls Settings, privacy controls, large forms
Play / Fullscreen Minimal or none (exit only) None No scroll (fixed layout) Gameplay, onboarding, immersive flows
