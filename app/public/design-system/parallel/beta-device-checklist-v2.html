beta-device-checklist-v2.html — Beta device setup checklist. Used before each beta session to verify device readiness. Two modes: Self-check (tester manually marks items) and Auto-check (app runs automated tests). Checklist categories: Network / Audio / Display / App Version / Data / Permissions. Pass threshold: all Required items must pass; Recommended items: warn if failed. Device coverage target: iOS 16+ / Android 12+ / Chrome 108+ / Safari 15+.

Database Schema
CREATE TABLE beta_device_checks (
  id              UUID         PRIMARY KEY,

Pass / Fail Rules
✅
overall_status = "ready"
All Required items PASS. Recommended items may be skipped or warned. Session may begin immediately.
❌
overall_status = "not_ready"
Any Required item FAILS. Session is blocked. Tester must resolve all required failures before proceeding.
⚠️
overall_status = "warnings"
All Required items PASS but one or more Recommended items FAIL. Session is allowed with a warning note in the beta report. Tester should document the limitation.
