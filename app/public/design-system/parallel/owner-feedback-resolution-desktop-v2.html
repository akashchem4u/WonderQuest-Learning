ENG NOTE — owner-feedback-resolution-desktop-v2.html
Post-resolution view for a feedback item in the owner workbench. Shown after a ticket is resolved, marked won't-fix, or escalated.
Layout: 2-col (main: original feedback + resolution note + owner reply + activity log) + sidebar (meta + related + reopen).
3 resolution states: Resolved (green), Won't Fix (neutral), Escalated (red). Each has distinct status banner + resolution note framing.
Reopen: Any resolved/won't-fix item can be reopened. Reopen sends item back to queue with previous priority. Activity log entry added on reopen.
Teacher notification: Teacher receives platform notification when owner replies AND when item is resolved (if reply was sent). Teacher sees status only (not internal resolution notes).

Component: owner-feedback-resolution-desktop-v2.html
Post-action view for a resolved/won't-fix/escalated feedback item. 3 resolution states covered. 2-col layout with activity log and sidebar meta.

Resolution States

Resolved: Green banner. Internal resolution note (not visible to teacher). Owner reply shown with delivery confirmation. Activity log shows full event trail. Reopen available.

Won't Fix: Neutral banner. Internal won't-fix reason. Owner reply (explaining to teacher in product-forward tone). Reopen available.

Escalated: Red banner — note this is still open (not resolved). PagerDuty incident number shown. Escalated items remain visible in queue until resolved.

Privacy

Internal resolution notes are visible to owner only — never to teachers. Teacher sees only the owner reply (delivered via platform notification). Individual teacher name never shown — school name + role only.

Reopen Behaviour

Any resolved or won't-fix item can be reopened. On reopen: item returns to queue with previous priority. New log entry added. If teacher had received a resolution notification, a correction notification is sent ("We're looking into this again").

SLA Tracking

P0: 4h target. P1: 24h. P2: 72h. P3: 7 days. Feature: best effort. SLA timers start at submission time, not at triage. Time-to-resolve shown in sidebar for resolved items (green if within SLA).
