No timer during play — this card uses only POST-SESSION duration data (recorded server-side when session ends).
Framing: Never imply child "should" spend more time — celebrate what they did.

Data source: sessions table — sum of session_duration_seconds for sessions in [week_start, week_end] window for this child_id.
No timer during play. Duration is recorded server-side when session ends. Never expose raw seconds to the client — always format as "Xh Ym".

Display Logic

ConditionPrimary value shown
total_minutes >= 60"Xh Ym" (e.g. "1h 24m")
total_minutes 1–59"Ym" (e.g. "32m")
total_minutes = 0"0m" in light grey — "No sessions this week"
First week (no prior data)No delta shown — "→ First week!" tag instead

Delta Colours

Framing Rule

Never imply the child "should" play more. On a down week, show delta but add positive framing: "Still a great effort this week!" Down arrows are OK in this parent context (weekly report space) but never frame as failure.
