ENG NOTE — parent-wrong-child-recovery-v2.html
Shown when a parent realises they linked the WRONG child — e.g. linked "Liam" but the account on the device belongs to "Maya".
Three sub-scenarios:
(A) Parent linked wrong child immediately after linking — wants to undo before any sessions start.
(B) Parent linked wrong child but sessions have already occurred — data is on wrong profile.
(C) Child swapped device — same child, new device, parent needs to re-authorise.
Key tone: Calm, non-accusatory. "It looks like…" framing. Never say "you made a mistake".
Data safety guarantee: Always show "progress data is never deleted" message.
COPPA note: No child data is shown in recovery screens except avatar + first name (already confirmed by parent).
