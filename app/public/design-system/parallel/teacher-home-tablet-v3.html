Breakpoints: Tablet layout at 768–1024px viewport width.
Navigation: Top nav bar replaces desktop left sidebar. Collapsed 56px icon rail used at borderline widths (≈768px portrait).

Layout differences vs Desktop

ElementDesktopTablet
Navigation240px left sidebar (always visible)Top nav bar, 52px tall
Stat row5-card inline flex, no overflowHorizontal scroll row (overflow-x: auto)
Student rosterFull sortable tableCard grid (3 columns) or compact list
Content grid3-col dashboard layout2-col grid
Collapsed railNot applicable56px icon-only sidebar (portrait mode fallback)

Student Grid Card

Tablet replaces the desktop sortable table with a 3-column card grid. Each card shows: first name only (FERPA), band chip, status dot (green = active today, grey = not played today, amber = in support queue), and stars / sessions / streak. Cards with queue items have a coloured border (amber = attention, blue = positive/ceiling).

Privacy Rules (same as desktop)

DataShown to teacher
Student namesFirst name only — no surname (FERPA)
Accuracy %YES (with context) — skill-category pattern level only, not per-question
Wrong answersPattern-level only — never specific question text or answer given
Parent contact infoNEVER shown in teacher portal
