DB Schema

feedback_items {
  id              UUID PRIMARY KEY,
  source          ENUM('parent','teacher','beta_tester'),
  raw_text        TEXT NOT NULL,
  classification  ENUM('bug','feature_request','ux_suggestion',
                       'observation','duplicate'),
  severity        ENUM('critical','high','medium','low'),
  status          ENUM('new','classified','routed','resolved'),
  classified_by   TEXT,
  classified_at   TIMESTAMPTZ,
  routed_to       TEXT,
  created_at      TIMESTAMPTZ DEFAULT NOW()
}

Classification Rules

B
Bug — reproducible error, crash, or incorrect system behaviour
F
Feature Request — new capability not currently in the product
U
UX Suggestion — existing feature works but friction or confusion reported
O
Observation — general comment, no specific action implied
D
Duplicate — matches an existing open item (link to original required)
