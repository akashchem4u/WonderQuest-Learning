Spec — Beta Cohort Data Model & Feature Flags
Engineering reference for beta-tester-home-desktop-v2.html
Database Schema SQL 
-- ── users table (beta flag columns) ──────────────────────────
CREATE TABLE users (
  id               UUID          PRIMARY KEY DEFAULT gen_random_uuid(),
  email            TEXT          NOT NULL UNIQUE,
  role             TEXT,         -- 'teacher' | 'parent' | 'owner_internal'

  -- Beta flags
  is_beta          BOOLEAN       DEFAULT false,
  beta_cohort      TEXT,         -- ENUM: 'teacher_pilot' | 'family_pilot' | 'owner_internal' | NULL
  beta_week        INT,          -- current week number (1–6), set server-side
  beta_cohort_id   UUID          REFERENCES beta_cohorts(id)
);

-- ── beta_tasks ────────────────────────────────────────────────
CREATE TABLE beta_tasks (
  id                   UUID  PRIMARY KEY DEFAULT gen_random_uuid(),
  cohort_type          TEXT,          -- 'teacher_pilot' | 'family_pilot' | 'all'
  week_number          INT,           -- 1–6
  title                TEXT NOT NULL,
  description          TEXT,
  completion_trigger   TEXT,          -- e.g. 'session_count>=3' | 'manual' | 'screen_visit:student_detail'
  unlock_condition     TEXT           -- e.g. 'day_of_week=Friday' | NULL (immediately available)
);

-- ── beta_task_completions ─────────────────────────────────────
CREATE TABLE beta_task_completions (
  user_id       UUID          REFERENCES users(id)       ON DELETE CASCADE,
  task_id       UUID          REFERENCES beta_tasks(id)  ON DELETE CASCADE,
  completed_at  TIMESTAMPTZ   DEFAULT now(),
  PRIMARY KEY (user_id, task_id)
);

-- ── beta_journal_entries ──────────────────────────────────────
CREATE TABLE beta_journal_entries (
  id          UUID          PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id     UUID          REFERENCES users(id) ON DELETE CASCADE,
  body        TEXT          NOT NULL,
  created_at  TIMESTAMPTZ   DEFAULT now()
);

-- ── beta_known_issues ─────────────────────────────────────────
CREATE TABLE beta_known_issues (
  id          UUID  PRIMARY KEY DEFAULT gen_random_uuid(),
  title       TEXT  NOT NULL,
  status      TEXT,  -- 'acknowledged' | 'in_progress' | 'investigating' | 'resolved'
  tracking_id TEXT,  -- e.g. '#KI-041'
  created_at  TIMESTAMPTZ  DEFAULT now(),
  resolved_at TIMESTAMPTZ
);

-- ── beta_cohorts ──────────────────────────────────────────────
CREATE TABLE beta_cohorts (
  id            UUID  PRIMARY KEY DEFAULT gen_random_uuid(),
  name          TEXT,  -- e.g. 'Cohort A' | 'Cohort B'
  cohort_type   TEXT,  -- 'teacher_pilot' | 'family_pilot'
  start_date    DATE,
  end_date      DATE
);
Beta Routing Rules Logic
Show beta home: user.is_beta = true AND current date is within beta active period (between beta_cohorts.start_date and beta_cohorts.end_date) → render beta-tester-home-desktop-v2 instead of standard home
After GA launch: is_beta flag is retained for analytics attribution. Beta home is hidden; user sees the standard production home regardless of flag.
Beta banner behavior: Non-dismissible for beta_cohort = 'teacher_pilot'. Dismissible once per week for beta_cohort = 'family_pilot' (dismiss stored per user, reset on beta_week increment).
Known issues source: Displayed issues are sourced from beta_known_issues table — NOT pulled from the user feedback queue.
Task unlock: Tasks with unlock_condition are hidden until condition resolves. beta_week is set server-side; never derived client-side.
Cohort Types & Display Rules Reference
beta_cohort value Home shown Banner dismissible Known Issues shown Journal shown
teacher_pilot Teacher/Staff Beta Home (Tab 1) No — non-dismissible Yes (top 3, compact) No
family_pilot Family Beta Home (Tab 2) Yes — once per beta_week No (not shown to families) Yes — beta_journal_entries
owner_internal Teacher/Staff Beta Home (Tab 1) + internal debug overlay Yes — always dismissible All issues (no limit) No
NULL Standard production home (no beta home) — No No
Frontend Feature Flags Flags 
// Pseudo-code: client-side routing guard
if (user.is_beta === true && isBetaActive(user.beta_cohort_id)) {
  const view = user.beta_cohort === 'family_pilot'
    ? 'beta-home-family'
    : 'beta-home-teacher';
  renderView(view);
} else {
  renderView('standard-home');
}

// Beta active check uses server-provided context — NOT client clock
function isBetaActive(cohortId) {
  return serverContext.beta_active === true;
}

// beta_week is always server-authoritative
// Never compute from client Date()
Post-GA Behavior GA
After GA: is_beta remains true in the database for cohort analytics. The beta home is suppressed by the routing guard. Beta testers see the standard home with no visual difference from non-beta users.
Beta journal entries and task completions are retained for product research. They are not exposed in the standard production UI.
beta_cohort value is preserved for segmentation in analytics dashboards (cohort performance, retention, NPS by cohort).
