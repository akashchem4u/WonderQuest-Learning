Dismiss Storage
Column Type Notes
user_iduuidFK → users
maintenance_iduuidFK → maintenance_windows
dismissed_attimestamptzWhen user dismissed
remind_attimestamptz | null"Remind me 24h before" = start_at − INTERVAL '24 hours'
Table: status_dismissals. When remind_at is reached, banner is re-shown regardless of previous dismiss. Session-only dismissals are stored in sessionStorage, not persisted to DB.
Non-Dismissible Conditions
  • Less than 1 hour to start_at — dismiss button not rendered in DOM
  • NOW() BETWEEN start_at AND end_at — maintenance in progress; no dismiss
  • Child persona — always non-dismissible regardless of timing; dismiss button never rendered
  • Banner height increases from 48px to 56px for non-dismissible <1h and in-progress states
Child Persona Rules
  • Always uses child-safe language — "WonderQuest is taking a short nap" not "undergoing maintenance"
  • "Your stars are safe!" shown on any system interruption — positive reassurance
  • No technical details, no time references, no links, no error tone
  • Non-dismissible at all times — keeps child informed without letting them hide it
  • During in-progress state: spinning 😴 icon, friendly red tint, positive framing

  • CSS: position: sticky; top: 56px; z-index: 150;
  • Below topbar, above all page content. Sticks when user scrolls down.
  • Full-width (100% of viewport). No max-width constraint on banner itself.
  • Height: 48px normal; 56px for non-dismissible and in-progress variants.
Maintenance Detection & Write Blocking
  • Maintenance mode entered when: NOW() BETWEEN start_at AND end_at
  • All write operations blocked server-side during window — clients receive a 503 Maintenance response
  • Reads served from cache (replica) during window — no stale data shown to users
  • Client polls maintenance_windows every 5 minutes; state machine drives banner variant automatically
  • Public status page updated automatically from maintenance_windows table; no manual update required
  • Write block includes: progress saves, badge awards, voice preference changes, PIN updates
  • Cached reads include: child progress display, quest state, reward balances (read-only)
