Release gate — automated checklist blocking or allowing a new product version launch.
Gate categories (5): Quality (test coverage, P0 bugs) | Performance (p95 latency, Lighthouse scores) | Content (question bank validation, COPPA scan) | Privacy (FERPA audit, data residency) | Stakeholder sign-off (founder + lead eng + curriculum).
Overall score: Weighted 0–100. Score ≥ 90 = gate open. < 90 = blocked.
Blockers: Any failed category item = hard blocker. Warnings = soft warnings, don't block gate alone.
Launch button: Disabled if any blocker. Enabled only when score ≥ 90 and all blockers cleared. Requires founder confirmation click.
Deployment: Launch button triggers CI/CD pipeline. Not a manual upload.

Gate Check Triggers
