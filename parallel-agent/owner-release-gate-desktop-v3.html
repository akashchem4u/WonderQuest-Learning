Tab 5 — Spec Routes, readiness formula, sign-off rules, launch gate logic, design tokens.
API Routes
MethodRoutePurpose
GET/owner/release-readinessFull readiness object: score, category breakdown, blockers, sign-off statuses
POST/owner/sign-offSubmit sign-off with optional note
PUT/owner/blocker/:idUpdate blocker state (assign / mark-fixed / escalate)
Readiness Formula
P0 Feedback Resolved = 25 pts · Routes Healthy = 25 pts · Content Coverage = 20 pts · QA Pass Rate = 20 pts · Sign-offs = 10 pts
Launch gate: Score ≥ 90 AND all blockers resolved AND all 5 sign-offs complete
Launch button is always disabled until all 3 conditions are met simultaneously
Sign-off Rules
Required signers: Engineering Lead / QA Lead / Content Lead / Product Owner / Legal
Sign-offs are append-only — cannot be revoked once submitted
If a new blocker is created after sign-off, the affected signer must re-sign
