Data model

learner_preference_profiles
Resolved profile row per child with current favorite tags, comfort filters, theme ranking, and voice hints.
preference_signal_events
Append-only source events from favorites, dislikes, theme switches, voice locks, and return-journey edits.
preference_profile_overrides
Parent pins and locks that supersede derived outputs.

Rules

Show only stable, explainable fields. Do not surface hidden scores, latent traits, or "predicted personality" labels. If a field is derived, explain it in parent language.

Update cadence

Explicit parent edits update immediately. Derived rankings can refresh asynchronously, but the profile should show when it was last recomputed.

Visibility

This profile is parent-facing. Teachers and owners may only see aggregated, anonymized pack preference trends for analytics or content planning.
