ENG NOTE — parent-quiet-hours-settings-v2.html
Parent can set "quiet hours" — time periods when WonderQuest will NOT send notifications to the parent's device AND will NOT allow the child to start new sessions on their device.
Two separate toggles:
(1) Quiet hours for notifications (parent device only) — e.g. 9pm–7am no notification pings.
(2) Quiet hours for play (child's device) — e.g. 8pm–7am child cannot start a new session (session in progress is allowed to complete).
COPPA note: Quiet hours for play apply to the child's WonderQuest session start permission only — NOT a device-level parental control. Child can still use other apps.
Defaults: Notifications quiet 9pm–7am (ON). Play quiet hours OFF (family's choice).

Session-in-Progress Rule

If quiet hours begin while a session is in progress: allow the session to complete naturally. Do NOT force-quit. Only block NEW session starts.

Child-facing Blocked Screen Tone

Child screen uses dark WonderQuest theme (not parent cream). Language: "WonderQuest is resting" / "Bedtime mode" / countdown to resume time. Never "you're blocked" or "your parent said no". Stars and badges always shown as safe.

COPPA Note

This is a WonderQuest-level restriction only — it does NOT lock the device or prevent use of other apps. Engineering must never market this as a parental control or device-level screen-time solution. It is a session scheduling feature only.
