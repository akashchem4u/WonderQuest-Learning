Spec · Accessibility Settings v2

Technical specification for the parent accessibility settings system

DB Schema
child_accessibility_prefs (
  child_id               UUID          PRIMARY KEY,
  display_mode           ENUM('dark','light','high_contrast') DEFAULT 'dark',
  text_size              ENUM('small','medium','large','xl')     DEFAULT 'medium',
  bold_text              BOOLEAN       DEFAULT false,
  reduced_motion         BOOLEAN       DEFAULT false,
  reduced_transparency   BOOLEAN       DEFAULT false,
  readout_enabled        BOOLEAN       DEFAULT true,
  music_enabled          BOOLEAN       DEFAULT true,
  sfx_enabled            BOOLEAN       DEFAULT true,
  celebration_sounds     BOOLEAN       DEFAULT true,
  voice_speed            ENUM('normal','slow')             DEFAULT 'normal',
  voice_speed_locked     BOOLEAN       DEFAULT false,
  large_buttons          BOOLEAN       DEFAULT false,
  high_contrast_buttons  BOOLEAN       DEFAULT false,
  show_question_number   BOOLEAN       DEFAULT false,
  reading_highlight      BOOLEAN       DEFAULT false,
  simplified_vocab       BOOLEAN       DEFAULT false,
  extra_time             BOOLEAN       DEFAULT false,
  updated_by_parent_id   UUID,
  updated_at             TIMESTAMPTZ
)
Apply Timing & Sync
  • Settings are applied on the next session start, not mid-session. Parent sees: "Changes will apply next time [child] plays"
  • Settings are synced across devices via server — not localStorage-only. All devices reflect the latest saved state within seconds.
  • If a child is currently in an active session when a parent saves, the new settings queue for the session that begins after the current one ends.

COPPA / FERPA Note
  • Accessibility preferences are functional data, not behavioral or performance data.
  • These prefs are not shared with teachers and are not surfaced in any teacher-facing dashboard.
  • Accessibility prefs are not used for analytics, personalization modeling, or advertising.
  • Stored under the parent's account scope; subject to standard COPPA deletion rights.
Accessibility preferences exist solely to make the product more usable for the child. They carry no inferential or commercial value and are treated as purely operational data.
