Component: owner-release-calendar-v2.html
Monthly release calendar. Grid + upcoming list. Event types: Release / Hotfix / Code Freeze / Curriculum Review / Post-Mortem.

Auto-Logged Events

Release and hotfix events are auto-logged to the calendar when the CI/CD pipeline completes a successful production deploy. Timestamp = deploy completion time. Post-mortem events are auto-created 1 day after P0 incident resolution (default: 3 days later, editable).

Manual Events

Code Freeze, Feature Freeze, Gate Review, and Curriculum Review deadlines are set manually by the owner (or engineering lead). These appear as amber or violet events and can be edited. Calendar sends reminder notifications to the owner 2 days before each manual event.

Release Windows

Recommended release windows: Tuesday–Thursday (not Monday or Friday). Hotfixes can be deployed any day. A warning is shown if a planned release is scheduled for a weekend or public holiday. School term blackout periods can be configured (no releases during first/last week of term).
