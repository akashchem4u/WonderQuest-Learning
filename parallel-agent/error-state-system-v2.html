error-state-system-v2.html — Error state components. 5 error categories: (1) Network Error (no connection / timeout), (2) Server Error (500/503 — our fault), (3) Not Found (404 — resource doesn't exist), (4) Permission Error (403 — user can't access), (5) Data Error (loaded but malformed/unexpected). Each has adult and child-safe variant. Child errors: never show technical details; always use friendly language + emoji. Adult errors: show error code + retry option + escalation path. Never blame the user.

ErrorStateProps — Component Interface

interface ErrorStateProps {
  /** One of 5 error categories */
  category:  'network' | 'server' | 'not_found' | 'permission' | 'data';

  /** Persona determines copy depth: child hides codes; owner sees all */
  persona:   'child' | 'teacher' | 'parent' | 'owner' | 'content';

  /** UUID-style — shown to teacher / owner only, for support reference */
  errorId?:  string;

  /** 404, 403, 500, 503, etc — rendered only to teacher / owner */
  httpStatus?: number;

  /** Shows [Try Again] CTA if true */
  retryable: boolean;

  onRetry?:  () => void;
  onBack?:   () => void;

  /** Controls layout/sizing */
  size:      'full_page' | 'full_card' | 'inline' | 'toast';
}

Error Taxonomy

Category HTTP codes Retryable Show Error ID Notes
network — Yes No Offline / timeout. Show progress-saved badge. Offer offline save if applicable.
server 500, 502, 503, 504 Yes Yes (adult) Always say team notified. Never blame user. Reload + Contact Support.
not_found 404 No No No error code in UI. Offer go-home / browse / support links.
permission 401, 403 No No Use "don't have access to this" — never "not allowed". Go Back + Request Access.
data 200 (bad payload) Yes Yes (adult) Data loaded but malformed. Refresh + Report Problem. Team should be alerted.

Error Message Copy Rules

🚫
Never blame the user. "You did something wrong" "Something went wrong on our end"
🚫
Never say "not allowed" for permission errors. "You're not allowed here" "You don't have access to this"
🚫
Never expose raw technical info to child or parent personas. No stack traces, no HTTP codes, no endpoint paths.
✅
Always include a way out. Every error state must offer go back, go home, retry, or contact support.
✅
Server errors: always say the team has been notified, even if the alert is async / background.
✅
Error IDs are UUID-style, shown only to teacher and owner personas, used for support reference.
✅
Child errors: use warm, playful language. Include stars-are-safe reassurance whenever data or progress may be at risk.
✅
No animations on error states — errors should be calm and still, not blinking or pulsing, to avoid anxiety.

Size / Layout Variants

size value Use case Container
full_page Route-level error, entire page failed to load Centered, max-width 560px, full viewport height
full_card Dashboard card or section failed Fills the card container, min-height 200px
inline Form field, table row, grid item Inline with surrounding content, no height change
toast Non-blocking notification, auto-dismissible Fixed bottom-center, z-index above all content
