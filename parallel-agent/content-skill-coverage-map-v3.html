Component: content-skill-coverage-map-v3.html
Curriculum coverage heatmap. Band (P0–P3) rows × Subject columns × Topic cells. 5 coverage levels (Full/Good/Partial/Thin/Gap). P3 G4–5 is historically underserved — v2.6 target. Gap analysis tab with prioritised gap list.

Coverage Calculation

Coverage per topic cell = COUNT(skills WHERE band=B AND topic=T AND review_status IN ('Published', 'Draft')). Levels: Full ≥ 8, Good 5–7, Partial 3–4, Thin 1–2, Gap 0. Draft skills count towards coverage to show in-progress work. Blocked skills are excluded from coverage count (they are not deliverable).

P3 G4–5 Priority

P3 G4–5 has the worst coverage (38 skills vs target 60+). This directly contributes to the P3 D30 retention problem (52% vs target 65%) — students exhaust available content. The v2.6 "Band Expansion" batch import targets 11 new P3 Maths skills. Gap analysis tab shows the remaining 16 topics with zero coverage, prioritised by curriculum alignment and teacher request data from feedback items.

Commission Action

Clicking "Commission" on a gap topic creates a content_commissions record with: topic, band, subject, target skill count, due date, assigned curriculum writer. Commissions appear in the curriculum team's sprint board. Once skills are authored and imported, the coverage cell updates automatically.

New Draft Badge

When Draft skills exist for a topic (from a recent import), the cell shows a green "+N new" badge with the draft count. This helps curriculum team see in-progress work without counting it as full coverage. The cell colour reflects published skills only for the coverage assessment; the badge shows the pipeline.
