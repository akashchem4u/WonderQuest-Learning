Database Schema
-- student_groups
CREATE TABLE student_groups (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  class_id    UUID NOT NULL REFERENCES classes(id),
  teacher_id  UUID NOT NULL REFERENCES teachers(id),
  group_name  TEXT NOT NULL,
  student_ids UUID[] NOT NULL DEFAULT '{}',
  group_color TEXT NOT NULL DEFAULT 'teal',
  created_at  TIMESTAMPTZ DEFAULT now()
);

-- group_assignments
CREATE TABLE group_assignments (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  group_id      UUID NOT NULL REFERENCES student_groups(id) ON DELETE CASCADE,
  assignment_id UUID NOT NULL REFERENCES assignments(id),
  assigned_at   TIMESTAMPTZ DEFAULT now()
);
Privacy Rules
NamesStudent first names only — no surnames displayed anywhere in this component
AccuracyNo accuracy percentages or score data exposed — engagement metrics only (started / session count / duration)
Group visibilityGroup composition is teacher-only. Parent portal shows individual child data exclusively — no group membership, no peer names
MessagingEncouragement messages route through the WonderQuest platform — no direct student contact; no personal contact details shared
FERPAGroup data is school-linked; governed under the institution's FERPA agreement. Export or sharing requires district-level consent
FramingStrictly positive language — "not started yet" not "struggling"; no deficit labels in any UI string
Business Rules
  • 1A teacher may only view and manage groups belonging to their own class_id. Cross-class group access is forbidden at the API layer.
  • 2A student may belong to at most one group per class at a time. Adding a student already in a group in the same class must prompt the teacher to confirm a move.
  • 3Group assignments cascade: deleting a group deletes all group_assignments rows for that group (ON DELETE CASCADE). Individual student progress records are never deleted.
  • 4The engagement tab surfaces session-level data (started, count, avg duration) only. Score, accuracy, or item-level data must never appear in this component.
  • 5Encouragement messages are queued asynchronously through the platform notification system. No real-time contact is established; message content is logged for safeguarding review.
  • 6Group color is cosmetic metadata only — it must never be used to infer ability grouping. UI copy and documentation must not describe colors as "ability" or "level" indicators.
