Spec — Data Model & Reporting Rules OWNER ONLY
🔒 is_owner=true gate
Reporting Data Sources
Table / Source Description Refresh
beta_readout_cache Pre-aggregated stats for this dashboard. Primary read source. Cohort-level only — no individual rows. Every 6 hours
play_sessions Source for session counts, avg duration, band engagement, retention calculation. Live (aggregated)
beta_issues Bug reports and issue tracking. Used for resolved/total counts and open issue breakdown per cohort. Live (aggregated)
feedback All beta feedback items. Categorized by type: bug, suggestion, feature_rating, observation. Live (aggregated)
beta_observations Structured observations logged by teachers and moderators during sessions. Live (aggregated)
beta_task_completions Skill start and completion events. Used to calculate skill completion rate. Live (aggregated)
All readout queries operate on cohort-level aggregates only. No individual teacher, parent, or child data is returned or displayed in this dashboard. Cohort IDs are the lowest granularity exposed.
Access Control
Rule Detail
is_owner gate Route and API protected by is_owner=true flag on user record. Distinct from teacher_analytics and parent_analytics roles.
No teacher/family data Owner readout cannot be used to identify individual tester behavior. Cohort minimum size: 3. Cohorts below 3 active testers are suppressed from display.
Audit log All readout page views and CSV exports are written to owner_access_log with timestamp and user_id.
Beta Status & Decision Log
-- Pilot status enum beta_status = 'on_track' | 'at_risk' | 'extended' | 'paused' | 'complete' -- Decision log table decision_log( id UUID PRIMARY KEY, week INT NOT NULL, -- beta week number (1–6) status beta_status NOT NULL, decision TEXT NOT NULL, -- e.g. "continue", "pause", "extend" rationale TEXT, owner_id UUID REFERENCES users(id), created_at TIMESTAMPTZ DEFAULT now() )
A decision_log entry should be created at the end of each beta week review. The status field drives the decision panel state shown in the Pilot Overview tab. Current week 3 status: at_risk (Family Cohort B/C low activity).
Reporting Rules
Metric Calculation Target
Retention rate Active testers in week N / active testers in week N-1 >= 85%
Skill completion rate Skills completed / skills started (cohort-level sum) >= 80%
Avg session length Sum(session_duration) / count(sessions) across all cohorts ~20 min
Sessions/family/week Total family sessions / (active families × weeks elapsed) >= 3.0
Issue resolution rate Resolved issues / total issues filed during beta >= 70%
Pilot health verdict Composite: retention + engagement + resolution rate. Green ≥80, Amber 60–79, Red <60 Green
