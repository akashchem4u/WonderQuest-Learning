Component overview
Item 243 — parent-grade-level-comparison-card-v2.html
Grade-level cohort comparison card for the parent dashboard. Parallels the age comparison (item 242), but uses grade_level as the cohort key instead of age_years. Parent-only; child never sees this component.
Cohort definition
The comparison cohort is children who share both the same grade_level AND the same band_id as Emma. This ensures apple-to-apple comparisons — a Grade 3 child on Band P0 is not compared with a Grade 3 child on Band P3.
Note: Same-grade cohorts may include children aged 7–9 because grade assignment varies. This is a feature, not a bug — the grade view captures curriculum-level peers, not age peers.
Privacy / cohort size gate
If cohort_size < 50 for this grade + band combination, the card shows the "Not enough data yet — check back next month" state instead of comparison data. No partial data is shown.
Database schema
child_grade_comparisons ( child_id UUID NOT NULL REFERENCES children(id), grade_level SMALLINT NOT NULL, -- e.g. 3 subject_id UUID NOT NULL REFERENCES subjects(id), comparison_bucket ENUM( 'earlier', 'similar', 'later' ) NOT NULL, cohort_size INT NOT NULL, -- must be >= 50 updated_at TIMESTAMPTZ NOT NULL DEFAULT now(), PRIMARY KEY (child_id, grade_level, subject_id) );
Parallels child_age_comparisons — same schema shape, different cohort key.
Disclaimer acknowledgment
The acknowledgment checkbox writes to parent_preferences.comparison_disclaimer_accepted_at. This single flag covers both the age comparison view (item 242) and this grade comparison view. If already acknowledged, the checkbox renders pre-checked and the body text is shown in muted confirmed style.
-- Check: has parent accepted disclaimer for either comparison view? SELECT comparison_disclaimer_accepted_at FROM parent_preferences WHERE parent_id = :parent_id;
Grade linkage
Emma's grade is stored in children.grade_level. This is set during onboarding and can be updated by the parent in child settings. The cohort query filters on grade_level = children.grade_level AND band_id = children.band_id.
Comparison bucket logic
Bucket Display label Bar position Pill colour
earlier Exploring earlier content Emma marker left of cohort median #ff7b6b
similar Similar pace Emma marker within cohort spread #50e890
later Exploring a bit ahead Emma marker right of cohort median #a78bfa
Note: "later" bucket is always framed positively ("exploring ahead") — never "advanced" or "above grade level".
Privacy & framing rules
🔒 Child never sees this card — parent-only route.
🔒 Cohort size never shown to parent — only aggregated bucket is shown.
🔒 No accuracy %, wrong-answer counts, or mastery scores shown in comparison.
✏️ All framing is exploration-based, positive. "Just starting" not "behind". "Exploring a bit ahead" not "advanced".
✏️ Disclaimer required at first view; shared acknowledgment flag with age comparison card.
📊 Grade comparison covers Grade 3 activities only — children exploring above-grade content are compared only on Grade 3 activity data.
