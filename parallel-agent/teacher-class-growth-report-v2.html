Data Scope

  • Class-level aggregates (band distribution, subject engagement) shown in Tab 1
  • Per-student band and engagement data for teacher of record only — not shared with other teachers
  • All DB queries aggregated at class level for display; per-student rows accessed only for the authenticated teacher's own class_id

Privacy

  • First name only in teacher view; surnames not stored in class_memberships display layer
  • Progress bars = relative visual representation — no numerical value shown to teacher or parent
  • No per-question accuracy data exposed anywhere in this view
  • "Needs Attention" flag identifies band groups, not individuals by name

"Needs Attention" Flag Logic

  • Flagged when engagement_level = 'exploring' for ≥ 2 consecutive terms, OR 0 sessions in last 14 days
  • Flag is automated — generated by system, surfaced to teacher in Subject Engagement table
  • Triggers Interventions suggestion in the Interventions tab (separate panel)

Band Advancement

  • System generates suggestion; teacher confirmation required before child advances
  • Confirmed advancement query:
    UPDATE children SET band_id = next_band WHERE teacher_confirmed = true
  • Band regression is never automated — teacher-initiated only

Export Rules

  • CSV includes: first_name, band, engagement_level — no accuracy or question data
  • PDF = formatted report with benchmark disclaimer appended
  • Both exports exclude student surnames
🛡 FERPA Compliance
  • Data accessible only to teacher of record for the class
  • School admin: read-only access
  • No data shared with third parties without parent consent
  • Class membership display layer stores first name only — no PII exposed through API response
