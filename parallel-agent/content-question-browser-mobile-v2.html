Component: content-question-browser-mobile-v2.html
Mobile question browser. List → tap → full-screen detail sheet. Wrong-answer-as-correct highlighted red. Mobile edit = answer key correction only (mark/unmark correct). Full edit on desktop.

Mobile Navigation Pattern

Mobile uses a list → detail sheet pattern. Tapping a question item opens a full-screen detail view (no two-pane split). Back arrow returns to list. Detail sheet is a modal-style overlay pushed from the right (iOS-style navigation). On Android, back gesture also returns to list.

Mobile Edit Constraints

The "Edit Answer Key" action on mobile opens a simplified answer-key-only editor: shows the 4 answer options, allows toggling is_correct on each option. This is the most critical mobile edit case (fixing wrong-answer-as-correct). All other edits (question text, images, audio, skill metadata) require desktop. The mobile edit saves to the same question_answers table as desktop.

Fix Reference Box

The SQL fix reference shown in the detail sheet is a read-only hint for engineers reviewing on mobile. The actual fix can be performed either through the mobile edit flow (answer key toggle) or the desktop editor. A note clarifies that full edits require desktop to avoid confusion about what's editable on mobile.
