Strip Dimensions

  • Segments: always 5 (one per session question)
  • Height: 6px
  • Border-radius: 3px per seg
  • Gap between segs: 4px
  • Container: display: flex; gap: 4px; flex: 1
  • Topbar padding: 16–20px per side

Fill Logic

  • Seg fills when child submits answer (correct OR wrong)
  • Fill = star-safe: no red segments — only filled/empty
  • Current question: pulsing animation (not filled yet)
  • Fill transition: scaleX(0→1) from left, 0.3s
  • ❌ Never show wrong/red segments on progress strip
  • ❌ Never decrement a filled segment

Current Segment Pulse

  • Animation: seg-pulse — opacity-based fade between 40% and 88% tint
  • Duration: 1.2s, ease-in-out, infinite
  • Shows on active question's segment only
  • Stops when answer submitted (segment fills immediately)
  • Each band has own pulse keyframe with band color

Boss Node

  • Last seg: position: relative; overflow: visible
  • ⭐ pip: ::after pseudo-element, centered via position: absolute
  • Boss pip visible BEFORE boss is active (teaser)
  • Boss active: pulsing gold animation
  • Boss complete: seg fills #ffd166 regardless of band
  • Boss frequency: ~20% of sessions

Star Badge

  • Position: right of strip, same topbar row
  • Format: ⭐ {count}
  • star-bump animation on increment (scale 1→1.3→1, 0.4s spring)
  • Border/text color matches band accent (not white)
  • ❌ Never show star count decreasing
  • ❌ Never show accuracy % here (stars only)

Band Color Tokens

  • Pre-K: filled #ffd166, badge #ffe08a text
  • K-1: filled #9b72ff, badge #c0a8ff text
  • G2-3: filled #58e8c1, badge #7af0d0 text
  • G4-5: filled #ff7b6b, badge #ffa090 text
  • Boss override: always #ffd166
  • All bands: unfilled seg #2a2050
