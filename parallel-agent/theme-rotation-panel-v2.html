Priority order

Rank Decision rule
1. Parent pinAlways wins until removed. Overrides all automatic rules.
2. Suppression listRespect disliked visuals and temporary snoozes. Never rotate into a muted pack.
3. Interest matchFavor themes linked to stated interests over inferred novelty. Transparent reason string required.
4. CooldownAvoid repeats inside the cooldown window. Only bypassed by explicit pin.

Do not do this

No mid-session swaps Keep the visual environment stable once a child launches play.
No pressure language Do not market a new theme as time-limited unless it is a true seasonal event surfaced to parents first.
No hidden experiments Rotation reasons must be inspectable; no silent A/B theme tests in child-facing surfaces.

Data model

child_theme_rotation_rules
Stores cadence, pin state, suppressed packs, and last rotation timestamp per child.
theme_rotation_events
Append-only log with old pack, new pack, reason, and actor.
theme_rotation_blackouts
Optional blackout windows for seasonal exams, travel, or parent-requested stability.

Session boundary rule

Theme changes are committed only when no session is active and no celebration modal is visible. If a queued swap misses that window, it rolls to the next launch.

Transparency requirement

Every recommendation needs a visible reason string such as "Because Emma likes animals and stories." Parents can inspect, override, or disable automation at any time.

Accessibility note

Rotation must preserve the child's chosen display mode, motion preference, and voice settings. Theme visuals change; accessibility settings do not.
