· No "You got X/Y right"
· No timer language

Component Props — SuccessStateProps

interface SuccessStateProps {
  // Which success pattern to render
  category:    'action_confirmed' | 'process_complete' | 'achievement' | 'milestone'

  // Who is seeing this success state
  persona:     'child' | 'teacher' | 'parent' | 'owner' | 'content'

  title:       string                          // e.g. "Changes saved" / "Amazing!"
  body?:       string                          // Optional sub-copy
  stats?:      Array<{ label: string; value: string | number }>

  autoDismiss?: number                        // ms; null = persistent. adult default: 3000
  cta?:         { label: string; action: string }  // Primary CTA

  // Child-only: confetti/stars. Always false for adult personas
  animated:    boolean

  // Required when persona === 'child'
  band?:       'p0' | 'p1' | 'p2' | 'p3'
}

Category Quick Reference

action_confirmed

Save / approve / submit / publish / resolve. Brief toast with green left-border. Auto-dismisses (3 s default). No animation, no confetti. Adult personas only.

process_complete

Multi-step operation finished (bulk import, export, sync). Full-card with summary stats. Persistent — user must dismiss. Adult personas.

achievement

Child-facing. Celebratory full-screen overlay. Confetti (significant milestones only, not every answer). Animated stars / badge reveal. Band-themed colors. Always includes XP amount. Never includes accuracy %.

milestone

Teacher / parent-facing. Positive but calm — green tint, clean layout. No confetti. Full-card, persistent. Examples: bundle published, release gate passed, class goal reached.

Copy Guidelines

Context Do write Do not write
Adult action confirmed "Saved" · "Approved" · "Published" "Amazing job saving!" · "Great work!"
Adult process complete Factual summary stats with counts "You crushed it! 🎉"
Child achievement "Amazing!" · "Awesome!" · "+15 XP ✨" "You got 8/10 correct" · "80% accuracy"
Child achievement "3 in a row! 🔥" · "New badge earned!" Any timer or countdown language
Level-up chip Show only when XP threshold crossed Show on every question answer
Confetti trigger Badge earned · streak milestone · level-up Every correct answer

Auto-Dismiss Defaults

  • ⏱
    action_confirmed (adult): 3000 ms default. Override via autoDismiss prop (e.g., 2000 for "Question saved", 5000 for "Audio uploaded").
  • 📌
    process_complete: autoDismiss: null — persistent. User must interact with the CTA to dismiss.
  • 🎯
    achievement (child): Persistent overlay — child taps [Keep Going!] / [Next Challenge!] etc. Never auto-dismiss.
  • 📌
    milestone (adult): Persistent card until the user navigates away or dismisses. No auto-dismiss.

Animated Prop Rules

  • ✅
    animated: true is only valid when persona === 'child'. Rendering engine enforces this — adult personas always render animated: false regardless of prop value.
  • ✅
    Confetti CSS animation triggered only for category === 'achievement' AND a significant milestone trigger (badge earned, level-up, streak milestone). Not fired on routine correct answers.
  • ✅
    Star-pop @keyframes used for earned stars on P0 band only. Badge-reveal blur animation for P2 badge unlock. XP bar fill for P3 level-up.
  • 🚫
    No JS canvas, no third-party animation libraries. All motion via CSS @keyframes only.
