beta-observation-log-panel-v2.html — Beta observation logging tool. Teacher beta testers observe children using WonderQuest and log structured notes. Distinct from feedback (opinions) — observations are factual notes about user behaviour. Stored in: beta_observations(id, observer_id, session_date, band, subject, observer_type ENUM teacher/researcher, observation_type ENUM behaviour/engagement/difficulty/confusion/delight, body TEXT, tags TEXT[], created_at). Used by: owner in beta readout to identify UX patterns.

Database Schema
CREATE TABLE beta_observations ( id UUID PRIMARY KEY, observer_id UUID REFERENCES users(id), session_date DATE NOT NULL, session_time TIME nullable, band ENUM('p0','p1','p2','p3') nullable, subject TEXT nullable, observer_type ENUM('teacher','researcher','parent'), observation_type ENUM('behaviour','engagement','difficulty','confusion','delight'), body TEXT NOT NULL, tags TEXT[], impact ENUM('low','medium','high'), screenshot_url TEXT nullable, beta_week INT, created_at TIMESTAMPTZ DEFAULT NOW() );
Observation Type Taxonomy
Type Purpose Example
👀 behaviour Factual account of actions taken "Child tapped X 3 times"
💡 engagement Level of interest / attention "Class stayed focused for 18 min"
😤 difficulty Task / content seemed hard "Re-read question 4 times"
😕 confusion UI / UX confusion "Couldn't find the back button"
😄 delight Positive emotional reaction "Said 'this is so fun!'"
Privacy Rules
🔒 observer_id is the teacher / researcher (not anonymised — they own their observations)
🧒 Children observed are NOT named in the database. Body text may contain first names but that is teacher-controlled.
🗓️ After beta: observations are anonymised after 90 days (observer_id → NULL, body scrubbed of names)
📊 Used by: owner for UX pattern analysis in beta readout only
