Component: content-bulk-import-review-v3.html
Bulk import validation + review screen. Step indicator. Summary row. Error/warning/valid classification per row. Inline fix actions. Skills imported as Draft. Post-import success state.

Import Flow

Upload CSV → Parse (detect encoding, delimiters) → Validate (field types, required fields, duplicate check, band values, asset URL check) → Review (this screen, show errors/warnings) → Confirm (import valid rows) → Draft skills created in content_skills table (review_status=Draft, delivery_locked=true) → Curriculum review workflow → Owner publish (review_status=Published, delivery_locked=false).

Error vs Warning

Blocking errors: Import cannot proceed for that row. Must fix or skip. Examples: no correct answer set, duplicate QID, invalid band value, empty required field. Warnings: Import can proceed but issue is flagged. Examples: possible duplicate title, asset 404 (skill imports without image), question count below minimum, missing optional field (defaults applied). Warnings are tracked in import_warnings log for post-import review.

Draft Status

All imported skills start as Draft. delivery_locked=true prevents delivery to any student. Auto-flag thresholds do not apply to Draft skills (no sessions, no error reports). Draft skills appear in the content browser with a "Draft" badge. They require: (1) Curriculum team review and approval, (2) Owner publish action, before becoming Published (delivery_locked=false).

CSV Format

Required columns: skill_title, subject, band (P0/P1/P2/P3), question_text, answer_a, answer_b, answer_c, answer_d, correct_answers (comma-separated letters, e.g. "A,C"), question_type (MCQ/TrueFalse/Sort/Match). Optional: difficulty (Easy/Medium/Hard), asset_url, explanation_text, curriculum_standard_ref. Template downloadable from Content → Settings → Import Template.
