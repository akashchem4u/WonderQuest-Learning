Error Recovery Decision Tree

Error occurs
Child persona?
YES Absorb silently · positive message · never block play
NO Show appropriate error with recovery action
Network error?
YES Local-first · continue offline · sync when reconnected
NO Retry with exponential backoff: 1s→2s→4s→8s→30s max
Auth error?
YES Session expired flow → sign in again
NO Categorise error type
Server 5xx?
YES "Our end" framing + error ID + auto-notify team (PagerDuty if >5/min)
NO Client error · check user state · suggest action
Unhandled exception?
YES App crash recovery · reload / home options

Core Principles

🛡 Child play never blocked
⭐ Stars always safe
📶 Local-first architecture
🔄 Exponential backoff retry
🔒 Session expiry for adults only
Retry sequence 1s → 2s → 4s → 8s → 30s max; surface after 3 retries
Child error rule All child errors: absorbed silently or shown as friendly positive message. Red blocking screens: never.
Adult error rule Show error with clear recovery action. Frame server errors as "our end" to reduce user anxiety.
Error ID visibility Shown to teacher / parent / owner only. Never shown to child.

Error ID Format

Format ERR-YYYYMMDD-NNNNN

Child Play — Never Blocked

Rule Any error in child play path is either (a) absorbed silently and skipped, or (b) shown as a positive friendly message. Never a blocking red error screen.
Content failure After 2 retries: silently skip to next question · log to failed_deliveries for content team QA
Offline play ServiceWorker cache: 20 questions pre-cached per current skill + audio pre-cached · ~20 questions of offline play guaranteed

Retry Strategy

Backoff sequence 1s → 2s → 4s → 8s → 30s max
Surface to user after 3 retries
Error notification Server 5xx auto-fires safety_flag + PagerDuty if >5 errors/min on same route
