Database Schema

-- Table 1 CREATE TABLE assignments ( id UUID PRIMARY KEY, title TEXT NOT NULL, assignment_type TEXT, -- 'batch' | 'free_explore' quest_batch_id UUID, -- FK → quest_batches.id skill_area TEXT, due_date DATE, -- nullable teacher_note TEXT, -- shown verbatim; no other student names status TEXT -- 'active' | 'completed' | 'archived' ); -- Table 2 CREATE TABLE child_assignment_progress ( id UUID PRIMARY KEY, child_id UUID NOT NULL, -- FK → children.id assignment_id UUID NOT NULL, -- FK → assignments.id started BOOLEAN DEFAULT false, sessions_on_assignment INT DEFAULT 0, last_active_at TIMESTAMPTZ );

Privacy Rules

  • Parent query is scoped to child.parent_id = auth.uid() — no cross-family data exposure.
  • Child is identified by first name only. No last names, IDs, or class-level identifiers shown in UI.
  • No accuracy percentages, scores, correct/incorrect counts, or ranking information displayed anywhere.
  • Teacher notes are displayed verbatim. Backend must validate that notes do not reference other students' names before delivery.
  • No cross-child comparison UI. Each child's data is displayed independently; aggregate or comparison views are prohibited.
  • FERPA: All data access is restricted to the authenticated parent's own children. Server-side row-level security enforced.

Business Rules

  1. 1 No timers. The UI must not display countdown timers, elapsed time, or any time-pressure indicators.
  2. 2 No scores. Accuracy percentages, point totals, streaks with numeric values, or correct/incorrect tallies are prohibited in parent views.
  3. 3 Positive language only. All status messages, progress indicators, and CTAs must use encouraging, non-evaluative language. Avoid "failed," "missed," "behind," or comparative terms.
  4. 4 Platform-routed messages only. Teacher messages must pass through the WonderQuest platform message system. Direct email or unmoderated text is not permitted in this component.
  5. 5 First name only. Children are referenced by first name only throughout the parent-facing UI. No surnames, student IDs, or class-level identifiers.
