How WonderQuest uses this

Dislike filters lower the chance of certain themes or sensory treatments appearing. When a filter conflicts with a recommended pack, the system should find the closest calmer substitute instead of showing nothing.

Example If "loud sounds" is active, space and sports can still be available, but their calmer audio variants should be preferred.
Good fallback Replace a confetti-heavy celebration with a badge glow card or soft sparkle state.
Expiry option Parents can set a filter to auto-expire after 14 or 30 days if it was only temporary.

Suppression behavior

Home personalization
Do not suggest filtered packs in the top slot if a close alternative exists.
Theme rotation
Filtered packs are skipped unless parent-pinned.
Return journey
Comfort filters override novelty logic during comeback sessions.

Language rules

Do say "Show less of this" or "keep things calmer here."
Do not say "Disordered," "sensitive kid," or any pathologizing language.
Child route Never expose the filter label directly to the child.

Data model

dislike_filters
Per-child soft suppression rows with filter key, strength, expires_at, and updated_by.
dislike_filter_events
Append-only log for add, remove, and expire actions.
theme_filter_overrides
Parent pin exceptions that temporarily override a soft suppression.

Decision rules

Filters are soft by default. Only child-safety or accessibility settings should act as hard blockers. Parent pins can override a soft filter, but the UI should clearly show that the child may still see the filtered theme.

Defaults

If no filters exist, the system behaves normally. If many filters stack at once, prioritize current favorite theme and low-motion variants rather than collapsing to a generic UI.

Audit note

Teacher and owner views may see only aggregated filter counts by pack family. Individual child comfort filters remain parent-private.
