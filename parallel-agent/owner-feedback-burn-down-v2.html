Data source
Primary table: feedback. Analytics are computed on status change events logged in feedback_status_events(feedback_id, from_status, to_status, changed_at, changed_by). Snapshots are materialized weekly into feedback_burn_snapshots.
Burn-down snapshot schema
feedback_burn_snapshots(week_start DATE, total_open INT, p0_open INT, p1_open INT, p2_open INT, feature_open INT, resolved_this_week INT, submitted_this_week INT) — refreshed every Monday at 00:00 UTC.
SLA tracking
sla_met = resolved_at < created_at + SLA_interval; evaluates to NULL while the item is still open (excluded from SLA % denominator). SLA intervals: P0 = 4h, P1 = 24h, P2 = 72h, Feature = 2 sprints (14 days).
Burn-down projection
Linear regression on last 4 weeks of net_change (submitted − resolved). Current trajectory:
On track to clear P1 backlog in 2 weeks
Privacy policy
Feedback submitters are identified by school name or cohort label only — individual teacher names are never surfaced in this view. Owner sees school name in the B2B contract context. Family cohort items show "Family cohort" only.
Access control
This dashboard is restricted to the owner role. API route: GET /internal/owner/feedback-burn. Requires x-owner-token header; returns 403 for all other sessions.
Refresh cadence
Stat cards: live (10-second polling via /internal/owner/feedback-burn/live). Burn-down chart: weekly snapshot, does not update intra-week. SLA %: recalculated on each page load from resolved items in the rolling 90-day window.
