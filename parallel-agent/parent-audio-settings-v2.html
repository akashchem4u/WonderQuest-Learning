DB Schema

CREATE TABLE child_audio_prefs ( child_id UUID PRIMARY KEY, audio_master BOOLEAN DEFAULT true, readout_enabled BOOLEAN DEFAULT true, music_enabled BOOLEAN DEFAULT true, music_volume INT DEFAULT 50 CHECK (music_volume BETWEEN 0 AND 100), sfx_enabled BOOLEAN DEFAULT true, celebration_sounds BOOLEAN DEFAULT true, voice_speed ENUM('normal', 'slow') DEFAULT 'normal', voice_speed_locked BOOLEAN DEFAULT false, reading_highlight BOOLEAN DEFAULT false, voice_locale ENUM('en-GB', 'en-US', 'en-AU') DEFAULT 'en-GB', quiet_hours_start TIME, quiet_hours_end TIME, quiet_hours_music_only BOOLEAN DEFAULT false, updated_by_parent_id UUID, updated_at TIMESTAMPTZ );

Voice Technology

MVP Pre-recorded audio assets served per locale. File naming convention: q_{id}_{locale}.mp3
Fallback Web Speech API speechSynthesis — used when a pre-recorded asset is unavailable or the device CDN is blocked. Rate and voice filtered by voice_locale and voice_speed.

Quiet Hours Logic

Server-side If NOW()::TIME BETWEEN quiet_hours_start AND quiet_hours_end, audio assets are not served. Enforced at CDN origin; no client-side bypass is possible.
Client-side TTS blocked via a pre-session flag injected into the session token payload: audio_allowed: false. All speechSynthesis.speak() calls are no-ops when the flag is false.

Privacy

No audio recordings are ever made on-device or server-side. No voice biometrics are collected. Voice locale is a purely functional preference (accent selection) and is not stored as identifying information.

WCAG 1.4.2 Compliance

Parent can disable all audio via the master toggle. Child can stop readout mid-play using an in-session stop button (accessible via keyboard and pointer). Background music never auto-starts above the user's system volume.

Child Control Boundary

Child-adjustable: Voice speed only, if voice_speed_locked = false. All other controls are parent-only and are not surfaced in the child UI.

Apply Timing

Settings are persisted immediately on save but take effect at the next session start. An in-progress session is not disrupted — changes are queued for the next session_init API call.
