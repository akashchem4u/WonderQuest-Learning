Rules
  • 1 A Fail status automatically creates a linked feedback_item in the feedback system with severity derived from the feature's P-level.
  • 2 All P0 features must be tested on every device before any release is gated as ready. Untested P0 cells block release.
  • 3 Untested cells appear as amber warnings in the release checklist — treated as a blocking risk until resolved.
  • 4 A full coverage report (JSON + CSV) is exported automatically as part of the release gate workflow and archived in the release artifact store.
  • 5 No child PII is stored or transmitted. All data in device_coverage is technical metadata only; tester identity uses owner-role user IDs.
