📋 play-answer-cards-mobile-v3 · Engineering Spec

Mobile Differences from Desktop

  • No :hover — uses :active { opacity: 0.75 } for tap feedback (touch-appropriate)
  • -webkit-tap-highlight-color: transparent on all cards — removes default browser tap highlight
  • Checkmark badge is a <div class="ac-check"> child (not CSS ::after) — more reliable on mobile browsers
  • Reduced padding (14px vs 16px), slightly smaller font (0.84rem vs 0.88rem)
  • Smaller sizes: ac-letter=76px, ac-numeral=64px, ac-standard=90px (vs 80/70/100 desktop)

Touch Target Compliance

  • All card variants meet iOS Human Interface Guidelines minimum of 44×44px
  • Compact row: 56px min-height (above 44px)
  • Letter: 76×76px
  • Standard: 90px min-height
  • Do NOT reduce below 44px for any interactive element

State Classes (same as desktop)

  • .correct — green, checkmark shown (via display: block on .ac-check child)
  • .wrong — red border + bg + shake animation
  • .dimmed — 35% opacity, pointer-events none
