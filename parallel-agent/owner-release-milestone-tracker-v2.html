Spec — owner-release-milestone-tracker-v2
Owner Only Item 251
Data Schema
Primary table
release_milestones
release_milestones( id UUID PRIMARY KEY, release_id UUID REFERENCES releases(id), title TEXT NOT NULL, owner_name TEXT, status ENUM('done','in_progress','at_risk','pending','blocked'), due_date DATE, completed_at TIMESTAMPTZ, notes TEXT )
Decision log table
release_decisions
release_decisions( id UUID PRIMARY KEY, release_id UUID REFERENCES releases(id), decision_text TEXT NOT NULL, decided_by TEXT, decided_at TIMESTAMPTZ DEFAULT NOW() ) -- Append-only; no UPDATE or DELETE permitted
At-Risk Trigger Logic
Trigger condition
status = 'in_progress' AND due_date <= NOW() + INTERVAL '3 days'
Automatically promotes row display to amber "AT RISK" badge with tooltip. Does not mutate status column — computed at read time.
Blocked state
Requires explicit status = 'blocked' set by owner. Triggers P1 alert immediately (see Notifications).

Milestone Notifications
🔔
At-risk daily digest
Any milestone matching the at-risk trigger sends a daily summary notification to the owner. Batched at 08:00 local time.
🚨
Blocked milestone — P1 alert
Immediate push + email to owner when any milestone transitions to status = 'blocked'. Bypasses digest batching.
✅
Milestone completion
Completion of any milestone triggers readiness score recalculation and updates the release tracker banner in real time.
Access Control
Visibility
Owner Only Not visible to teachers, parents, or content team. Requires role = 'owner' claim in JWT.
Decision log
Append-only. Only owner role may add decisions. No edit or delete operations permitted on historical decisions.
Milestone mutations
Status updates, completion marks, and notes edits restricted to owner role. Read-only queries available to internal analytics role.
