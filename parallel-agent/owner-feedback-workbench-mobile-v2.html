Component: owner-feedback-workbench-mobile-v2.html
Mobile version of feedback workbench. List → detail navigation pattern. All same data and actions as desktop — compressed for single-column.

Navigation Pattern

Queue list is the default view (reached via Feedback tab in bottom nav). Tapping a feedback card pushes to full-screen detail view. Back arrow returns to queue. Deep links from push notifications open directly to item detail.

Privacy Rules

School name (and teacher role) shown in both list and detail views. Individual teacher name is NOT shown. Owner reply is delivered via platform notification panel — not email or SMS. Reply textarea placeholder reinforces this rule.

P0 Behaviour on Mobile

P0 items are always shown at top of list regardless of active filter. Push notification for P0 items fires immediately on submission (FCM/APNs). P0 banner also shown on owner home screen (non-dismissible).

Actions

All 5 actions available: Resolve (closes item, optional note) / Bump priority (P3→P2→P1→P0) / Escalate to on-call (fires PagerDuty) / Won't Fix (requires reason) / Add to Roadmap (links to roadmap ticket). Actions are flex-wrap row of compact buttons fitting mobile width.
