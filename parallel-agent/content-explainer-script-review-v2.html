Component

ExplainerScriptReview — Content team tool for reviewing voice coach explainer scripts. Ensures child-safe language, positive framing, and adventure style across all script types before audio recording and publication.


Route: /content/scripts/[script_id]/review

Auth: Content Team role required. No child PII in scripts.

Child Safety Language Rules

Prohibited terms are detected automatically. All scripts must pass before approval.

Prohibited Term Applies To Required Replacement
wrong All types (mandatory: error_recovery) "Let's try a different path!" / "Keep exploring!"
incorrect All types (mandatory: error_recovery) "Let's try again!" / "Every explorer keeps going!"
mistake All types (mandatory: error_recovery) "A new discovery!" / "Part of the adventure!"
failed / fail All types "Not yet — explorers keep going!"
bad All types Reframe with positive observation
stupid All types Remove entirely; never address ability negatively
Error Recovery MUST: use positive, adventure framing. Approved phrases: "Let's try a different path!", "Every explorer keeps going!", "Let's try again!", "Keep going, Explorer!", "That's part of the adventure!". NEVER use: wrong, incorrect, mistake.
Celebration scripts must be unconditionally positive — no qualifiers, no conditions, no comparisons.
Hint scripts must not create anxiety — no urgency language, no "hurry", no "you must".

Status Lifecycle

draft  →  in_review  →  approved  →  published

  • draft — Initial creation; editable by script authors.
  • in_review — Submitted to content team; auto-flag scan runs on submission.
  • approved — Content team sign-off; child_safety_approved=TRUE required.
  • published — Live in app; audio_recorded=TRUE required to publish.

6 Business Rules

1 error_recovery scripts must NEVER contain the words "wrong", "incorrect", or "mistake" — the Approve button is disabled until the script passes the auto-flag scan.
2 child_safety_approved must be TRUE before a script can transition from approved to published.
3 audio_recorded must be TRUE before a script can reach published status.
4 celebration scripts must be unconditionally positive — no conditional language ("if you try", "when you do better") is permitted.
5 hint scripts must not contain anxiety-inducing language such as "hurry", "you must", "time is running out".
6 No child PII (names, ages, school names, progress data) may appear in any script_text field; scripts are shared across all child accounts in the same band.
