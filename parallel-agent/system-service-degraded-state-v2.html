Detection
Route health check polls every 60 s (drops to 10 s during an active incident). Results written to platform_status table by the health check job.
Degraded Trigger
p95_latency_ms > 300 OR error_rate > 0.005 on any monitored route triggers status = 'degraded'.
Child Isolation
Child play session never shows system degradation UI. Local-first architecture ensures play continues uninterrupted. Errors are absorbed silently and surfaced only in operator logs.
Local-first During Degraded
IndexedDB cache serves questions. Answers queued in sync buffer. XP / stars applied optimistically. "Saving locally" chip shown to parents only — never to children.
Auto-recovery
When health check returns to normal thresholds for 3 consecutive polls: banner auto-dismisses + green "All systems restored" flash shown for 3 s.
Persona Routing
user.persona field determines copy level and which UI elements render. Child persona → degradation UI suppressed entirely. Owner persona → full technical detail at all levels.
Cached Data Timestamp
All degraded data views display "As of [HH:MM]" to set accurate user expectations. Timestamp sourced from platform_status.updated_at.
Copy Principles
Children: positive only — no "error", "fail", "problem". Parents: "temporary issue" framing, no jargon. Teachers: operational impact language. Owner: technical — p95, error rate, route names.
DB Schema — platform_status
CREATE TABLE platform_status (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  status TEXT NOT NULL CHECK(status IN ('ok', 'slow', 'degraded', 'outage')),
  affected_services TEXT[], -- e.g. ARRAY['reports','search']
  p95_latency_ms INTEGER,
  error_rate NUMERIC(6,4),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_by TEXT NOT NULL DEFAULT 'health-check-job'
);
