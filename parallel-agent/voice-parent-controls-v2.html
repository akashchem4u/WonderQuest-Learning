⚙️ Engineering Hint
Parent audio/voice controls hub. Parents can set voice coach, speed, quiet hours, and per-child locks from one place. All changes write to child_audio_prefs via optimistic UI → sync queue. Changes take effect on next child session start. Schema: parent_voice_control_changes { id, parent_id, child_id, setting_key, old_value, new_value, changed_at }. Privacy: child first name only; no accuracy data; no DOB.

Behaviour Rules

  • Coach change invalidates child session cache — next login fetches fresh voice pack.
  • Speed is stored per-child, not per-device. Changing device does not reset speed.
  • Quiet hours use local device time (parent's timezone at save time).
  • Child-controlled items: speed (when unlocked), replay readout, stop readout.
  • Parent-controlled items: coach, speed-lock, quiet hours window.
  • No accuracy data surfaced in this view — WonderQuest privacy policy.
  • All writes are optimistic → sync queue; UI confirms immediately.
  • Changes take effect on the child's next session start, not mid-session.
