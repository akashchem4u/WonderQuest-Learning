Keyboard Navigation — Play Screen Tab Order

Full keyboard operability per WCAG 2.1.1. Arrow keys navigate within answer group; Tab moves between groups.

1
Speaker / Read Aloud button
First focusable element. Press Enter or Space to trigger audio playback of the question text.
2
Answer button A (Five)
Tab moves here. ↓ / ↑ arrow keys cycle through answers A–D without leaving the group.
3
Answer buttons B, C, D
Navigate with ↓ / ↑. Press Enter or Space to select the focused answer. Selected answer is announced via aria-live="assertive".
4
Next Question button (shown after answer)
Appears after answer is selected. Tab moves focus here automatically. Press Enter to advance.
5
Exit / Back button
Always reachable via Tab from top bar, or press Escape anywhere on the play screen to exit to the home hub.
6
Accessibility Settings button
Top-right control. Toggle high-contrast mode, font size, audio settings. Press Escape to close settings panel and return focus to this button.

Screen Reader ARIA Patterns

Required ARIA markup for the HC play screen. All dynamic content regions must be labelled so assistive technologies announce changes.

CSS Override Pattern — .mode-high-contrast
/* Applied via .mode-high-contrast on <html> */ .mode-high-contrast { --bg-base: #000000; --bg-card: #0a0a0a; --text-primary: #ffffff; --accent: #ffff00; --border: 2px solid #ffffff; --btn-bg: #000000; --btn-text: #ffffff; --btn-hover-bg: #ffff00; --btn-hover-text: #000000; } /* Strip all visual complexity */ .mode-high-contrast * { animation: none !important; transition: none !important; background-image: none !important; text-shadow: none !important; box-shadow: none !important; } /* Universal focus ring */ .mode-high-contrast *:focus { outline: 3px solid #ffff00 !important; outline-offset: 2px !important; } /* Also respect OS-level prefers-contrast */ @media (forced-colors: active) { .answer-btn { forced-color-adjust: none; /* Preserve our explicit HC colours in Windows HC mode */ } }
Non-Colour Indicators — WCAG 1.4.1 Compliance

Correct answer

Never rely on green colour alone. Always pair with a text symbol and/or shape change.

✓ Correct!
= #00ff00 background (colour)
+ ✓ checkmark symbol (shape)
+ "Correct!" text (text)

Incorrect answer

Red border alone is not sufficient. Pair with ✗ symbol and explanatory text.

✗ Try again
= #ff6b6b border (colour)
+ ✗ cross symbol (shape)
+ "Try again" text (text)

Progress / active state

Progress is expressed as "Question 4 of 8" text — never as colour-only dots or bars without numeric labels.
