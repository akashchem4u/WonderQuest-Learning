Auto-trigger implementation
const DISCLAIMER_KEYWORDS = { outcomes: 'learning-effectiveness', efficacy: 'learning-effectiveness', comparison: 'comparison', benchmark: 'comparison', learning: 'standard', improvement: 'standard', progress: 'standard' }; function checkDisclaimerTrigger(text) { const lower = text.toLowerCase(); for (const [keyword, type] of Object.entries(DISCLAIMER_KEYWORDS)) { if (lower.includes(keyword)) return type; } return null; } // Returns: 'standard' | 'comparison' | 'learning-effectiveness' | null // Evaluation order matches DISCLAIMER_KEYWORDS key order. // First match wins — no cascading.
Trigger scope
The checkDisclaimerTrigger() function is called against:
  • – Dashboard section headings and metric labels
  • – Report card titles and sub-headings
  • – Export / PDF document titles and section headers
  • – Chart axis labels and legend text

Checks run on adult personas only (parent / teacher / owner / content). Child persona is never shown comparison or benchmark data and does not receive disclaimer overlays.

Non-removable in exports
PDF / export rules
  • PDF generation sets include_disclaimer = true as a hardcoded constant — cannot be overridden by user settings, admin flags, or API parameters.
  • Variant 5 block is injected before the first content section and on the final page of every export.
  • Removing the disclaimer block from a PDF template is a build-time error; the CI pipeline checks for disclaimer presence in test exports.
Personas
Who sees disclaimers
Adult personas Parent, Teacher, Owner, Content — all disclaimer variants apply.
Child persona Never shown comparison or benchmark data. No disclaimer overlays rendered. Disclaimer checks are bypassed at the persona level before keyword scanning.
Version control
Re-acknowledgment on version change
  • Disclaimer text is versioned via disclaimer_version in app config (currently v2.1).
  • When disclaimer_version is incremented, all existing acknowledgment records are invalidated.
  • Users who previously acknowledged Variant 4 must re-sign with the new version text before accessing learning-effectiveness data again.
  • Version string is stored in disclaimer_displays.disclaimer_version and learning_effectiveness_acknowledgments.disclaimer_version.
