Video Asset Rules

  • Max clip length: 20 seconds
  • Dimensions: 16:9, rendered 350px wide in sheet
  • Always muted on start (browser autoplay policy)
  • Mute badge tap → video.muted = false
  • Always has captions track (<track kind="captions">)
  • ❌ No explainer clips longer than 20s

State Transitions

  • Loading → Playing: canplay event fires
  • Playing → Paused: child taps video area
  • Paused → Playing: child taps pause overlay
  • Playing → Ended: ended event fires
  • Ended → Playing: "Watch again" button
  • Any state → Dismissed: Skip or "Got it!" (after ended)

Progress Bar

  • Position: absolute; top: 0 of video container
  • Height: 4px; border-radius 0 2px 2px 0 on fill
  • Color: #9b72ff playing → #50e890 ended
  • Update via timeupdate event: (currentTime / duration) * 100%
  • ❌ Never show timestamp as primary time cue (anxiety)
  • Timestamp shown in small text, low opacity — secondary

Fallback

  • If video fails to load → render text explainer panel instead
  • Text panel: play-explainer-panel-v3.html
  • Failure detection: error event on <video>
  • No error message shown to child — silent swap to text panel
  • ✅ Skip always visible regardless of load/error state
  • ❌ Never show "Video failed to load" to child

"Got it!" Gating

  • Button disabled (opacity 0.45, pointer-events none) during Loading + Playing + Paused
  • Button enabled after: video ended OR child taps Skip
  • Skip always enabled (no gating)
  • "Watch again" only shown in Ended state
  • ❌ Never auto-advance to question after video ends
  • Child must tap "Got it!" to proceed

Wave Bars (Playing)

  • 5 bars, bottom-right of video container
  • Color: #9b72ff (band color)
  • wave-dance keyframe: height 4px → var(--max-h)
  • Staggered animation-delay per bar
  • Bars hidden in Paused + Ended states
  • Bars represent TTS/video audio — cosmetic only
