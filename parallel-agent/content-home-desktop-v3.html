Component: content-home-desktop-v3.html
Content CMS home page. Left sidebar navigation. 6-stat row. Blocked alert banner. Flagged skills grid. Recently published. Persona: curriculum/content team (NOT owner console).

Persona Distinction

This is the content team CMS, not the owner console. Content team members can: view all skills, edit skill content, flag/unflag skills, submit for review, unblock skills (after fix + approval). They cannot: view school-level data, see individual teacher/student names, access governance log, or access financial KPIs. Owner sees a read-only summary of content health from owner-content-health-desktop-v2.html.

Skill Status Flow

Published (live) → Under Review (auto-flag triggered) → Blocked (threshold exceeded, delivery_locked=true). Unblock path: fix in CMS → curriculum review approval → owner confirms → deploy → delivery_locked=false → back to Published. Content team initiates fix; owner confirms unblock.

Auto-Flag Thresholds

Skip rate >60% → Under Review. Wrong-answer-as-correct reports ≥ 2 → Under Review; ≥ 3 → Blocked. Asset 404 → Blocked immediately. Abort mid-skill rate >40% → Under Review. These thresholds can be adjusted by owner in Settings → Content Thresholds.

Left Sidebar Navigation

Sidebar provides direct access to filtered views: All Skills (full library, 284), Flagged (8 — all items needing attention), Blocked (3 — delivery locked), Under Review (5 — flagged but still live). Browse by subject (Maths/Literacy/Science) or band (P0–P3). Badge counts on Flagged/Blocked update in real time.
