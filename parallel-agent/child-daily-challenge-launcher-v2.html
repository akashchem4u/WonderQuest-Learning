DB Schema — daily_challenges
CREATE TABLE daily_challenges (
  id             UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  child_id       UUID NOT NULL REFERENCES children(id),
  challenge_date DATE NOT NULL,
  quest_id       UUID NOT NULL REFERENCES quests(id),
  status         ENUM('available','started','completed','skipped')
                 NOT NULL DEFAULT 'available',
  completed_at   TIMESTAMPTZ,
  xp_awarded     INT,            -- NULL until status = 'completed'
  UNIQUE (child_id, challenge_date)
);
Behaviour Rules
  • Challenge is optional — skipping carries zero penalty, no negative language is ever shown
  • No countdown timer, no streak pressure counter, no accuracy percentage displayed anywhere on this card
  • Challenge resets daily at midnight local time — a new row with status available is created server-side
  • Only one active challenge per child per day — enforced by the UNIQUE constraint on (child_id, challenge_date)
  • Quiet hours (midnight – 6:59 am local): card renders but the Accept CTA is hidden; replaced with a soft "Available after 7 am" message
  • xp_awarded is only written to the DB when status transitions to completed; it remains NULL for skipped or in-progress rows
  • Quest selected for the day must match child's current_band — no cross-band assignments
  • Privacy: only child_first_name (first name) is rendered — no last name, no email, no parent name
  • "Explore freely instead" link is always visible alongside the CTA — it routes to the main subject grid with no challenge context attached
  • All touch targets are ≥ 44 × 44 px; component meets WCAG 2.1 AA colour contrast for both text and interactive elements
  • Card is fully mobile-responsive; max-width 420 px centered on wide viewports, full-width on narrow
