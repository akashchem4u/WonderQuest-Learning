Spec — Completion Tracking & Skip Rules
Engineering reference for first-run logic, database flag, skip conditions, and language level constraints.
Completion Flag SQL
-- Flag first run as complete UPDATE children SET first_run_completed_at = NOW(), avatar_key = :selected_avatar, preferred_themes = :selected_themes WHERE id = :child_id;
Written only on Step 5 completion ("Start Exploring!" tap). If the user closes before Step 5, first_run_completed_at remains NULL and the flow is shown again next login.
Admin / Beta Reset SQL
-- Re-enable first run (beta session reset) UPDATE children SET first_run_completed_at = NULL WHERE id = :child_id;
Used in the Beta Session Reset Panel. Setting the flag to NULL forces re-display of the full first-run guide on next login. Beta testers may trigger this to replay the experience.
Step Skip Rules
Step Name Skip Condition Result
1 Welcome Never skipped Always shown
2 Avatar Avatar already confirmed during household onboarding Auto-skip
3 Themes preferred_themes.length ≥ 1 Auto-skip
4 Quest Preview Never skipped Always shown
5 Go! Never skipped Always shown
Abbreviated flow: If both steps 2 and 3 are skippable (avatar confirmed + themes set), show Steps 1 → 4 → 5 only (3-step abbreviated flow). Progress dots update to reflect 3 dots instead of 5.
