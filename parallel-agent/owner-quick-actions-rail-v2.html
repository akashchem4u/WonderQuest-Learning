Component: owner-quick-actions-rail-v2.html
Persistent left icon rail. 56px wide. Always visible on desktop. 9 destinations + dividers. Badge counts. Active state. Tooltip on hover.

Badge Logic

Feedback badge: count of P0+P1 open items. Route Health badge: count of DOWN routes (red badge); degraded routes shown as amber dot on icon. Release Gate: green dot (gate open) or amber dot (gate score < 90) — no number badge. Content Health: amber dot if any blocked skills. Governance/Roadmap/Calendar: no badge.

Active State

Current page's rail item highlighted: background rgba(80,232,144,.12), icon color #50e890. During P0 incident: Route Health item turns red (background rgba(248,81,73,.1), icon color #f85149). These states can co-exist (e.g., active = Route Health + P0 incident → red active state).

Tooltip

Tooltip appears 200ms after pointer enters the icon. Disappears immediately on pointer leave. Tooltip text includes badge context (e.g., "Feedback (6 open)" not just "Feedback"). Tooltips are suppressed if the section label is already visible (wider layout).

Mobile

Rail is not shown on mobile — replaced by bottom tab bar (5 items: Home / Routes / Feedback / Release / More). The "More" tab contains the items not in the primary 4: Content Health / Command Center / Roadmap / Calendar / Governance Log / Settings.
