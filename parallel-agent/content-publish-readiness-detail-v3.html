Component: content-publish-readiness-detail-v3.html
Per-skill publish readiness checklist. 4 categories (Content/Quality/Assets/Sign-off = 100). Threshold = 90. Blocked vs warned vs passed items. Two sign-offs required (curriculum + owner). Not the same as Release Gate.

Publish Readiness vs Release Gate

Publish Readiness (this component) = per-skill quality gate managed by curriculum team. Applies to individual skills going from Draft → Published. Release Gate (owner-release-gate-*.html) = per-release quality gate managed by owner. Applies to the whole version release (vX.Y). A skill can be publish-ready but still only deployed in a batch release.

Score Categories

Content Completeness (30): Title set (+10), band set (+5), subject set (+5), difficulty set (+5), min 5 questions (+5). Question Quality (30): All questions have correct answer (+15), no duplicate text (+5), answer options make sense (curriculum review flag, +10). Asset Completeness (20): All referenced assets present (+20; partial score if some questions are text-only by design). Review Sign-off (20): Curriculum reviewer approval (+15), owner publish action will add final (+5 from action itself).

Publish Action

When score ≥ 90 AND curriculum sign-off collected: "Request Owner Publish" button becomes active. Owner receives a notification with link to this readiness detail page. Owner reviews, then clicks Publish → review_status=Published, delivery_locked=false → skill enters live delivery pool. The owner publish action is logged to the governance log (Content category).
