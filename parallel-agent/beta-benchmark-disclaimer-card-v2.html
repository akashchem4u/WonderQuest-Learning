beta-benchmark-disclaimer-card-v2.html — Benchmark disclaimer component.
Shown whenever: (1) beta readout shows engagement/retention metrics, (2) any metric is compared to an external benchmark, (3) learning effectiveness claims are made.
Context: beta data is from a small non-representative sample. No causal learning claims can be made from beta data alone. Legal review required before using beta data in marketing.
4 disclaimer variants: Metric disclaimer, Comparison disclaimer, Learning effectiveness disclaimer, Sample size disclaimer.

Display Rules
Disclaimer Type Trigger Condition Trigger Method Acknowledgment Required?
Metric disclaimer ALL beta readout metrics, automatically Auto No — informational only
Comparison disclaimer Any metric compared to an external benchmark Auto No — informational only
Learning effectiveness Keywords: "learning", "improvement", "outcomes", "efficacy", "effectiveness" in metric context Keyword match Yes — checkbox required
Sample size n < 30 for any specific cohort/segment metric Auto (n threshold) No — informational only
Legal Review Requirements
⚠️
External communications using beta metrics — requires legal sign-off before publication. All beta metrics must be reviewed by legal before appearing in any external document, presentation, or marketing material.
⛔
Learning effectiveness claims — prohibited until independent evaluation is completed. No claim of "WonderQuest improves learning outcomes" may be made using beta data.
⚠️
NPS / satisfaction data — can be used internally. Cannot be cited publicly as "X% of users recommend WonderQuest" without legal review and caveat language.
⚠️
Marketing use — ALL beta metrics must be reviewed by legal before publication. This includes case studies, app store copy, investor decks, and press materials.
✅
Internal team readouts — beta metrics may be shared freely within the WonderQuest team for product decisions, prioritisation, and roadmap planning.
Acknowledgment Rules
Learning Effectiveness — Checkbox Acknowledgment (required)
🎓 Learning Effectiveness Disclaimer
WonderQuest has not been independently evaluated for learning outcomes during the beta period. Metrics in this section are engagement proxies, not validated measures of learning. No causal claims about learning improvement may be made from this data.

All other disclaimer types (metric, comparison, sample size) are informational only — displayed automatically with no acknowledgment required.

Database Schema
-- Tracks every disclaimer shown and whether it was acknowledged
CREATE TABLE disclaimer_displays (
  id               UUID          PRIMARY KEY DEFAULT gen_random_uuid(),
  disclaimer_type  TEXT          NOT NULL,  -- 'metric' | 'comparison' | 'learning_effectiveness' | 'sample_size'
  context          TEXT,                          -- where it was shown (e.g. 'owner_pilot_readout', 'engagement_dashboard')
  user_id          UUID          REFERENCES users(id),
  acknowledged_at  TIMESTAMPTZ  NULLABLE,      -- null = informational only / not yet acknowledged
  created_at       TIMESTAMPTZ  DEFAULT NOW()
);
Keyword Trigger List (Learning Effectiveness)

The following keywords, when detected in metric labels, section titles, or context strings, trigger the learning effectiveness disclaimer automatically:

  • learning — "learning outcomes", "learning improvement", "learning gains"
  • improvement — "skill improvement", "reading improvement"
  • outcomes — "educational outcomes", "student outcomes"
  • efficacy — "program efficacy", "curriculum efficacy"
  • effectiveness — "learning effectiveness", "teaching effectiveness"

Match is case-insensitive substring match. False positive rate should be low given product vocabulary. Log all triggered instances to disclaimer_displays for audit.

Export / PDF Rules
  • Learning effectiveness disclaimer banner: always rendered in exports, cannot be hidden or removed by presenter
  • Metric disclaimers: rendered as footer footnotes in PDF exports
  • Comparison disclaimers: rendered inline below the relevant metric in exports
  • Sample size disclaimers: rendered inline as a note below the metric value
